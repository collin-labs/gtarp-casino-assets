## 2.1 Blackjack Classico (Standard) - A Base

**Mecanica padrao (encontrada em TODOS os cassinos online e fisicos):**
- Objetivo: chegar o mais perto de 21 sem ultrapassar (bust). Vencer a mao do dealer.
- Cartas: numeros valem o numero, figuras (J/Q/K) valem 10, As vale 1 ou 11.
- Jogador recebe 2 cartas face-up. Dealer recebe 1 face-up, 1 face-down (hole card).
- Acoes do jogador: HIT (mais carta), STAND (parar), DOUBLE DOWN (dobrar aposta, 1 carta), SPLIT (separar par), SURRENDER (desistir, perde metade).
- Dealer DEVE hit ate 17+. Regra varia: soft 17 (S17 = stand) vs hard 17 (H17 = hit).
- Blackjack natural (As + 10/J/Q/K nos 2 primeiros) paga 3:2 (standard) ou 6:5 (desfavoravel).
- Insurance: quando dealer mostra As, jogador pode apostar metade que dealer tem BJ. Paga 2:1.

**Configuracao padrao da industria (2025-2026):**
- 6-deck ou 8-deck shoe.
- Dealer stands on soft 17 (S17) = melhor para jogador.
- Double on any two cards. Double after split (DAS) permitido.
- Split ate 4 maos. Resplit Aces geralmente NAO permitido.
- Blackjack paga 3:2 (NUNCA aceitar 6:5 — aumenta house edge em 1.39%).
- No surrender na maioria das mesas (late surrender em algumas).

## 2.2 Variantes Evolution Gaming (Padrao Global)

### Infinite Blackjack
- Unlimited players jogam a MESMA mao inicial (2 cartas).
- Apos deal, cada jogador decide INDIVIDUALMENTE (hit/stand/double/split).
- Resultados diferentes para cada jogador com as mesmas 2 cartas iniciais.
- 4 side bets opcionais: Any Pair, 21+3, Hot 3, Bust It.
- 6 Card Charlie: 6 cartas sem bust = win automatico (ate contra BJ do dealer).
- RTP: 99.47%.

- 8 decks COM 9s e 10s REMOVIDOS (64 cartas a menos no shoe).

## 3.2 Componente B - Deck Shuffle (Fisher-Yates)

**Algoritmo Fisher-Yates (Knuth Shuffle):**
```javascript
function fisherYatesShuffle(array) {
  const shuffled = [...array];
  for (let i = shuffled.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [shuffled[i], shuffled[j]] = [shuffled[j], shuffled[i]];
  }
  return shuffled;
}
```

**Para Provably Fair:** O `Math.random()` e substituido por bytes do HMAC-SHA256:
```javascript
function provablyFairShuffle(deck, serverSeed, clientSeed, nonce) {
  const shuffled = [...deck];
  for (let i = shuffled.length - 1; i > 0; i--) {
    const hmac = createHmac('sha256', serverSeed);
    hmac.update(`${clientSeed}:${nonce}:${i}`);
    const hash = hmac.digest();
    const j = hash.readUInt32BE(0) % (i + 1);
    [shuffled[i], shuffled[j]] = [shuffled[j], shuffled[i]];
  }
  return shuffled;
}
```

**52 cartas por deck × 8 decks = 416 cartas.** Fisher-Yates faz 415 swaps. Com HMAC, sao 415 createHmac calls. Para otimizar: usar cursor pattern do Stake.com (4 HMACs geram 32 bytes = 8 indices por HMAC, cobrindo 416 posicoes em ~52 HMACs).

- AnimatePresence: funcional para card entry/exit.
- layout animations: funcional para reposicionamento de cartas na mao.
- `animate={{ rotateY }}`: funcional para card flip.

### Comunicacao NUI <-> Server para Blackjack
```
[Jogador clica BET + DEAL no NUI]
  -> fetchNui('casino:bj:deal', { bet, sideBets, clientSeed })
  -> [Lua Client] TriggerServerEvent('casino:bj:deal', source, bet, sideBets, clientSeed)
  -> [JS Server Handler] valida saldo, gera deck via Provably Fair, deal 4 cartas
  -> callback: { playerCards, dealerUpCard, dealerHoleCardHash, hashPre, handId }
  -> [NUI React] anima deal, mostra cartas

[Jogador clica HIT]
  -> fetchNui('casino:bj:action', { handId, action: 'hit' })
  -> [Server] distribui proxima carta do deck pre-shuffled
  -> callback: { card, handTotal, isBust }
  -> [NUI React] anima carta nova, atualiza total

[Jogador clica STAND]
  -> fetchNui('casino:bj:action', { handId, action: 'stand' })
  -> [Server] resolve mao do dealer (hit ate 17+), calcula resultado
  -> callback: { dealerCards, dealerTotal, result, payout, serverSeed }
  -> [NUI React] revela hole card (flip), anima dealer hits, mostra resultado
```
