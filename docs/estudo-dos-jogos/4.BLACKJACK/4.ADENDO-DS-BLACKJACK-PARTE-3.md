# SECAO 2 - ATMOSFERAS POR ESTADO DO JOGO

## 2.1 Estado BETTING (Idle - Posicionando Apostas)

Atmosfera de antecipacao calma. Mesa vazia, chips disponiveis:

- **Mesa:** feltro #1a472a com vinheta inset, areas de bet tracejadas rgba(212,168,67,0.2). Shoe visivel no canto.
- **Chips:** 5 denominacoes no selector inferior. Hover scale 1.1 + glow dourado. Click adiciona ao stack na area de bet. Som `chip_place.mp3` a cada posicionamento.
- **Side bets:** areas PP e 21+3 com label sutil rgba(212,168,67,0.5). Hover intensifica border.
- **PF Badge:** visivel, hash truncado, cursor pointer.
- **Botao DEAL:** verde CTA, desabilitado enquanto bet = 0. Ativo: glow verde sutil.
- **Sons:** `chip_place.mp3` (ka-chunk 100ms) a cada chip. `chip_stack.mp3` (click leve 60ms) se stack > 1.
- **Sem cartas na mesa.** Dealer area mostra texto "DEALER" sutil rgba(255,255,255,0.1).

## 2.2 Estado PLAYING (Player Turn - Decisoes)

Atmosfera de tensao e controle. Cartas na mesa, botoes ativos:

- **Cartas:** dealer com 1 face up + 1 face down. Player com 2 face up. Badge total atualizado.
- **Botoes acao:** 5 botoes ativos/disabled conforme regras. HIT verde brilhante, STAND dourado solido. Disabled: opacity 0.35.
- **Side bets:** resultado ja avaliado. WIN: glow verde + payout. LOSE: fade opacity 0.4.
- **Timer:** opcional server-side (30s max turn). Se ativo: JetBrains Mono dourado, urgente <5s vermelho pulsante.
- **Sons:** `card_slide.mp3` + `card_flip.mp3` a cada HIT. `sidebet_win.mp3` se side bet ganhou.
- **Split:** se splitou, 2 maos lado a lado, borda verde na ativa, dot indicador animando entre maos.

## 2.3 Estado DEALER_TURN (Observando - Controles Dimmed)

Atmosfera de antecipacao passiva. Jogador assiste:

- **Controles:** TODOS 5 botoes desabilitados (opacity 0.2). Texto "VEZ DO DEALER" sutil no centro.
- **Hole card:** flip dramatico 400ms spring 300/25 com glow dourado transitorio. Som `dealer_reveal.mp3`.
- **Dealer hits:** cartas saem do shoe com stagger 500ms. Badge total atualiza com bounce a cada carta.
- **Badge dealer:** cor muda: #A8A8A8 (< 17, vai hitar) -> #00E676 (17-21, para) -> #FF1744 (> 21, bust).
- **Sons:** `dealer_reveal.mp3` (flip dramatico), `card_flip.mp3` por hit, `bust.mp3` se dealer bust.
- **Tensao crescente:** cada carta do dealer aumenta a expectativa. Pausa 500ms entre cartas amplifica.

## 2.4 Estado RESULTADO (Overlay + Feedback)

5 atmosferas distintas por tipo de resultado:

**Vitoria normal:**
- Overlay `.bj-overlay--win`: blur 6px, fade-in spring 300/25.
- Titulo "VOCE VENCEU!" verde #00E676 com text-shadow glow.
- Countup payout 0 -> valor em 1.5s (ease-out quint).
- Saldo: `bjBalanceWin` flash APOS countup terminar.
- Som: `win_normal.mp3` (chime duplo 500ms).

**Bust:**
- Overlay `.bj-overlay--bust`: blur 4px, fade-in 0.3s.
- Titulo "ESTOUROU!" vermelho #FF1744 com shake 0.4s.
- Sem confetti. Sem countup (valor negativo direto).
- Saldo: `bjBalanceLose` flash 1x vermelho sutil.
- Som: `bust.mp3` (buzz descendente 400ms).

**Blackjack Natural:**
- Overlay `.bj-overlay--natural`: blur 8px, scale spring 200/20.
- Titulo "BLACKJACK!" dourado #FFD700 com shimmer sweep 1.5s.
- Confetti GoldParticles 30 particulas 2.5s.
- Countup payout 0 -> 3:2 do bet em 1.5s.
- Saldo: `bjBalanceWin` flash.
- Som: `blackjack_hit.mp3` (fanfarra ascendente 1.5s).

**Push:**
- Overlay `.bj-overlay--push`: blur 4px, fade-in 0.3s.
- Titulo "EMPATE" dourado #FFD700 tom neutro.
- Chips retornam ao saldo com slide up.
- Sem confetti, sem shake, sem countup.
- Som: `push.mp3` (tom neutro breve 300ms).

**Surrender:**
- Overlay `.bj-overlay--surrender`: blur 4px, fade-in 0.3s.
- Titulo "DESISTENCIA" dourado sutil rgba(212,168,67,0.6).
- Metade das chips retorna. Cartas fade out.
- Payout positivo (metade devolvida) em #D4A843.
- Som: `lose.mp3` (tom descendente suave 400ms).

# SECAO 3 - TIMELINES EXATAS

## 3.1 Timeline Deal Dispatch (desde DEAL ate cartas posicionadas)

```
t=0ms       Jogador clica DEAL (bet > 0, saldo validado).
            Botao DEAL transiciona para disabled.
            Chip selector desaparece (AnimatePresence exit).

t=0-20ms    fetchNui('casino:bj:deal', { bet, sideBets }) ->
            Lua Client -> TriggerServerEvent

t=20-80ms   Server valida saldo, deduz bet, gera deck (se necessario),
            distribui 4 cartas, avalia side bets.

t=80ms      Server retorna: { cards, sideBetResults, phase }
            Client inicia animacao.

t=100ms     Carta 1 (player): slide do shoe + flip face up.
            Som: card_slide.mp3 + card_flip.mp3
            Duracao: 400ms slide + 200ms flip overlap

t=350ms     Carta 2 (dealer): slide do shoe + flip face up.
            Som: card_slide.mp3 + card_flip.mp3

t=600ms     Carta 3 (player): slide do shoe + flip face up.
            Som: card_slide.mp3 + card_flip.mp3

t=850ms     Carta 4 (dealer): slide do shoe, permanece face DOWN.
            Som: card_slide.mp3 (sem card_flip — hole card)

t=1050ms    Deal completo. 4 cartas posicionadas.
            Player badge total aparece com bounce.

t=1100ms    Side bets avaliadas visualmente:
            PP area: WIN -> glow verde + payout text | LOSE -> fade
            21+3 area: WIN -> glow verde + sub-result | LOSE -> fade
            Som: sidebet_win.mp3 se algum ganhou.

t=1300ms    Verificar: dealer upcard == As?
            SIM -> fase INSURANCE_PROMPT (modal aparece)
            NAO -> verificar BJ natural player/dealer

t=1300ms    Verificar: player tem BJ natural?
            SIM -> resolver imediatamente (overlay BLACKJACK)
            NAO -> fase PLAYER_TURN (botoes acao ativam)

t=1500ms    Botoes de acao aparecem:
            Framer Motion: stagger 0.05s entre botoes.
            HIT, STAND sempre ativos.
            DOUBLE: ativo se saldo >= bet.
            SPLIT: ativo se par + saldo >= bet.
            SURRENDER: ativo (Late Surrender, 2 cartas iniciais).

TOTAL deal cycle: ~1.5s ate botoes ativos
```

## 3.2 Timeline Dealer Reveal + Hits

```
t=0ms       PLAYER_TURN encerra (stand, bust, ou double auto-stand).
            Botoes de acao transicionam para disabled (opacity 0.2).
            Texto "VEZ DO DEALER" aparece sutil.

t=300ms     Hole card flip inicia:
            Framer Motion: animate={{ rotateY: 0 }}
            Spring stiffness 300, damping 25.
            Glow transitorio: box-shadow 0->20px->0 rgba(212,168,67,0.4)
            Som: dealer_reveal.mp3

t=700ms     Hole card revelada. Badge dealer atualiza total.
            Bounce: animate{{ scale: [1, 1.15, 1] }} 0.3s

t=700ms     Verificar: dealer total >= 17?
            SIM (17-21) -> pular para resultado
            SIM (bust > 21) -> pular para resultado
            NAO (< 17) -> dealer hit

t=1200ms    Dealer hit #1: carta slide do shoe + flip.
            Stagger: 500ms apos reveal.
            Badge atualiza com bounce. Cor muda conforme total.
            Som: card_slide.mp3 + card_flip.mp3

t=1700ms    Verificar novamente: total >= 17?
            Repetir hits ate condicao.
            Stagger 500ms entre cada hit.

t=final     Dealer para (stand ou bust).
            Delay 500ms antes do resultado.

t=final+500ms  Overlay resultado aparece (conforme tipo).

TOTAL dealer turn: ~1.5-3s (1 reveal + 0-3 hits)
```

## 3.3 Timeline Resultado (overlay + payout + saldo)

```
t=0ms       Resultado determinado pelo server.
            Overlay aparece: Framer Motion spring scale+opacity.

t=0ms       Titulo aparece (ex: "VOCE VENCEU!").
            Se BJ Natural: shimmer sweep inicia (1.5s).
            Se Bust: shake x [-3,3,-2,2,0] (0.4s).

t=200ms     Se BJ Natural: confetti GoldParticles inicia.
            30 particulas, life 2.5s, gravity 0.18.

t=300ms     Countup payout inicia: 0 -> valor em 1.5s.
            Incremento a cada 16ms.
            Ease-out quint: 1 - Math.pow(1-t, 5).

t=1800ms    Countup termina. Valor final exibido.

t=1900ms    Saldo atualiza:
            Win: bjBalanceWin flash (0.8s verde->branco alternando).
            Lose: bjBalanceLose flash (0.5s verde->vermelho 1x).
            Push: sem flash (saldo nao muda).
            Surrender: bjBalanceWin flash sutil (metade devolvida).

t=2200ms    Botoes pos-resultado aparecem:
            NOVA MAO + VOLTAR com Framer Motion fade-in.

t=2200ms+   Cooldown 3s server-side.
            Jogador pode clicar NOVA MAO apos cooldown.

TOTAL resultado: ~2.2s ate botoes visiveis + 3s cooldown
```
