## 2.1 Formatos de Giveaway/Raffle em Cassinos Online

A pesquisa identificou 4 formatos distintos de sistemas de giveaway operando em cassinos online em 2025-2026:

### Formato A — Weekly Raffle Pool (Stake, Roobet)

O formato dominante no mercado. Mecanica central: jogadores acumulam tickets durante a semana via wagering, e um draw ao vivo seleciona N vencedores que dividem um pool fixo.

Stake opera o modelo mais maduro: $75.000 semanais divididos entre 15 vencedores ($5.000 cada). Cada $1.000 apostado gera 1 ticket. O draw acontece ao vivo no stream do Eddie no Kick.com todo sabado as 14h GMT, usando o Google Random Number Generator oficial. Os tickets sao visiveis em tempo real na aba Promotions do perfil. O sistema aceita wagers de casino E sportsbook, e nao impoe limite de tickets — jogadores ativos podem acumular centenas de entries por semana. Premios pagos em Bitcoin direto na wallet apos verificacao KYC.

Roobet opera modelo similar com pool de $100.000 semanal e 100 vencedores selecionados por random drawing. A principal diferenca eh que Roobet usa wagering volume (nao quantidade fixa de $1k) para determinar entries.

A chave desse formato eh o elemento social: o draw ao vivo cria tensao, engajamento e comunidade. Jogadores assistem o stream torcendo pelo proprio numero. O countdown ate o sabado cria anticipacao durante toda a semana.

### Formato B — Item/Skin Raffle (CSGOEmpire, Gamdom, DatDrop)

Popular no ecossistema CS2/CSGO. A plataforma coloca 1 item especifico (skin, faca, arma) para raffle. Jogadores compram tickets para aquele raffle especifico. O ticket vencedor leva o item. Principio universal: mais tickets comprados = mais chances, mas o custo total dos tickets nao deve exceder o valor do item (senao nao vale a pena).

CSGOEmpire hospeda giveaways regulares em social media com free cases ao atingir milestones. Gamdom (founded 2016) oferece raffles, jackpot, roulette, crash e tradeup — tudo provably fair. Duelbits (founded 2020) combina raffles com blackjack, roulette, crash e dice duels.

A diferenca fundamental do Formato A: aqui o jogador escolhe EM QUAL raffle participar (cada um tem item e preco diferente), enquanto no Stake o raffle eh unico e semanal.

### Formato C — Ticket Diario/Task-Based (Skinflow, WRewards)

Modelo freemium: jogadores recebem tickets gratuitos diariamente (1 por dia) e ganham tickets adicionais por completar tarefas (follow social media, share post, email confirmation, sell skins). Giveaways diarios, semanais e mensais com premios diferentes. Winners notificados na pagina, prize history publica.

Skinflow exemplifica: claim 1 ticket a cada 24h, complete bonus tasks para entries extras, participe de daily/weekly/monthly giveaways. Sem custo para participar, mas engajamento (tarefas) aumenta chances.

### Formato D — Drum Draw ao Vivo (LCB Awards, eventos)

Modelo de evento: tickets acumulados por participacao (ex: 1 vote = 1 ticket, max 8 votes/dia em 8 categorias). Vencedores selecionados por random drawing from drum durante livestream. Exemplo real: LCB Awards 2025 com 45 vencedores de pool de $5.000 + 10 consolation prizes de pool de $1.000. Modelo hibrido onde participacao ativa (votar) gera tickets.

## 3.2 Justificativas (5 razoes concretas)

1. **Engagement de longo prazo:** O countdown cria anticipacao durante DIAS. O jogador volta ao cassino para conferir progresso, comprar mais tickets, verificar se ganhou. Isso aumenta retencao mais que qualquer jogo instantaneo.

Diferente de um jogo onde o house edge eh fixo, aqui o admin controla diretamente a margem: se o total de tickets vendidos > valor do premio, o cassino lucra. Se quer ser generoso, cria sorteios gratuitos.

3. **Drum reveal eh espetaculo visual:** A animacao de scroll vertical desacelerando ate parar no vencedor eh uma das experiencias mais tensas em jogos de azar. Funciona como o drum do Stake, a roda do Fortune, ou o reveal do Case Opening. Cria pico de adrenalina que jogadores compartilham e falam sobre.

4. **Adequacao a aba EVENTOS:** O Giveaway (#20) fica na aba EVENTOS junto com Loteria (#18), Daily-Free (#19). Sorteios se encaixam perfeitamente nessa aba — sao eventos com começo, meio e fim, nao jogos instantaneos.

5. **Reaproveitamento de mecanicas:** O countdown pode reutilizar logica do Daily-Free (#19). O drum reveal pode reutilizar animacao similar ao Case Opening (#14). O ticket system usa o mesmo modelo de compra do Lottery (#18). Sinergia com 3 jogos existentes.

## 4.1 Opcoes Tecnicas Comparadas

### Opcao A — Framer Motion (Motion v12) + useAnimate

Motion (ex-Framer Motion) eh a lib de animacao JA INSTALADA no projeto (framer-motion 11.18.x). A versao atual (v12, 2026) suporta hardware-accelerated scroll, layout animations, e o novo AnimateNumber para ticker/countdown. Para o drum reveal: usar motion.div com animate={{ y: finalPosition }} + transition={{ type: "tween", ease: [0.2, 0, 0, 1], duration: 4 }} para simular desaceleracao natural. A combinacao useAnimate + variants permite sequenciar: scroll rapido (0-2s) → desaceleracao (2-4s) → bounce final (4-4.5s) → highlight winner (4.5-5s).

Vantagens: ja instalado, sem dependencia nova, API de animacao madura, suporte a layout animations para transicoes suaves.
Desvantagens: nao tem physics engine nativa, desaceleracao precisa ser simulada com easing curves.

### Opcao B — GSAP ScrollTrigger + Timeline

GSAP (GreenSock) oferece timelines com controle preciso de sequenciamento. ScrollTrigger poderia vincular a posicao do scroll ao progresso da animacao. Timeline permite: label("drumStart") → to(drumContainer, { y: totalScroll, duration: 2, ease: "power1.in" }) → to(drumContainer, { y: winnerPosition, duration: 3, ease: "power4.out" }).

Vantagens: controle granular de timeline, easing curves profissionais, ScrollTrigger para scroll-linked.
Desvantagens: dependencia NOVA (nao instalada), weight adicional (~50kb minified), API diferente do resto do projeto que usa Motion.

### Opcao C — CSS @keyframes + requestAnimationFrame

Abordagem pura sem libs: CSS animation para o countdown (flip cards), requestAnimationFrame para o drum scroll com calculo de desaceleracao manual (velocity *= 0.98 por frame).

Vantagens: zero dependencias, performance maxima, controle total.
Desvantagens: muito codigo manual, dificil de manter, sem variants/stagger do Motion.

## 4.2 Decisao Tecnica

**Escolha: Opcao A — Framer Motion (Motion v12) + useAnimate.**

Justificativa: ja instalado no projeto (framer-motion 11.18.x), toda a base de codigo usa Motion para animacoes, manter consistencia eh mais importante que features extras do GSAP. O AnimateNumber nativo resolve o countdown, e useAnimate resolve o sequenciamento do drum reveal.

```typescript
// Conceito: DrumRevealOverlay.tsx
// Scroll vertical de tickets desacelerando ate o vencedor

const DrumRevealOverlay = ({ tickets, winnerIndex, onComplete }) => {
  const [scope, animate] = useAnimate();
  const ITEM_HEIGHT = 80; // px por ticket
  const TOTAL_SCROLL = tickets.length * ITEM_HEIGHT * 3; // 3 voltas completas
  const WINNER_POS = -(winnerIndex * ITEM_HEIGHT + TOTAL_SCROLL);

  useEffect(() => {
    const sequence = async () => {
      // Fase 1: scroll rapido (2s) — blur de motion
      await animate(scope.current, { y: -TOTAL_SCROLL * 0.6 }, {
        duration: 2, ease: "linear"
      });
      // Fase 2: desaceleracao (3s) — tension building
      await animate(scope.current, { y: WINNER_POS }, {
        duration: 3, ease: [0.2, 0, 0, 1] // ease-out dramatico
      });
      // Fase 3: bounce final (0.3s) — emphasis
      await animate(scope.current, { y: WINNER_POS + 10 }, {
        duration: 0.15, ease: "easeIn"
      });
      await animate(scope.current, { y: WINNER_POS }, {
        duration: 0.15, ease: "easeOut"
      });
      onComplete();
    };
    sequence();
  }, []);

  return (
    <div className="drum-viewport" style={{ height: ITEM_HEIGHT * 3, overflow: "hidden" }}>
      <motion.div ref={scope} className="drum-strip">
        {tickets.map((ticket, i) => (
          <div key={i} style={{ height: ITEM_HEIGHT }}>
            {ticket.playerName} — #{ticket.number}
          </div>
        ))}
      </motion.div>
    </div>
  );
};
```
