## RELATORIO DA PESQUISA (REGRA X0)

**Rodada 1 — MySQL raffle schema + oxmysql FiveM:**
- Query: "MySQL raffle giveaway system schema tickets winner random draw HMAC SHA256 provably fair 2025"
- Fontes: github.com/ftc2/fair-raffle, stake.com/provably-fair/implementation, rafflebase.com, sourcecodester.com/raffle-draw, github.com/nucleare/DeepSearch_HashGames
- Descobertas: (1) Raffle PF padrao: HMAC_SHA256(serverSeed, clientSeed:nonce) → primeiros 8 chars hex → parseInt → % totalTickets = winnerIndex. (2) fair-raffle usa chain de tickets dependentes (cada ticket depende do anterior) para impedir retroactive injection — APLICAVEL ao Giveaways. (3) Stake.com: cursor so itera se >8 outcomes necessarios — para raffle 1 winner, cursor=0 fixo. (4) Rafflebase usa "bank-grade randomness" sem expor seeds — nosso modelo e superior (provably fair verificavel pelo jogador).
- Impacto: Confirma modelo PF do Mega Estudo. winnerIndex = parseInt(hmac.slice(0,8), 16) % totalTickets. Sem cursor, sem complexidade extra.

**Rodada 2 — oxmysql prepared statements + transactions:**
- Query: "oxmysql FiveM transaction MySQL prepared statement INSERT SELECT UPDATE atomic 2025 2026"
- Fontes: oxmysql.com, coxdocs.dev/oxmysql, github.com/overextended/oxmysql, vertexmods.com/fivem-database-guide, fivemx.com/mysql-async-to-oxmysql
- Descobertas: (1) OxMySQL usa MySQL.prepare com ? placeholders (NAO named). (2) MySQL.transaction() para operacoes atomicas multi-query (debitar saldo + inserir tickets). (3) MySQL.update retorna affectedRows (util para verificar se UPDATE condicional funcionou). (4) OxMySQL arquivado Abr/2025 mas continua funcional e padrao FiveM. (5) MariaDB recomendado sobre MySQL 8 (reserved keywords issue). (6) execute para INSERT/UPDATE/DELETE, query para SELECT.
- Impacto: Usar transaction para buyTickets (debitar saldo + inserir N tickets atomicamente). UPDATE condicional para free ticket cooldown.

**Rodada 3 — Node.js crypto HMAC-SHA256 para raffle winner:**
- Query: "Node.js crypto createHmac sha256 winner selection raffle modulo total tickets provably fair 2025"
- Fontes: nodejs.org/api/crypto (createHmac docs), stake.com/provably-fair/implementation, medium.com/nsoft/provably-fair, geeksforgeeks.org/node-js-crypto-createhmac, mailtoui.com/building-provably-fair
- Descobertas: (1) createHmac NAO reutilizavel apos .digest() — criar novo objeto por operacao. (2) Padrao: crypto.randomBytes(32).toString('hex') para server seed. (3) SHA256 do server seed como hash publico pre-draw. (4) Para raffle: parseInt(hmacHex.slice(0,8), 16) % totalTickets = winnerIndex. (5) Nonce = giveaway.id (auto-increment, simplicidade maxima).
- Impacto: Codigo real no engine: generateServerSeed(), calculateWinner(), verifyResult(). Zero dependencias externas (usa crypto nativo Node.js).

**O que teriamos feito sem a pesquisa:**
- Math.random() em vez de HMAC (nao verificavel, nao provably fair)
- Named placeholders em vez de ? (erro oxmysql)
- createHmac reutilizado apos digest (crash silencioso)
- Transaction desnecessaria para INSERT simples (overhead sem beneficio)

## 1.1 Criar arquivo `sql/casino_giveaways_schema.sql`

**Arquivo NOVO — codigo completo executavel:**

```sql
-- ====================================================
-- BLACKOUT CASINO — GIVEAWAYS (#20) — SCHEMA MySQL
-- Versao 1.0 - 26/03/2026
-- Executar: copiar-colar no MySQL/MariaDB
-- ====================================================

-- Tabela principal de sorteios
CREATE TABLE IF NOT EXISTS casino_giveaways (
  id                  INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  title               VARCHAR(128)  NOT NULL                      COMMENT 'Titulo do sorteio (ex: MEGA GIVEAWAY DE NATAL)',
  description         TEXT          DEFAULT NULL                  COMMENT 'Descricao curta PT-BR',
  description_en      TEXT          DEFAULT NULL                  COMMENT 'Descricao curta EN',
  prize_value         INT UNSIGNED  NOT NULL                      COMMENT 'Valor do premio em GCoins',
  prize_description   VARCHAR(256)  DEFAULT NULL                  COMMENT 'Descricao do premio (ex: G$ 50.000 + VIP Pass)',
  ticket_price        INT UNSIGNED  NOT NULL DEFAULT 100          COMMENT 'Preco de cada ticket em GCoins',
  max_tickets_total   INT UNSIGNED  NOT NULL DEFAULT 500          COMMENT 'Maximo de tickets vendaveis neste sorteio',
  max_tickets_per_player INT UNSIGNED NOT NULL DEFAULT 10         COMMENT 'Maximo de tickets por jogador',
  free_ticket_enabled TINYINT(1)    NOT NULL DEFAULT 1            COMMENT '1=permite ticket gratuito diario, 0=nao',
  tickets_sold        INT UNSIGNED  NOT NULL DEFAULT 0            COMMENT 'Contador de tickets vendidos (cache)',
  min_tickets_to_draw INT UNSIGNED  NOT NULL DEFAULT 10           COMMENT 'Minimo de tickets vendidos para draw acontecer',
  status              ENUM('upcoming','active','drawing','completed','cancelled')
                      NOT NULL DEFAULT 'upcoming'                 COMMENT 'Estado do sorteio',
  draw_at             DATETIME      NOT NULL                      COMMENT 'Data/hora programada do draw (UTC)',
  drawn_at            DATETIME      DEFAULT NULL                  COMMENT 'Data/hora real do draw (UTC)',
  server_seed_hash    VARCHAR(128)  DEFAULT NULL                  COMMENT 'SHA-256 do server seed (publicado ao criar sorteio)',
  server_seed         VARCHAR(128)  DEFAULT NULL                  COMMENT 'Server seed revelado APOS o draw',
  client_seed         VARCHAR(128)  DEFAULT NULL                  COMMENT 'Client seed (combinacao dos primeiros 3 compradores)',
  nonce               INT UNSIGNED  DEFAULT NULL                  COMMENT 'Nonce = id do sorteio',
  hmac_result         VARCHAR(128)  DEFAULT NULL                  COMMENT 'HMAC-SHA256 resultado completo',
  winner_index        INT UNSIGNED  DEFAULT NULL                  COMMENT 'Indice do ticket vencedor (0-based)',
  winner_ticket_id    INT UNSIGNED  DEFAULT NULL                  COMMENT 'FK para casino_giveaway_tickets.id',
  winner_player_id    VARCHAR(64)   DEFAULT NULL                  COMMENT 'Steam ID do vencedor',
  created_by          VARCHAR(64)   DEFAULT NULL                  COMMENT 'Steam ID do admin que criou',
  created_at          DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at          DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at          DATETIME      DEFAULT NULL                  COMMENT 'Soft delete (REGRA X13)',

  INDEX idx_status (status),
  INDEX idx_draw_at (draw_at),
  INDEX idx_winner (winner_player_id),
  INDEX idx_deleted (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- Tabela de tickets comprados
CREATE TABLE IF NOT EXISTS casino_giveaway_tickets (
  id                  INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  giveaway_id         INT UNSIGNED  NOT NULL                      COMMENT 'FK para casino_giveaways.id',
  player_id           VARCHAR(64)   NOT NULL                      COMMENT 'Steam ID do comprador',
  ticket_number       INT UNSIGNED  NOT NULL                      COMMENT 'Numero sequencial do ticket (0-based)',
  is_free             TINYINT(1)    NOT NULL DEFAULT 0            COMMENT '1=ticket gratuito, 0=comprado',
  price_paid          INT UNSIGNED  NOT NULL DEFAULT 0            COMMENT 'Preco pago (0 se gratuito)',
  purchased_at        DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP,

  INDEX idx_giveaway (giveaway_id),
  INDEX idx_player (player_id),
  INDEX idx_giveaway_player (giveaway_id, player_id),
  UNIQUE idx_giveaway_number (giveaway_id, ticket_number),
  CONSTRAINT fk_ticket_giveaway FOREIGN KEY (giveaway_id)
    REFERENCES casino_giveaways(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- Tabela de configuracao admin
CREATE TABLE IF NOT EXISTS casino_giveaway_config (
  id                  INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  config_key          VARCHAR(64)   NOT NULL UNIQUE               COMMENT 'Chave da configuracao',
  config_value        VARCHAR(256)  NOT NULL                      COMMENT 'Valor da configuracao',
  description         VARCHAR(256)  DEFAULT NULL                  COMMENT 'Descricao para admin',
  updated_at          DATETIME      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
