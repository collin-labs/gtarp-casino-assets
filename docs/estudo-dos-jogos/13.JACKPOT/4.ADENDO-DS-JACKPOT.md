## 3.1 Construção Dinâmica do Donut (conic-gradient)

```javascript
// Gera o CSS conic-gradient a partir do array de jogadores
function buildConicGradient(players) {
  const stops = [];
  let currentDeg = 0;
  
  players.forEach(p => {
    const degrees = (p.percent / 100) * 360;
    const endDeg = currentDeg + degrees;
    // Hard color stop (sem transição suave entre setores)
    stops.push(`${p.color} ${currentDeg.toFixed(1)}deg ${endDeg.toFixed(1)}deg`);
    currentDeg = endDeg;
  });
  
  return `conic-gradient(${stops.join(', ')})`;
}

// Uso no React:
// style={{ backgroundImage: buildConicGradient(players) }}
```

**Transição ao receber novo depósito:** O donut atualiza com CSS `transition: background-image` (não suportado nativamente). Solução: animar via opacity fade (donut antigo fade out 200ms, novo fade in 200ms) ou simplesmente atualizar instantaneamente (aceitável pois depósitos são discretos).

// Server envia: winningTicket, totalTickets
// Client calcula o ângulo final
function calculateWinnerAngle(winningTicket, totalTickets) {
  // Posição proporcional no donut (0-1)
  const position = winningTicket / totalTickets;
  // Converter para graus (0-360)
  const angle = position * 360;
  // O pointer está no TOPO (0deg = 12h). 
  // Ajustar para que o ângulo alvo pare alinhado ao pointer:
  // Rotação final = 360 - angle (porque a roda gira no sentido horário)
  return 360 - angle;
