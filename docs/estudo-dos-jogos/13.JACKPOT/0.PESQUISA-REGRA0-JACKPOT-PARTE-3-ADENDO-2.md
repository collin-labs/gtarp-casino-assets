## 5.2 Técnica recomendada: CSS conic-gradient + rotate

A pesquisa revelou que `conic-gradient()` é a técnica ideal para o spinner do Jackpot:

**Por quê conic-gradient:**
- Suportado no Chromium 93+ (FiveM CEF) — sem polyfill
- Cria setores de "pizza" com ângulos exatos proporcionais às porcentagens dos jogadores
- Setores com hard color stops (sem transição suave) — visual limpo
- Border-radius 50% transforma o gradiente em roda circular
- Pode ser atualizado dinamicamente via CSS custom properties ou style inline
- Performance: é uma imagem de fundo CSS, não requer Canvas ou SVG

**Construção dinâmica dos setores em React:**
```javascript
function buildConicGradient(players) {
  // players: [{ id, name, percentage, color }]
  let stops = [];
  let currentAngle = 0;
  
  players.forEach(p => {
    const endAngle = currentAngle + (p.percentage * 3.6); // 360 * percentage/100
    stops.push(`${p.color} ${currentAngle}deg ${endAngle}deg`);
    currentAngle = endAngle;
  });
  
  return `conic-gradient(${stops.join(', ')})`;
}

// Uso:
// style={{ backgroundImage: buildConicGradient(players), borderRadius: '50%' }}
```

**Animação de spin:**
```css
/* A roda inteira gira via CSS rotate */
.jackpot-wheel {
  background-image: conic-gradient(...);
  border-radius: 50%;
  transition: transform 8s cubic-bezier(0.17, 0.67, 0.12, 0.99);
  /* cubic-bezier: rápido no início, desacelera suavemente */
}

/* Quando resultado chega, aplicar rotação final */
.jackpot-wheel.spinning {
  transform: rotate(calc(1800deg + var(--final-angle)));
  /* 1800deg = 5 voltas completas + ângulo final do vencedor */
}
```

**Pointer/seta:** Elemento fixo (triângulo CSS ou imagem PNG) posicionado no topo da roda. A roda gira sob o pointer.

## 5.3 Alternativa: react-custom-roulette (npm)

Lib npm pronta com componente `<Wheel>` que aceita data array e prizeNumber:
```jsx
import { Wheel } from 'react-custom-roulette';
<Wheel mustStartSpinning={spin} prizeNumber={winnnerIndex} data={wheelData} />
```

**Prós:** Rápido de implementar, animação pronta.
**Contras:** Menos customizável, dependência externa, visual pode não combinar com DS do Blackout Casino.

**Decisão:** NÃO usar lib externa. Implementar com conic-gradient + CSS rotate. Mais controle sobre visual, sem dependência, combina perfeitamente com o DS.

# 7. SÍNTESE — O QUE TROUXE DE NOVO VS. SEM PESQUISA
# ═══════════════════════════════════════════════════

## 7.1 O que teríamos criado SEM a pesquisa

- Jackpot como "sorteio simples" — sem spinner visual, apenas resultado texto
- Roda usando Canvas 2D (complexo, pesado) ou lib externa (react-custom-roulette)
- Sem sistema de tickets — Math.random() ponderado sem verificação
- Sem timer de rodada — jogadores não saberiam quando a rodada fecha
- Sem tiers Low/Mid/High — high-rollers dominariam contra small bets
- Sem bots — rodadas morreriam com <2 jogadores
- Visual genérico sem combinar com DS dourado do cassino
- Sem entender que Jackpot é MULTIPLAYER (2-10), não 1v1 como Coinflip

## 7.2 O que a pesquisa trouxe de concreto

1. **Mecânica precisa** com timer de 60s, fases OPEN→CLOSING→CLOSED→SPINNING→RESULT, sistema de tickets proporcional
2. **CSS conic-gradient** como técnica ideal para a roda — nativo no CEF, dinâmico, sem Canvas/lib
3. **Easing de spin** com cubic-bezier(0.17, 0.67, 0.12, 0.99) — aceleração rápida → desaceleração suave e dramática
4. **Ticket system provably fair** com SHA256 modulo totalTickets + mapeamento para ranges de jogadores
5. **10 cores distintas** para setores dos jogadores, seguindo hierarquia de raridade do DS
6. **Bots** como feature crítica (Howl.gg confirma), preenchendo rodada quando poucos jogadores
7. **Jackpot PvP não existe em FiveM** — seria pioneiro no mercado de scripts
8. **Performance <1.5% FPS** com CSS puro (conic-gradient + rotate)
9. **Roda dourada ornamentada** como adaptação visual para o DS do Blackout Casino
10. **Variantes Low/Mid/High** como feature futura de separação por faixa de valor
