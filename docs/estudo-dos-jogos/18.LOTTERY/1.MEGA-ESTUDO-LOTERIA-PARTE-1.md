**Decisão estratégica da pesquisa:** O jogo #18 LOTERIA será um híbrido de Mega-Sena brasileira (6 números de 60, identidade cultural BR forte) com o sistema Mega Ball da Evolution Gaming (multiplicador 2x-50x após sorteio principal). Essa decisão veio da pesquisa — sem ela, teríamos feito um clone genérico de Keno.

**8 descobertas concretas da pesquisa:**
1. Loteria em cassinos online existe em 3 formatos distintos (Keno, Mega Ball, Loteria Nacional) — cada um com mecânica e UX completamente diferentes
2. Matter.js é viável para globo animado — artigo FreeCodeCamp demonstra implementação funcional de bolas quicando dentro de container esférico, com código aberto (MIT)
3. Container circular no Matter.js requer polyline de 24-32 segmentos retangulares posicionados em arco — não existe "circular wall" nativo
4. ZERO dos 8 scripts de loteria FiveM encontrados têm globo animado, multiplicador, ou visual premium
5. 20 bolas no globo (não 60) é o ponto ótimo: 0.3ms/tick = 60fps garantido, contra 1.2ms/tick para 60 bolas
6. Provably Fair para loteria usa HMAC_SHA256 com esquema commit-reveal (hash publicado antes, seed revelado depois)
7. Evolution Mega Ball tem fluxo em 4 fases com timings específicos (compra 15s → sorteio 45s → Mega Ball 15s → resultado 10s)
8. Mega-Sena real usa 2 tambores separados (dezena 0-5, unidade 0-9) — detalhe que enriquece ambientação

**Total handlers: 7 handlers, ~380 linhas estimadas.**

**Fluxo completo de uma aposta (passo a passo no server):**

```
1. Player chama casino:lottery:buy
   → Server valida tudo (6 checks)
   → Server gera serverSeed = crypto.randomBytes(32).hex()
   → Server calcula seedHash = SHA256(serverSeed)
   → Server gera clientSeed (se player não forneceu)
   → Server busca nonce do player (incrementa automaticamente)
   → Server deduz bet do saldo do player
   → Server calcula poolContribution = bet * (config.poolPercent / 100)
   → Server UPDATE casino_lottery_jackpot SET current_pool = current_pool + poolContribution
   → Server INSERT casino_lottery_draws (status = 'pending')
   → Server INSERT casino_lottery_daily (ON DUPLICATE KEY UPDATE tickets_used + 1)
   → Server INSERT casino_lottery_audit (action = 'BUY')
   → Retorna { success: true, seedHash, drawId }

2. Player chama casino:lottery:draw
   → Server busca o draw (status = 'pending')
   → Server calcula resultado:
     hash = HMAC_SHA256(serverSeed, clientSeed + ":" + nonce)
     drawnNumbers = extrair 6 números únicos de 1-60 do hash
     megaBallHash = HMAC_SHA256(serverSeed, clientSeed + ":" + nonce + ":megaball")
     megaBallMultiplier = mapear para tabela de multiplicadores
   → Server compara drawnNumbers vs ticketNumbers → matches
   → Server calcula basePrize = payoutTable[matches] * bet
   → Server calcula finalPrize = basePrize * megaBallMultiplier
   → Se matches === 6 → jackpotHit = true, jackpotAmount = current_pool
   → Server aplica maxPayout cap
   → Server credita finalPrize (+ jackpotAmount se aplicável) no saldo
   → Se jackpotHit → UPDATE casino_lottery_jackpot (reset pool para seed, gravar winner)
   → Server UPDATE casino_lottery_draws (status = 'completed', todos os campos)
   → Server UPDATE casino_lottery_daily (total_won, best_match)
   → Server INSERT casino_lottery_audit (action = 'DRAW' ou 'JACKPOT_HIT')
   → Retorna resultado completo para animação no NUI
```
