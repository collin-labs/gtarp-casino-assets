## 3.1 Animação do Globo de Bolas

A pesquisa revelou **3 abordagens técnicas** para animar bolas dentro de um container (globo):

### Opção A — Matter.js (Física 2D Realista)

**Fonte:** freecodecamp.org/news/how-to-create-a-lotto-balls-blowing-effect/
**Como funciona:**
- Matter.js cria "bodies" (círculos = bolas) dentro de "walls" (container = globo)
- Gravidade, colisão ball-ball e ball-wall são automáticas
- Bolas quicam realisticamente dentro do container
- "Air blow" effect: aplica força ascendente periódica para simular sopro de ar
- Render via Canvas 2D (Matter.js tem renderer próprio ou customizado)

**Prós:**
- Física realista sem código manual de colisão
- Bolas se comportam naturalmente (quicam, empurram, giram)
- Performance boa para 20-60 bolas (~60fps)
- Biblioteca madura (v0.20+, 10k+ stars GitHub, atualizada 2024/2025)

**Contras:**
- Bundle size: ~87KB minified (aceitável)
- Container circular (globo esférico) requer custom constraint ou path walls
- Overhead de physics engine para "apenas" animação decorativa

**Veredicto:** ✅ **Melhor opção para animação realista do globo.** Já existe exemplo funcional de lotto balls (FreeCodeCamp). Container circular possível com custom walls em arco.

### Opção B — Canvas 2D Puro (Física Manual)

**Fontes:** MDN Advanced Animations, GeeksForGeeks, múltiplos CodePen
**Como funciona:**
- Cada bola: {x, y, vx, vy, radius, color, number}
- Loop: requestAnimationFrame → update position → check collision → draw
- Gravidade: vy += g (constante)
- Bounce: inverter velocidade ao tocar parede
- Colisão ball-ball: distância entre centros < soma dos raios → inverter componentes

**Prós:**
- Zero dependências (Canvas 2D nativo)
- Performance máxima
- Controle total sobre cada frame
- Código mais leve

**Contras:**
- Colisão ball-ball manual é trabalhosa para 60 bolas
- Container circular requer math de colisão com arco (mais complexo)
- Física menos realista (sem rotação, sem friction natural)

**Veredicto:** ⚠️ **Viável mas mais trabalhoso.** Bom para fallback se Matter.js causar problemas de performance no NUI.

### Opção C — CSS + Framer Motion (Sem Física)

**Como funciona:**
- Bolas como elementos DOM com animação CSS/Framer Motion
- Posições pré-calculadas ou random
- Sem colisão real — apenas animação visual

**Prós:**
- Mais simples de implementar
- Integra com stack React/Framer Motion existente
- Sem dependência extra

**Contras:**
- NÃO parece realista (bolas não colidem, não quicam)
- Limitado a ~20 elementos DOM animados simultaneamente
- Não escala para globo com 60 bolas

**Veredicto:** ❌ **Não recomendado para o globo.** Pode ser usado para elementos UI (bilhete, números reveal) mas não para a animação principal.
