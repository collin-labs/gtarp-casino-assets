## 2.1 Tipos de Slot Machine (Classificacao por Mecanica)

### TIPO A - Classic Slots (3 Reels)
**Origem:** Liberty Bell, Charles Fey, 1895. Mecanica mais antiga e simples.
**Grid:** 3 reels, 1-5 paylines fixas.
**Simbolos:** Cherry, Bar, Bell, Lucky 7. Poucos (5-8 tipos).
**Mecanica:** Match simbolos em payline horizontal. Sem bonus, sem wilds (versao pura). Spin e resultado imediato.
**Volatilidade:** Baixa-media. Wins frequentes, payouts pequenos.
**RTP tipico:** 90-95%.
**Exemplos:** Double Diamond (IGT), Mega Joker (NetEnt).
**Para o Blackout Casino:** Modo nostalgico/rapido. 3 reels, visual classico com estetica dourada. Bom para "Classic Mode" com partidas rapidas.

### TIPO B - Video Slots (5 Reels, Paylines)
**Origem:** Evolucao digital dos anos 2000.
**Grid:** 5 reels, 3-5 rows, 10-50 paylines configuráveis.
**Simbolos:** Tematicos (10+ tipos), incluindo Wilds e Scatters.
**Mecanica:** Match simbolos em paylines (esquerda→direita). Wilds substituem qualquer simbolo. Scatters trigam bonus/free spins independente de posicao. Multiplicadores aleatorios.
**Volatilidade:** Media-alta. Bonus rounds concentram wins grandes.
**RTP tipico:** 94-97%.
**Exemplos:** Starburst (NetEnt), Book of Dead (Play'n GO), The Dog House (Pragmatic).
**Para o Blackout Casino:** Formato tradicional. 5 reels, paylines visiveis, wilds e scatters. Bom para jogadores que conhecem slots "normais".

### TIPO C - Scatter Pays + Tumble/Cascade (Sweet Bonanza / Gates of Olympus)
**Origem:** Pragmatic Play, ~2019. Revolucionou o formato.
**Grid:** 6x5 (30 posicoes). Sem paylines tradicionais.
**Simbolos:** Tematicos. 8+ simbolos iguais ANYWHERE no grid = win.
**Mecanica core - Scatter Pays:** Nao precisa de paylines. Qualquer 8+ simbolos iguais em qualquer posicao pagam. Quanto mais simbolos, maior o payout (8/9/10/11/12+ tiers).
**Mecanica core - Tumble:** Apos win, simbolos vencedores SOMEM do grid. Simbolos acima CAEM para preencher. Novos simbolos CAEM do topo. Se formam novo win, tumble repete. Continua ate nao haver mais wins.
**Mecanica core - Multiplicadores:** Durante tumbles (especialmente em free spins), simbolos de multiplicador (2x-100x no Sweet Bonanza, 2x-500x no Gates of Olympus) podem aparecer. No fim da sequencia de tumbles, TODOS os multiplicadores somam e aplicam ao win total.
**Bonus - Free Spins:** 4+ scatters = 10-15 free spins. Multiplicadores acumulam ao longo das free spins (Gates: running total multiplier). Retrigger possivel.
**Bonus - Buy Feature:** Pagar 100x stake para trigger free spins imediatamente. Ante Bet: +25% stake para dobrar chances de trigger.
**Volatilidade:** Alta. Muitos spins sem win, seguidos de explosoes massivas nas free spins.
**RTP tipico:** 96-96.5%.
**Max win:** Sweet Bonanza 21.175x, Gates of Olympus 5.000x, Gates 1000 15.000x, Sweet Bonanza 1000 25.000x.
**Exemplos:** Sweet Bonanza, Gates of Olympus, Sugar Rush, Fruit Party.
**Para o Blackout Casino:** FORMATO PRINCIPAL RECOMENDADO. Grid 6x5, scatter pays, tumble, multiplicadores. E a mecanica mais popular de 2024-2026, comprovada pela dominancia do Sweet Bonanza e Gates of Olympus nos rankings globais.

### TIPO D - Megaways (Reels Variaveis)
**Origem:** Big Time Gaming (BTG), 2015. Dragon Born.
**Grid:** 6 reels, 2-7 simbolos por reel (varia CADA spin). Ate 117.649 ways to win.
**Mecanica:** Cada reel mostra numero aleatorio de simbolos (2 a 7). Resultado = produto das quantidades (7x7x7x7x7x7 = 117.649 max). Win quando simbolos identicos em reels adjacentes esquerda→direita. Sem paylines fixas.
**Mecanica Tumble:** Igual ao Tipo C — wins somem, novos caem.
**Volatilidade:** Muito alta. Grid pode estar "pequeno" (2x2x2x2x2x2 = 64 ways) ou "enorme" (117.649 ways).
**Exemplos:** Extra Chilli Megaways (20.000x jackpot), Bigger Bass Bonanza Megaways, Dog House Megaways.
**Para o Blackout Casino:** Complexidade alta de implementacao (grid dinamico). Possivel como feature futura, mas NAO recomendado para v1. O Tipo C (scatter pays + tumble) oferece experiencia comparavel com implementacao mais simples.

### TIPO E - Cluster Pays (Grid Grande)
**Origem:** NetEnt, Aloha Cluster Pays.
**Grid:** 7x7 ou maior. Sem paylines nem reels tradicionais.
**Mecanica:** 5+ simbolos adjacentes (horizontal/vertical) formam cluster = win. Clusters maiores pagam mais. Apos win, simbolos do cluster somem e novos caem (tumble). Pode chain-react.
**Exemplos:** Sugar Rush (7x7), Reactoonz (Play'n GO), Jammin Jars.
**Para o Blackout Casino:** Visual interessante mas muito diferente de slots tradicionais. Pode confundir jogadores. NAO recomendado como formato principal. Possivel como "modo especial" futuro.

## 3.1 Componente A - Animacao de Reels (Spinning)

### Opcao A1 - CSS Transform + Transition (DOM)

**Como funciona:** Cada reel e um container com simbolos empilhados verticalmente. Spin = translateY negativo rapido (scroll para cima). Parada = transition ease-out para posicao final pre-calculada.

**Implementacao encontrada (CodePen antibland):**
- Sprite sheet de simbolos como background-image
- `background-position` transition: 0.3s ease-in-out
- `will-change: backgroundPosition` + `transform: translateZ(0)` para GPU acceleration
- 3 reels com timers escalonados (1000ms, 1400ms, 2200ms) para efeito cascata
- forwardRef + useImperativeHandle para controlar spin/reset do parent

**Pros:** Zero dependencias, performatico, funciona em NUI/CEF.
**Contras:** Limitado a scroll vertical. Nao suporta tumble/cascade nativamente.

### Opcao A2 - slot-machine-gen (npm)

**Como funciona:** Lib JS sem dependencias que cria reels com efeito de cilindro 3D (faux-panoramic) usando apenas CSS transforms. Configuravel: simbolos com peso (weight), sound clips, callbacks.

**Features relevantes:**
- Reel rotation via CSS (sem Canvas)
- Cada simbolo tem `title`, `position`, e `weight` (peso para RNG)
- Callback no resultado retorna array de simbolos selecionados
- Payline evaluation no callback
- Import como modulo ES (compativel React)

**Pros:** Leve, sem Canvas, selecao por peso (RTP facil de calibrar), som integrado.
**Contras:** Ultima versao npm e de 2019. Pode precisar fork/adaptacao para React 19.

### Opcao A3 - Framer Motion + CSS Grid (Custom)

**Como funciona:** Cada celula do grid e um motion.div. Spin = animate Y com ciclos. Parada = spring para posicao final. Tumble = animate Y para baixo + opacity para fade. Novos simbolos = AnimatePresence com entrada de cima.

**Implementacao proposta:**
- Grid 6x5 de motion.div (30 celulas)
- Estado: `grid[6][5]` array de symbol IDs
- Spin: animate cada coluna com stagger + translateY rapido
- Parada: coluna por coluna (stagger 200ms), spring easing
- Tumble: simbolos win fadeOut (opacity 0, scale 0.5), simbolos acima slideDown, novos slideIn do topo
- Multiplicadores: motion.div com scale pulse + glow

**Pros:** Ja instalado no projeto (Framer Motion 11), efeitos premium nativos, tumble nativo, compativel com DS.
**Contras:** Mais codigo para gerenciar estado do grid manualmente.

### Decisao Tecnica: Opcao A3 (Framer Motion Custom)

**Motivo:** O Blackout Casino ja usa Framer Motion em TODOS os componentes. Criar reels customizados com motion.div garante consistencia visual (glow, spring, stagger) e torna o tumble nativo. As opcoes A1/A2 precisariam de "wrapper" para integrar com o DS, enquanto A3 ja nasce integrada.

Para o Modo Classic (3 reels): simplificar para 3 colunas com translateY scroll.
Para o Modo Video (6x5): grid completo com tumble/cascade.

## 3.2 Componente B - Tumble/Cascade Animation

**Mecanica:** Apos avaliar wins, simbolos vencedores somem. Simbolos acima caem. Novos simbolos entram pelo topo. Ciclo repete.

**Implementacao tecnica:**
1. Avaliar wins no grid atual (scatter pays: contar simbolos iguais)
2. Marcar posicoes vencedoras
3. AnimatePresence: fade out + scale down nos marcados (200ms)
4. Gravity drop: simbolos acima movem para baixo (animate Y, spring)
5. New symbols: gerar novos, AnimatePresence entry de cima (slide + fade)
6. Re-avaliar wins no novo grid
7. Repetir ate sem wins

**Performance:** Cada ciclo tumble tem ~400-600ms de animacao. Em media 2-5 tumbles por sequencia. Total: 1-3 segundos — aceitavel para NUI.

## 3.3 Componente C - Sistema Provably Fair para Slots

**Adaptacao para slots (baseado na pesquisa):**

```
ANTES DO SPIN:
1. Server gera server_seed (random 32 bytes)
2. Server calcula hash = SHA-256(server_seed)
3. Hash enviado ao client (NUI) ANTES do spin
4. Client gera client_seed (automatico ou manual)
5. Nonce incrementa a cada spin

NO SPIN:
6. Result = HMAC-SHA256(server_seed, client_seed + ":" + nonce)
7. Hex result convertido para numeros (posicoes de reel OU simbolos do grid)
8. Para grid 6x5: 30 posicoes, cada uma mapeada para index de simbolo
   usando modulo do hash por chunk

APOS O SPIN:
9. Server revela server_seed ao fim da sessao (ou apos N spins)
10. Client pode verificar: SHA-256(revealed_server_seed) == hash_recebido
11. Se match: resultado foi justo. Se nao: manipulacao detectada
```

**Para FiveM:** O server handler JS gera seeds e calcula resultados. O NUI (React) recebe o hash antes e o resultado depois. Verificacao pode ser feita no client ou via modal "Provably Fair" com campos de seed visíveis.
