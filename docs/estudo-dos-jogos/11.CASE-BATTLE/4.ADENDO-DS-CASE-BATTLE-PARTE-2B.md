## 4.1 Timeline: Lobby → Criar → Espera → Inicio Batalha

```
t=0ms       Jogador abre aba PVP no painel do cassino.
            CaseBattleLobby monta. fetchNui('casino:cases:battle_list').
            Mostra skeletons (.battle-skeleton-card × 3).

t=50-150ms  Resposta do servidor. Skeletons substituidos por BattleCards.
            Cards entram com stagger 40ms cada.
            FM: initial={{ opacity: 0, x: -20 }} animate={{ opacity: 1, x: 0 }}

t=any       Jogador clica CRIAR BATALHA.
            play('modal_open'). Modal CaseBattleCreate abre.
            FM: scale 0.92→1.0, opacity 0→1, duration 250ms.

t=any       Jogador seleciona cases clicando nos CaseCards.
            Cada selecao: play('button_click'). Badge R1/R2/R3 aparece.
            FM badge: scale 0→1 spring stiffness 500.
            Entry fee atualiza com countup 200ms.

t=any       Jogador seleciona modo (toggle Regular/Crazy/Terminal/Private).
            Toggle ativo transiciona: background preenche 200ms ease.
            Descricao muda: opacity 0→1, 150ms.

t=any       Jogador seleciona formato (1v1/4-way/2v2).
            Botao ativo: border-color #D4A843, 200ms.

t=any       Jogador clica CRIAR BATALHA (botao vermelho).
            play('button_click').
            Botao: disabled true, spinner interno.
            fetchNui('casino:cases:battle_create', { maxPlayers, battleMode, caseSequence }).

t=+50-100ms Servidor responde com battleId + serverSeedHash.
            Modal fecha: FM exit scale 0.92, opacity 0, 200ms.
            Transicao para CaseBattleWaiting.

t=+0ms      Sala de espera monta.
            Player circle do criador: FM scale 0→1 spring.
            play('lobby_join') para si mesmo.
            Circulos vazios: border dashed, animation pulseWaiting 2s infinite.

t=+5-15s    Se nenhum jogador real entra: bot entra automaticamente.
            play('lobby_join'). Circulo preenche: animation circleEnter 500ms.
            Nome do bot (NPC GTA): slide-in 300ms.

t=+0ms      Todos os slots preenchidos (jogadores + bots).
            COUNTDOWN INICIA.
            Overlay escuro: opacity 0→0.6, 300ms.

t=+0ms      "3" aparece. play('tick_countdown').
            FM: scale 2→1, opacity 0→1→0, 800ms.
            .countdown-number--3: color #D4A843.

t=+800ms    "2" aparece. play('tick_countdown').
            .countdown-number--2: color #FF8C00.

t=+1600ms   "1" aparece. play('tick_countdown').
            .countdown-number--1: color #EB4B4B.

t=+2400ms   "FIGHT!" aparece. play('go_fight').
            .countdown-fight: color #FFD700, scale 1.5→1.0.
            Overlay faz fadeOut 500ms.

t=+3000ms   Transicao para CaseBattleGame.
            Layout split-screen monta.
            Strips aparecem no estado --waiting (opacity 0.6, dashed).

TOTAL lobby→inicio: ~3-20s (depende de espera por jogadores)
```

## 4.2 Timeline: Strip Reveal (1 Round Completo)

```
t=0ms       Servidor envia evento 'casino:cases:battle_round_start'
            com { roundNumber, results: [{ playerId, itemId, targetIndex }] }.

t=0ms       Round bar: segmento atual muda para --current (pulsando).
            Mini-countdown "1" aparece brevemente (500ms).

t=500ms     Strips entram no estado --animating.
            will-change: transform ativado em todos os strips.
            Marcadores centrais ganham glow var(--mode-glow).

t=500ms     Strip jogador 1 inicia animacao.
            play('strip_scroll') em loop.
            FM: animate={{ x: targetX }}
            transition={{ duration: stripDuration, ease: [0.15, 0.85, 0.35, 1.0] }}
            stripDuration = performanceOk ? 4000 : 2500 (ms)

t=550ms     Strip jogador 2 inicia (+50ms stagger).
t=600ms     Strip jogador 3 inicia (+100ms stagger, se 4-way).
t=650ms     Strip jogador 4 inicia (+150ms stagger, se 4-way).

t=500-2000ms  Fase RAPIDA: itens passam pelo marcador em blur.
              ~80ms entre cada item no pico de velocidade.
              play('strip_tick') a cada item cruzando o centro (volume baixo).

t=2000-3500ms Fase DESACELERACAO: itens passam mais devagar.
              ~150-300ms entre cada item.
              play('strip_tick') com pitch crescente (mais agudo conforme desacelera).
              Easing curve [0.15, 0.85, 0.35, 1.0] produz desaceleracao suave.

t=3500-4500ms Fase FINAL: ultimo item oscila perto do marcador.
              translateX faz micro-ajustes (efeito "quase parou no anterior").
              Tensao maxima visual.

t=4500ms    Strip jogador 1 PARA. play('strip_stop').
            Item sob o marcador recebe estado --winner:
              border-width 2px, box-shadow glow var(--rarity-color),
              animation pulseReveal 2s infinite.
            play('reveal_{rarity}') — som especifico da raridade.
            will-change removido.
            Strip muda para --finished.

t=4550ms    Strip jogador 2 PARA (stagger). Mesmo processo.
t=4600ms    Strip jogador 3 PARA (se 4-way).
t=4650ms    Strip jogador 4 PARA (se 4-way).

t=4650ms    Todos os strips pararam.
            Valor acumulado de cada jogador ATUALIZA:
              countup animation 200ms via useMotionValue.
              play('value_update') — tick discreto.
            Score bar recalcula: width transition 500ms ease.
            Jogador liderando: fill ganha glow (.score-fill--leading).

t=4650ms    Round bar: segmento atual muda para --past.
            SE nao e o ultimo round:
              Pausa de 1500ms antes do proximo round.
              Voltar para t=0ms do proximo round.

t=4650ms    SE e o ultimo round:
              Todos os strips mostram todos os itens revelados.
              Calcular vencedor (ou tiebreaker se diferenca < 1%).
              Transicionar para Resultado (Timeline 4.3).

TOTAL 1 round: ~4.5-5.0s (normal) ou ~2.8-3.0s (fallback performance)
TOTAL 3 rounds: ~16-18s incluindo pausas entre rounds
```

## 4.3 Timeline: Resultado → PF → Replay

```
t=0ms       Ultimo round completado. Vencedor determinado.
            SE empate (diferenca < 1% pot):
              Ir para sub-timeline TIEBREAKER (abaixo).
            SE vencedor claro:
              Continuar.

t=0ms       CaseBattleResult monta.
            Background: fade-in 300ms com radial gradient dourado.

t=300ms     Winner banner aparece.
            FM: scale 0→1, opacity 0→1, spring stiffness 300 damping 15.
            play('victory_fanfare') — brass stinger 3s.
            Shimmer sweep ::after inicia animation infinite.

t=500ms     Nome do vencedor: fadeIn 300ms.
            Nome(s) do(s) perdedor(es): fadeIn 300ms com opacity 0.6.
            Loser label "DERROTA": fadeIn com delay 200ms.
            play('loss_gentle') para o perdedor — volume 50%.

t=800ms     Pot display: countup de 0 ao valor final em 2000ms.
            FM: useMotionValue 0→winnerPayout, ease easeOut.
            Glow verde pulsa durante countup.

t=1000ms    Itens do vencedor aparecem na coluna verde.
            Stagger 80ms cada item. FM: opacity 0→1, x -10→0.

t=1200ms    Itens do perdedor aparecem na coluna cinza.
            Stagger 80ms cada, opacity 0.6.

t=2800ms    Countup finaliza. Valor estabiliza.
            play('value_final') — chime de confirmacao.

t=3000ms    Botoes de acao aparecem.
            FM: stagger 50ms, opacity 0→1, y 10→0.
            "JOGAR NOVAMENTE" + "VERIFICAR" + "VER HISTORICO" + "VOLTAR"

t=any       Jogador clica VERIFICAR.
            play('modal_open'). ProvablyFairModal abre.
            Campos preenchidos: serverSeed (revelado), combinedSeed,
            roundNumber, playerPosition.

t=any       Jogador clica CALCULAR no modal PF.
            play('button_click').
            Web Crypto calcula HMAC-SHA256 localmente.
            Compara hash calculado com serverSeedHash recebido antes.
            SE hash bate: .pf-status--fair (verde, "JUSTO").
            SE hash nao bate: .pf-status--alert (vermelho, "ALERTA").
            play('pf_result') — chime positivo ou buzz negativo.

--- SUB-TIMELINE: TIEBREAKER ---

t=0ms       Empate detectado. play('tiebreaker_buildup') — suspense 1s.
            Colunas empatadas: .result-column--tied (animation tiedPulse).

t=1000ms    Moeda dourada aparece no centro.
            .tiebreaker-coin com FM: scale 0→1, spring.
            play('tiebreaker_coin') — metalico 2s.

t=1200ms    Moeda gira: FM animate={{ rotateY: 1800 }}, duration 2s, easeOut.

t=3200ms    Moeda para. Resultado: cara ou coroa.
            play('tiebreaker_result') — impact + reveal.
            Coluna vencedora: .result-column--tied → .result-column--winner.
            Coluna perdedora: .result-column--tied → .result-column--loser.

t=3500ms    Continuar timeline normal de resultado (t=300ms acima).

TOTAL resultado: ~3-4s (sem tiebreaker) ou ~7s (com tiebreaker)
```

# =============================================
# SECAO 5 — PROIBIDO NO ADENDO DS (15 ITENS)
# =============================================

```
1.  NUNCA usar cores sem valor hex explícito — "cor dourada" NAO EXISTE.
    Sempre #D4A843 ou rgba(212,168,67,X). "Vermelho PvP" = #EB4B4B.

2.  NUNCA copiar CSS do Adendo sem as variaveis de modo.
    TODA cor que depende do modo DEVE usar var(--mode-color) ou var(--mode-glow).
    Hardcodar #EB4B4B onde deveria ser var(--mode-color) quebra Crazy/Terminal.

3.  NUNCA criar componente sem BEM naming.
    .battle-[componente]__[elemento]--[modificador].
    Exemplos: .battle-card__beam--terminal, .btn-join--disabled.

4.  NUNCA entregar componente sem TODOS os estados.
    Minimo: normal + hover + active + disabled (se interativo).
    Componentes de modo: + variante por modo (4).
    Componentes de raridade: + variante por raridade (5).

5.  NUNCA usar z-index sem coordenacao.
    Hierarquia definida: lobby cards (1), strip container (2),
    strip marcador (5), strip fade (3), modais (100-101),
    countdown overlay (70), result (65), history (90).

6.  NUNCA animar propriedades que causam reflow.
    PERMITIDO: transform (translate, scale, rotate), opacity, box-shadow, filter.
    PROIBIDO: width, height, margin, padding, top, left, right, bottom.

7.  NUNCA usar font-family sem fallback stack.
    Cinzel: 'Cinzel', serif. Inter: 'Inter', sans-serif.
    JetBrains Mono: 'JetBrains Mono', monospace.

8.  NUNCA usar transition sem especificar propriedade.
    ERRADO: transition: all 0.2s ease (muito caro).
    CERTO: transition: transform 0.2s ease, opacity 0.2s ease, border-color 0.2s ease.
    Excecao: componentes simples com <= 3 propriedades animaveis.

9.  NUNCA revelar que um jogador e bot visualmente.
    Player circles de bots DEVEM ser identicos aos de jogadores reais.
    A badge "NPC" so aparece para admin em modo debug (display: none por padrao).

10. NUNCA usar px fixo para fontes ou padding sem clamp().
    Excecoes permitidas: border-width (1px, 2px, 3px), border-radius pequeno (3-4px),
    gap minimo (2-3px), box-shadow offsets.

11. NUNCA usar box-shadow com uma unica camada em frames dourados.
    Minimo 2 camadas: glow externo + inset shadow.
    Recomendado 3 camadas para premium: glow + halo + inset.

12. NUNCA usar background: #cor direta em botoes.
    SEMPRE linear-gradient para botoes primarios (profundidade visual).
    Botoes ghost: background transparent com border.

13. NUNCA misturar unidades em clamp().
    CERTO: clamp(8px, 1.2vw, 14px) — px/vw/px.
    ERRADO: clamp(0.5rem, 1.2vw, 14px) — rem/vw/px.

14. NUNCA omitir min-height: 44px em elementos clicaveis (REGRA X4).
    Touch target minimo para mobile. Aplica-se a: botoes, tabs, toggles,
    cards clicaveis, links, areas de selecao.

15. NUNCA usar color: white ou color: black.
    SEMPRE: #FFF (branco), #0A0A0A (preto base), #CCC (texto secundario),
    #888 (texto terciario), #555 (texto discreto), #333 (texto quase invisivel).
```
