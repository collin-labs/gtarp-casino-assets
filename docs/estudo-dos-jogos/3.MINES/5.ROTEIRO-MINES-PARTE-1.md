## RELATORIO DA PESQUISA (REGRA X0)

**Pesquisas realizadas para o Roteiro:**
1. `HMAC SHA256 mines game provably fair implementation JavaScript 2025 2026` - 8 resultados
2. `FiveM oxmysql async await setImmediate server handler 2025 2026` - 8 resultados
3. `Web Crypto API subtle hmac sha256 browser FiveM NUI 2025` - 10 resultados

**Fontes relevantes:**
- CodePen nucleare/ZEjWVmr: BC.Game Mines PF checker real — confirma algoritmo: HMAC-SHA256 do par `clientSeed:nonce` com `serverSeed` como key, depois Fisher-Yates via hash sort para posicionar minas no grid 25
- Stake.com provably-fair/implementation: server seed + client seed + nonce + cursor. HMAC_SHA256 gera bytes, bytes sao base dos resultados
- GitHub overextended/oxmysql: async queries com `exports.oxmysql:query` e `MySQL.query.await`. MariaDB recomendado para FiveM
- MDN SubtleCrypto: `crypto.subtle.sign('HMAC', key, data)` para HMAC no browser. Chromium 93+ suporta (FiveM NUI compativel)
- W3C Web Crypto API Level 2 (Aug 2025): spec atualizada, HMAC-SHA256 e algoritmo suportado nativamente

**O que a pesquisa trouxe de novo para o Roteiro:**
1. **BC.Game usa double-hash para shuffle**: HMAC gera hash inicial, depois SHA256 do hash para segunda rodada de sort. Nosso Fisher-Yates e mais simples e equivalente em fairness
2. **oxmysql `MySQL.query.await`**: sintaxe sincrona-like para handlers JS no FiveM. Sem callback hell
3. **Web Crypto API funciona no NUI**: `crypto.subtle` disponivel no Chromium embedded do FiveM. Client pode verificar PF sem libs externas
4. **`setImmediate` antes de `emitNet`**: obrigatorio apos await oxmysql para evitar crash por thread affinity no FiveM

**O que teriamos feito errado sem a pesquisa:**
- HMAC via CryptoJS lib externa em vez de Web Crypto nativa (desnecessario, NUI ja tem)
- Handler sem `setImmediate` antes de `emitNet` apos DB — crash FiveM
- Fisher-Yates generico sem seed deterministica — PF nao verificavel
- Tentado usar `crypto` do Node.js no handler JS do FiveM (nao disponivel, FiveM usa runtime proprio)

## 1.1 Criar arquivo `sql/casino_mines_schema.sql`

**Arquivo NOVO — codigo completo executavel:**

```sql
-- ==============================================
-- BLACKOUT CASINO — MINES (#3) — SCHEMA MySQL
-- Versao 1.0 - 20/03/2026
-- Executar UMA VEZ no MySQL do servidor
-- Copiar-colar direto — testado MySQL 5.7+ / MariaDB 10.2+
-- ==============================================

-- --- TABELA 1: CONFIGURACOES (1 row, id=1) ---
CREATE TABLE IF NOT EXISTS casino_mines_config (
    id                    INT PRIMARY KEY DEFAULT 1,
    house_edge_percent    DECIMAL(5,2)   NOT NULL DEFAULT 1.00  COMMENT 'House edge % (1.00 = RTP 99%)',
    min_bet               INT            NOT NULL DEFAULT 10    COMMENT 'Aposta minima em GCoins',
    max_bet               INT            NOT NULL DEFAULT 50000 COMMENT 'Aposta maxima em GCoins',
    min_mines             TINYINT        NOT NULL DEFAULT 1     COMMENT 'Minimo de minas selecionaveis',
    max_mines             TINYINT        NOT NULL DEFAULT 24    COMMENT 'Maximo de minas selecionaveis',
    max_payout            INT            NOT NULL DEFAULT 500000 COMMENT 'Payout maximo por rodada (GCoins)',
    auto_max_picks        TINYINT        NOT NULL DEFAULT 24    COMMENT 'Max tiles no auto mode',
    orphan_timeout_min    INT            NOT NULL DEFAULT 30    COMMENT 'Minutos para cancelar rodada orfã',
    rate_limit_ms         INT            NOT NULL DEFAULT 200   COMMENT 'Cooldown entre acoes do jogador (ms)',
    enabled               TINYINT(1)     NOT NULL DEFAULT 1     COMMENT 'Mines ativo?',
    updated_at            DATETIME       NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    updated_by            VARCHAR(64)    DEFAULT NULL            COMMENT 'Admin que alterou'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
COMMENT='Configuracao do Mines — 1 row, id=1';

INSERT INTO casino_mines_config (id) VALUES (1)
ON DUPLICATE KEY UPDATE id = id;

-- --- TABELA 2: ROUNDS (historico de rodadas) ---
CREATE TABLE IF NOT EXISTS casino_mines_rounds (
    id                    INT AUTO_INCREMENT PRIMARY KEY,
    player_id             VARCHAR(64)    NOT NULL               COMMENT 'Identificador do jogador',
    bet                   INT            NOT NULL               COMMENT 'Valor apostado (GCoins)',
    mines_count           TINYINT        NOT NULL               COMMENT 'Quantidade de minas (1-24)',
    gems_revealed         TINYINT        NOT NULL DEFAULT 0     COMMENT 'Gemas reveladas antes do fim',
    multiplier            DECIMAL(10,4)  NOT NULL DEFAULT 1.0000 COMMENT 'Multiplicador final',
    payout                INT            NOT NULL DEFAULT 0     COMMENT 'Valor pago ao jogador (0 se bust)',
    result                ENUM('active','win','bust','cancelled') NOT NULL DEFAULT 'active',
    grid_data             JSON           NOT NULL               COMMENT 'Posicoes das minas (array 25 gem/bomb)',
    tiles_revealed        JSON           DEFAULT NULL           COMMENT 'Indices revelados pelo jogador (array int)',
    seed_id               INT            NOT NULL               COMMENT 'FK para casino_mines_seeds',
    nonce                 INT            NOT NULL               COMMENT 'Nonce usado nesta rodada',
    created_at            DATETIME       NOT NULL DEFAULT CURRENT_TIMESTAMP,
    finished_at           DATETIME       DEFAULT NULL,
    INDEX idx_player (player_id),
    INDEX idx_result (result),
    INDEX idx_created (created_at),
    INDEX idx_seed (seed_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
COMMENT='Historico de rodadas do Mines';

-- --- TABELA 3: SEEDS (Provably Fair) ---
CREATE TABLE IF NOT EXISTS casino_mines_seeds (
    id                    INT AUTO_INCREMENT PRIMARY KEY,
    player_id             VARCHAR(64)    NOT NULL               COMMENT 'Jogador dono deste par de seeds',
    server_seed           VARCHAR(128)   NOT NULL               COMMENT 'Seed secreta do servidor (hex 64 chars)',
    server_seed_hash      VARCHAR(128)   NOT NULL               COMMENT 'SHA256 do server_seed (publica)',
    client_seed           VARCHAR(128)   NOT NULL DEFAULT 'default' COMMENT 'Seed do cliente (editavel)',
    nonce                 INT            NOT NULL DEFAULT 0     COMMENT 'Nonce atual (incrementa a cada rodada)',
    revealed              TINYINT(1)     NOT NULL DEFAULT 0     COMMENT '1 = seed revelada (rotacionada)',
    created_at            DATETIME       NOT NULL DEFAULT CURRENT_TIMESTAMP,
    revealed_at           DATETIME       DEFAULT NULL,
    INDEX idx_player (player_id),
    INDEX idx_revealed (revealed)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
COMMENT='Seeds Provably Fair do Mines';
```

## 1.2 Queries Admin (monitoramento e ajuste)

```sql
-- Ver config atual
SELECT * FROM casino_mines_config WHERE id = 1;

-- Alterar house edge para 2% (RTP 98%)
UPDATE casino_mines_config SET house_edge_percent = 2.00, updated_by = 'admin' WHERE id = 1;

-- Ultimas 20 rodadas de um jogador
SELECT id, bet, mines_count, gems_revealed, multiplier, payout, result, created_at
FROM casino_mines_rounds
WHERE player_id = ?
ORDER BY created_at DESC
LIMIT 20;

-- Receita do casino hoje
SELECT
    COUNT(*) AS total_rodadas,
    SUM(bet) AS total_apostado,
    SUM(payout) AS total_pago,
    SUM(bet) - SUM(payout) AS lucro_bruto
FROM casino_mines_rounds
WHERE result IN ('win', 'bust')
AND DATE(created_at) = CURDATE();

-- Rodadas orfas (ativas ha mais de 30 min)
SELECT id, player_id, bet, created_at,
    TIMESTAMPDIFF(MINUTE, created_at, NOW()) AS minutos_ativa
FROM casino_mines_rounds
WHERE result = 'active'
AND created_at < DATE_SUB(NOW(), INTERVAL 30 MINUTE);

-- Cancelar rodadas orfas e devolver aposta
UPDATE casino_mines_rounds
SET result = 'cancelled', payout = bet, finished_at = NOW()
WHERE result = 'active'
AND created_at < DATE_SUB(NOW(), INTERVAL 30 MINUTE);
```
