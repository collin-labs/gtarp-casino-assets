As 16 features listadas como "FUNCIONAL" na Parte 1 merecem descricao detalhada de COMO funcionam, nao apenas QUE existem. Abaixo as 6 mais importantes com profundidade (as demais sao autoexplicativas pela tabela).

## F1. Intro Cinematografica (BlackoutCasino.tsx linhas 39-48)

A abertura do painel segue uma sequencia de 4 stages controlada por setTimeout encadeados. Nao usa Framer Motion timeline — usa state puro com `introStage` que incrementa de 0 a 4. Cada stage revela um elemento:

- **Stage 0 (t=0ms):** Nada visivel. Estado inicial.
- **Stage 1 (t=50ms):** Overlay escuro aparece com `opacity: 0→1` em 0.5s ease via Framer Motion `animate`. Background: `rgba(0,0,0,0.78)` + `backdropFilter: blur(3px)`. Cobre toda a tela do GTA.
- **Stage 2 (t=150ms):** Painel flutuante aparece com `scale: 0.85→1`, `opacity: 0→1`, `y: 40→0` em 0.8s ease via Framer Motion. O painel tem fundo `bg-casino.png` com camadas de vinheta (`radial-gradient ellipse`) e escurecimento (`rgba(0,0,0,0.15)`).
- **Stage 3 (t=600ms):** Letreiro dourado aparece no topo com `y: 20→0`, `opacity: 0→1`, `scale: 0.9→1` em 0.6s. Posicionado com `translateY(-60%)` — fica "saindo" da borda superior do painel. Halo dourado pulsante atras. Texto marquee: "BLACKOUT CASINO · O CASSINO MAIS EXCLUSIVO DE LOS SANTOS".
- **Stage 4 (t=850ms):** Dock aparece na base com `y: 20→0`, `opacity: 0→1`, `scale: 0.9→1` em 0.55s. Posicionado com `translateY(42%)` — "saindo" da borda inferior. 5 abas visiveis.

**Problema tecnico:** Os timings sao hardcoded com setTimeout, nao com Framer Motion `staggerChildren`. Se o browser lag durante a intro (carregando imagens), os stages podem se sobrepor. Uma timeline Framer Motion seria mais robusta.

## F2. GameCard com 7 Camadas de Efeitos (GameCard.tsx — 219 linhas)

O GameCard eh o componente visual mais elaborado do painel. Cada card renderiza 7 camadas de efeitos sobrepostos que criam a sensacao premium:

**Camada 1 — Container com borda dupla:** `border: 2px solid rgba(212,168,67, 0.3-0.6)` normal, `rgba(212,168,67, 0.7)` hover. `outline: 1.5px solid rgba(191,149,63, 0.1-0.45)` com `outline-offset: 2-4px`. Hover muda outline para verde `rgba(0,230,118, 0.4)`.

**Camada 2 — Imagem de fundo:** `<img src={game.cardUrl}>` com `object-fit: cover`, `width/height: 100%`. Imagens variam por jogo (1152x768, 1024x1024, 768x1024, etc conforme posicao no grid).

**Camada 3 — Gradiente escurecimento topo:** `linear-gradient(to bottom, rgba(0,0,0,0.8) 0%, rgba(0,0,0,0.3) 60%, transparent 100%)`. Garante legibilidade do titulo do jogo.

**Camada 4 — Gradiente escurecimento base:** `linear-gradient(to top, rgba(0,0,0,0.8) 0%, rgba(0,0,0,0.3) 50%, transparent 100%)`. Garante legibilidade da area inferior.

**Camada 5 — Feixo verde permanente (base):** `height: 3px`, `background: linear-gradient(90deg, transparent 5%, rgba(0,230,118,0.3) 20%, #00E676 50%, rgba(0,230,118,0.3) 80%, transparent 95%)`, `box-shadow: 0 0 8px rgba(0,230,118,0.3)`. Linha neon verde SEMPRE visivel na base do card.

**Camada 6 — Linha dourada (topo):** `height: 1px`, `background: linear-gradient(90deg, transparent 10%, rgba(212,168,67,0.3) 30%, rgba(255,215,0,0.5) 50%, rgba(212,168,67,0.3) 70%, transparent 90%)`. Sutil mas presente.

**Camada 7 — Efeitos hover (3 sub-efeitos):**
- **Mouse spotlight:** `radial-gradient(circle 120px at ${mouseX}% ${mouseY}%, rgba(0,230,118,0.2), rgba(255,215,0,0.06) 40%, transparent 70%)`. Segue o cursor em tempo real.
- **Shimmer sweep:** Div com `animate-shimmer` (CSS keyframe em globals.css). Gradiente dourado que percorre o card horizontalmente em loop.
- **Neon line hover:** Intensifica o feixo verde base e adiciona linha dourada no topo com `#FFD700`.

## F3. BentoGrid com 5 Layouts Internos (BentoGrid.tsx — 503 linhas)

O BentoGrid nao eh um grid simples — eh um componente que renderiza 5 layouts COMPLETAMENTE DIFERENTES conforme a aba selecionada. Cada layout eh uma funcao interna separada:

- **ArcadeLayout** (linhas 116-177): Hero ocupa ~60% esquerda. 4 cards em grid 2x2 a direita. Hero renderiza HeroCarousel com imagem pedestal. Cards usam GameCard com efeitos.
- **CasinoLayout** (linhas 178-272): Hero grande a esquerda com HeroCarousel. 4 cards 2x2 a direita. 2 banners slim (730x130) na parte inferior cobrindo toda a largura. Layout mais complexo — 3 zonas (hero + grid + banners).
- **TripleLayout** (linhas 273-315): 3 cards GRANDES lado a lado, sem hero. Cada card ocupa ~33% da largura. Usado na aba PVP (Case Battle, Coinflip, Jackpot).
- **QuadLayout** (linhas 316-371): 4 cards em grid 2x2 igual, sem hero. Usado na aba LOJA (Cases, Upgrade, Marketplace, Inventory).
- **SpotlightLayout** (componente separado — 372 linhas): 3 cards com spotlight no selecionado. Logo do jogo no topo de cada card. Card ativo tem efeitos premium amplificados. AnimatePresence para transicao entre spotlight.

Adicionalmente, BentoGrid contem 2 componentes decorativos internos: **HeroBeams** (linhas 372-426) que renderiza raios de luz dourada atras do hero, e **HeroBrokenBorder** (linhas 427-502) que renderiza uma borda decorativa "quebrada" com segmentos dourados.

## F4. GoldParticles — Canvas 2D Completo (GoldParticles.tsx — 208 linhas)

O sistema de particulas funciona assim: no mount, cria array de particulas com posicao/velocidade/opacidade aleatorias. Cada frame (requestAnimationFrame), move cada particula, verifica colisao com bordas (bounce), aplica "repel" do mouse (particulas se afastam do cursor), e renderiza no Canvas 2D com `arc()` + `fill()`. Cores alternadas entre dourado `rgba(212,168,67,X)` e branco `rgba(255,255,255,X)` com opacidade variavel (0.1-0.7).

**Metricas atuais:** ~50 particulas simultaneas. RAF a 60fps (SEM throttle). Radius 1-3px. Mouse repel radius ~100px. Canvas resolution = devicePixelRatio (2x em telas HiDPI = 4x os pixels).

**Impacto performance:**
- Main thread bloqueado por draw loop a cada 16.6ms
- Em tela 1920x1080 com DPR 2: canvas de 3840x2160 pixels renderizado a cada frame
- 50 particulas x arc() + fill() = 100 draw calls por frame
- Nao pausa quando jogo abre (GPU waste confirmado)
