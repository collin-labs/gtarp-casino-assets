# INSTRUCAO PARA O V0

Este documento explica como TODAS as imagens do Painel Flutuante foram criadas e onde cada uma esta armazenada. As imagens foram geradas via GPT Image 1.5 seguindo estudo de biblioteca de imagens com 7 varreduras obrigatorias nos 8 documentos anteriores do painel.

**COMO FOI FEITO:**
1. Analisamos 8 documentos do painel (Pesquisa, Mega Estudo, Guia, Prompt AI, Adendo DS, Roteiro, Prompt-V0, CSS-Componentes) + codigo fonte (10 componentes TSX + lib/assets.ts + lib/games.ts)
2. Executamos 7 varreduras sistematicas para encontrar CADA referencia de imagem no projeto
3. Mapeamos 142 imagens: 8 UI compartilhadas + 110 assets dos 22 jogos (5 por jogo) + 12 icons X21 + 4 badges + 8 adicionais (bgAlt, dockComTexto, logo1x1, etc.)
4. Geramos 24 prompts individuais para as imagens NOVAS com estrutura OBJECT/STYLE/RENDER/COMPOSITION/TECHNICAL/NEGATIVE
5. Cada imagem gerada via GPT Image 1.5 com `background:"transparent"` (PNG alpha)
6. Imagens salvas nas pastas do projeto seguindo nomenclatura padrao de lib/assets.ts

**REGRA PARA O V0:** Ao gerar componentes React, usar os paths EXATOS listados neste guia via `import { UI } from '@/lib/assets'` e `import { GAMES } from '@/lib/games'`. NAO inventar paths. NAO usar placeholders. Cada `<img src=...>` e `backgroundImage: url(...)` deve apontar para os arquivos reais.

**REGRA DE ICONES (X21):** O painel usa icones PNG customizados dourados, NAO Lucide icons. Todos os icones do painel estao em `/assets/ui/panel-icons/`. Tamanho: width/height com clamp(16px, 2vw, 24px). Hover: `filter: brightness(1.2)` + `transform: scale(1.1)`. Lucide permitido APENAS para: botao fechar [X] de DepositModal.
