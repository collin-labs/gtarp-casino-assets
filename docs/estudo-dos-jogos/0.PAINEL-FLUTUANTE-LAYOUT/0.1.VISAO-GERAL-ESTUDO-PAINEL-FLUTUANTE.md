# 1. O QUE E O PAINEL FLUTUANTE

O Painel Flutuante e a moldura/shell que abriga os 22 jogos do Blackout Casino dentro do GTA RP (FiveM). Ele nao e um jogo — e a interface principal que o jogador ve ao abrir o cassino. Aparece centralizado na tela (80vw, aspect ratio 16/8, max 78vh) como um overlay flutuante sobre o mundo do GTA, com blur no fundo e uma entrada cinematografica em 4 stages.

O painel contem 5 abas (CASSINO, ARCADE, PVP, LOJA, EVENTOS) com 22 jogos distribuidos, navegacao por Dock ornamentado na base, carrossel hero, cards com 7 camadas de efeitos visuais, particulas douradas flutuantes, spotlight que segue o cursor, e scan line HUD.

# 3. O QUE OS 10 DOCUMENTOS CRIARAM

15.183 linhas em 27 partes. Cada documento serve um proposito especifico:

## Doc 0 — Pesquisa REGRA 0 (959L, 3 partes)

**Proposito:** Fundamentar todas as decisoes com dados reais.

Analisou cassinos reais (Stake, BC.Game, CSGOEmpire, Roobet, Gamdom) para entender padroes de navegacao, transicoes, responsividade, dark theme. Pesquisou CSS 2026 (@property, container queries, view transitions), Framer Motion para paineis, Canvas para particulas, limitacoes FiveM NUI/CEF (Chromium Embedded Framework). Resultado: 14 descobertas concretas que guiaram todas as decisoes dos documentos seguintes.

Exemplos de descobertas que impactaram o projeto:
- BC.Game usa grid flexivel com hero ocupando 50% — adotamos no layout CASSINO
- CSGOEmpire confirma que 50 particulas e o maximo antes de impactar FPS
- Container queries ainda nao sao seguros em CEF — mantemos clamp()
- Framer Motion layoutId e melhor que calcular offsetLeft para tab indicator

## Doc 1 — Mega Estudo (1.181L, 3 partes)

**Proposito:** Inventario completo do que existe e diagnostico.

Mapeou todos os 10 componentes com linhas, props, dependencias, o que funciona, o que esta parcial, o que falta. Identificou problemas de performance (GoldParticles sem throttle, sem lazy load, sem paused prop). Catalogou todos os dados de lib/games.ts e lib/assets.ts. Resultado: plano de 5+ fases com metricas de sucesso.

## Doc 2 — Guia do Painel (274L, 1 parte)

**Proposito:** Visao geral para quem nunca viu o painel.

Documenta as 5 abas, fluxos de navegacao (como o jogador abre o painel, navega entre abas, abre um jogo, deposita), features unicas (7 camadas por card, atmosferas por aba, intro cinematografica), diferenciais vs concorrentes.

## Doc 3 — Prompt AI (2.479L, 4 partes)

## 5.5 Performance (Doc 5 Fase 3)

| Melhoria | Antes | Depois |
|----------|-------|--------|
| GoldParticles GPU | Roda sempre | paused={true} quando jogo ativo (0% GPU) |
| GoldParticles FPS | Sem controle | Throttle 30fps com delta time clamp |
| Bundle Framer Motion | ~30kb | LazyMotion domAnimation ~15kb |
| Jogos carregamento | Todos no bundle | React.lazy + Suspense (~50KB a menos) |
| Tab indicator | Calcula offsetLeft | layoutId spring (Framer calcula automatico) |

## 5.6 Interacoes Refinadas (Doc 5 Fase 6)

- **ESC handler hierarquico:** Jogo ativo → fecha jogo. Modal aberto → fecha modal. Painel → fecha painel. Hoje provavelmente fecha tudo de uma vez.
- **10 sons:** panel_open, tab_switch, card_hover, card_click, modal_open, game_launch, game_back, panel_close, deposit_confirm, error_buzz
- **25 tooltips bilingues:** TODOS os elementos interativos com title={{ br: "...", en: "..." }}
- **Dock layoutId:** A barra ativa DESLIZA suavemente entre abas com spring 500/30 (hoje snap instantaneo)

## 5.7 Backend FiveM (Doc 5 Fase 1)

| Componente | Funcao |
|-----------|--------|
| casino_panel_schema.sql | 3 tabelas: config (seeds, limites), transactions (depositos/saques), audit (log completo) |
| server/handlers/panel.js | 5 endpoints: getBalance, deposit, withdraw, getConfig, logAction — MySQL.prepare (nunca concatenar SQL) |
| client/panel_client.lua | RegisterNUICallback com cb({}) obrigatorio — ponte entre NUI e servidor |

Hoje o painel e front-end puro. Com o backend, ele se conecta ao servidor FiveM real: saldo do jogador, depositos, saques, configuracoes do admin.
