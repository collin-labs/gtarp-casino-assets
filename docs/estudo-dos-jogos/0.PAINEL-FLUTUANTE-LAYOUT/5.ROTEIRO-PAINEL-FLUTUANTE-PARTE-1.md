**Fontes relevantes:**
- FiveM Docs (docs.fivem.net/docs/scripting-manual/nui-development/nui-callbacks/): RegisterNUICallback com POST data parsed como JSON automaticamente. Callback OBRIGATORIO mesmo sem retorno — `cb({})` ou JS trava aguardando
- Cfx-Store/react-nui (github.com): Pattern `fetchNui<T>(event, data, mockData)` com TypeScript generics + mock para desenvolvimento browser sem FiveM ativo
- FiveM-React-Lua-Boilerplate (enty.gitbook.io): fetchNui e wrapper simples do fetch API padrao, POST para `https://cfx-nui-${GetCurrentResourceName()}/${event}`
- OxMySQL Docs (coxdocs.dev/oxmysql): `MySQL.prepare` com `?` positional params, `ON DUPLICATE KEY UPDATE` preferido sobre check-then-insert, npm `@communityox/oxmysql` para TypeScript
- GitHub overextended/oxmysql: node-mysql2 base, prepared statements, connection pooling, promise/await async por default
- fivemx.com (Ago 2025): MariaDB recomendado sobre MySQL 8 para FiveM — sem reserved keywords conflicts, melhor performance
- Motion docs (motion.dev/docs/react-layout-animations): `layout` prop usa CSS transform (compositor thread, 60fps). `layoutId` para shared element transitions entre abas
- inhaq.com (Mar 2026): Motion v12 com hybrid engine (Web Animations API + JS fallback). `LazyMotion` com `domAnimation` reduz bundle de 30kb para 15kb
- react-news.com (Dez 2025): `mode="popLayout"` remove exiting element do layout flow imediatamente, permitindo itens restantes deslizarem enquanto item deletado faz fade out

**O que a pesquisa trouxe de novo para o Roteiro:**
1. **fetchNui com mock** — pattern `fetchNui<T>(event, data, mockData)` permite desenvolvimento web sem FiveM ativo. O hook `useGameAPI` DEVE implementar esse fallback para BC testar no browser
2. **Callback `cb({})` obrigatorio** — mesmo quando nao retorna dados, o Lua RegisterNUICallback DEVE chamar `cb({})` ou o JS trava aguardando resposta indefinidamente
3. **MySQL.prepare obrigatorio** — oxmysql com `?` params previne SQL injection. NUNCA concatenar strings SQL
4. **ON DUPLICATE KEY UPDATE** — preferido sobre check-then-insert para seeds e config (idempotente)
5. **MariaDB > MySQL 8** — MariaDB recomendado para FiveM (sem reserved keywords conflicts, melhor perf)
6. **LazyMotion domAnimation** — pode reduzir bundle Framer de 30kb para 15kb, relevante para performance NUI
7. **mode="popLayout"** — essencial se futuramente o painel tiver listas dinamicas

**O que teriamos feito sem a pesquisa:**
- fetchNui sem mock → impossivel testar no browser, so funciona dentro do FiveM
- RegisterNUICallback sem `cb({})` → JS travaria silenciosamente aguardando resposta
- SQL com concatenacao de strings → vulneravel a SQL injection
- MySQL 8 em producao → conflitos com reserved keywords do FiveM
- Framer Motion bundle completo (30kb) ao inves de LazyMotion (15kb) → impacto desnecessario no CEF

## 1.2 Criar arquivo `server/handlers/panel.js`

**Arquivo NOVO — handler completo do painel com 6 endpoints.** Cada endpoint valida input, consulta banco, registra audit, retorna resultado. Zero pseudocodigo.

```js
// ====================================================
// BLACKOUT CASINO — PAINEL FLUTUANTE — SERVER HANDLER
// server/handlers/panel.js
// Versao 1.0 - 23/03/2026
// Dependencias: oxmysql (MySQL.prepare, MySQL.single, MySQL.insert)
// ====================================================

// -- Cache de config (atualiza a cada 60s ou em CONFIG_CHANGE)
let panelConfig = null;
let configLastFetch = 0;
const CONFIG_TTL = 60000; // 60 segundos

async function getConfig() {
  const now = Date.now();
  if (panelConfig && (now - configLastFetch) < CONFIG_TTL) {
    return panelConfig;
  }
  const row = await MySQL.single.await(
    'SELECT * FROM casino_panel_config WHERE id = 1'
  );
  if (!row) {
    console.error('[PANEL] Config nao encontrada — tabela vazia?');
    return null;
  }
  // Parse JSON fields
  row.vip_tiers = typeof row.vip_tiers === 'string'
    ? JSON.parse(row.vip_tiers) : row.vip_tiers;
  row.vip_thresholds = typeof row.vip_thresholds === 'string'
    ? JSON.parse(row.vip_thresholds) : row.vip_thresholds;
  panelConfig = row;
  configLastFetch = now;
  return panelConfig;
}

// -- Helper: obter saldo GCoin do jogador (depende do framework: vRP/ESX/QBCore)
// ADAPTAR para o framework do servidor do BC
async function getPlayerGCoinBalance(playerId) {
  const row = await MySQL.single.await(
    'SELECT gcoin_balance FROM casino_players WHERE player_id = ?',
    [playerId]
  );
  return row ? row.gcoin_balance : 0;
}

// -- Helper: atualizar saldo GCoin
async function setPlayerGCoinBalance(playerId, newBalance) {
  await MySQL.prepare.await(
    'UPDATE casino_players SET gcoin_balance = ?, updated_at = NOW() WHERE player_id = ?',
    [newBalance, playerId]
  );
}

// -- Helper: obter saldo R$ do servidor (framework-dependent)
async function getPlayerServerMoney(playerId) {
  // vRP: vRP.getMoney(user_id) via server-side
  // ESX: xPlayer.getMoney()
  // QBCore: Player.Functions.GetMoney('cash')
  // ADAPTAR para framework do BC
  const row = await MySQL.single.await(
    'SELECT money FROM vrp_user_moneys WHERE user_id = ? AND money_type = ?',
    [playerId, 'cash']
  );
  return row ? row.money : 0;
}

// -- Helper: debitar R$ do servidor
async function debitServerMoney(playerId, amount) {
  await MySQL.prepare.await(
    'UPDATE vrp_user_moneys SET money = money - ? WHERE user_id = ? AND money_type = ?',
    [amount, playerId, 'cash']
  );
}

// -- Helper: creditar R$ do servidor
async function creditServerMoney(playerId, amount) {
  await MySQL.prepare.await(
    'UPDATE vrp_user_moneys SET money = money + ? WHERE user_id = ? AND money_type = ?',
    [amount, playerId, 'cash']
  );
}

// -- Helper: audit log
async function logAudit(playerId, action, details) {
  await MySQL.insert.await(
    'INSERT INTO casino_panel_audit (player_id, action, details) VALUES (?, ?, ?)',
    [playerId, action, details ? JSON.stringify(details) : null]
  );
}

// -- Helper: total depositado hoje por jogador
async function getTodayDeposits(playerId) {
  const row = await MySQL.single.await(
    'SELECT COALESCE(SUM(amount_server), 0) AS total FROM casino_panel_transactions WHERE player_id = ? AND type = ? AND DATE(created_at) = CURDATE()',
    [playerId, 'deposit']
  );
  return row ? row.total : 0;
}

// -- Helper: total sacado hoje por jogador
async function getTodayWithdraws(playerId) {
  const row = await MySQL.single.await(
    'SELECT COALESCE(SUM(amount_gcoin), 0) AS total FROM casino_panel_transactions WHERE player_id = ? AND type = ? AND DATE(created_at) = CURDATE()',
    [playerId, 'withdraw']
  );
  return row ? row.total : 0;
}

// -- Helper: calcular VIP tier baseado no total depositado
function calculateVipTier(totalDeposited, config) {
  const tiers = config.vip_tiers;
  const thresholds = config.vip_thresholds;
  let tier = tiers[0]; // bronze padrao
  for (let i = thresholds.length - 1; i >= 0; i--) {
    if (totalDeposited >= thresholds[i]) {
      tier = tiers[i];
      break;
    }
  }
  return tier;
}

// ====================================================
// ENDPOINT 1: openCasino — jogador abriu o painel
// ====================================================
RegisterNUICallback('casino:panel:open', async (data, cb) => {
  try {
    const playerId = source; // FiveM source do jogador
    const config = await getConfig();

    if (!config || !config.enabled) {
      cb({ success: false, error: 'Cassino desabilitado' });
      return;
    }

    if (config.maintenance_mode) {
      cb({ success: false, error: config.maintenance_message });
      return;
    }

    const gcoinBalance = await getPlayerGCoinBalance(playerId);
    const serverMoney = await getPlayerServerMoney(playerId);

    // Calcular total depositado (lifetime) para VIP
    const totalRow = await MySQL.single.await(
      'SELECT COALESCE(SUM(amount_server), 0) AS total FROM casino_panel_transactions WHERE player_id = ? AND type = ?',
      [playerId, 'deposit']
    );
    const totalDeposited = totalRow ? totalRow.total : 0;
    const vipTier = calculateVipTier(totalDeposited, config);

    await logAudit(playerId, 'OPEN_PANEL', { gcoinBalance, vipTier });

    cb({
      success: true,
      gcoinBalance: gcoinBalance,
      serverMoney: serverMoney,
      vipTier: vipTier,
      config: {
        exchangeRate: parseFloat(config.exchange_rate),
        minDeposit: config.min_deposit,
        maxDeposit: config.max_deposit,
        minWithdraw: config.min_withdraw,
        maxWithdraw: config.max_withdraw,
        depositEnabled: !!config.deposit_enabled,
        withdrawEnabled: !!config.withdraw_enabled,
        dailyDepositLimit: config.daily_deposit_limit,
        dailyWithdrawLimit: config.daily_withdraw_limit
      }
    });
  } catch (err) {
    console.error('[PANEL] Erro em openCasino:', err);
    cb({ success: false, error: 'Erro interno do servidor' });
  }
});

// ====================================================
// ENDPOINT 2: deposit — depositar R$ servidor → GCoin
// ====================================================
RegisterNUICallback('casino:panel:deposit', async (data, cb) => {
  try {
    const playerId = source;
    const amount = parseInt(data.amount, 10);
    const config = await getConfig();

    // Validacoes
    if (!config || !config.deposit_enabled) {
      cb({ success: false, error: 'Depositos desabilitados' });
      return;
    }
    if (isNaN(amount) || amount <= 0) {
      cb({ success: false, error: 'Valor invalido' });
      return;
    }
    if (amount < config.min_deposit) {
      cb({ success: false, error: `Minimo: R$ ${config.min_deposit}` });
      return;
    }
    if (amount > config.max_deposit) {
      cb({ success: false, error: `Maximo: R$ ${config.max_deposit}` });
      return;
    }

    // Verificar limite diario
    const todayTotal = await getTodayDeposits(playerId);
    if (todayTotal + amount > config.daily_deposit_limit) {
      const remaining = config.daily_deposit_limit - todayTotal;
      cb({ success: false, error: `Limite diario atingido. Restante: R$ ${remaining}` });
      return;
    }

    // Verificar saldo R$ servidor
    const serverMoney = await getPlayerServerMoney(playerId);
    if (serverMoney < amount) {
      cb({ success: false, error: `Saldo insuficiente. Disponivel: R$ ${serverMoney}` });
      return;
    }

    // Calcular GCoins a receber
    const exchangeRate = parseFloat(config.exchange_rate);
    const gcoinAmount = Math.floor(amount * exchangeRate);
    const currentGCoin = await getPlayerGCoinBalance(playerId);
    const newGCoin = currentGCoin + gcoinAmount;

    // Executar transacao
    await debitServerMoney(playerId, amount);
    await setPlayerGCoinBalance(playerId, newGCoin);

    // Registrar transacao
    await MySQL.insert.await(
      'INSERT INTO casino_panel_transactions (player_id, type, amount_server, amount_gcoin, exchange_rate, balance_before, balance_after) VALUES (?, ?, ?, ?, ?, ?, ?)',
      [playerId, 'deposit', amount, gcoinAmount, exchangeRate, currentGCoin, newGCoin]
    );

    await logAudit(playerId, 'DEPOSIT', { amount, gcoinAmount, exchangeRate, balanceBefore: currentGCoin, balanceAfter: newGCoin });

    cb({
      success: true,
      gcoinAmount: gcoinAmount,
      newBalance: newGCoin,
      serverMoneyRemaining: serverMoney - amount
    });
  } catch (err) {
    console.error('[PANEL] Erro em deposit:', err);
    cb({ success: false, error: 'Erro interno do servidor' });
  }
});

// ====================================================
// ENDPOINT 3: withdraw — sacar GCoin → R$ servidor
// ====================================================
RegisterNUICallback('casino:panel:withdraw', async (data, cb) => {
  try {
    const playerId = source;
    const gcoinAmount = parseInt(data.amount, 10);
    const config = await getConfig();

    if (!config || !config.withdraw_enabled) {
      cb({ success: false, error: 'Saques desabilitados' });
      return;
    }
    if (isNaN(gcoinAmount) || gcoinAmount <= 0) {
      cb({ success: false, error: 'Valor invalido' });
      return;
    }
    if (gcoinAmount < config.min_withdraw) {
      cb({ success: false, error: `Minimo: G$ ${config.min_withdraw}` });
      return;
    }
    if (gcoinAmount > config.max_withdraw) {
      cb({ success: false, error: `Maximo: G$ ${config.max_withdraw}` });
      return;
    }

    // Verificar limite diario
    const todayTotal = await getTodayWithdraws(playerId);
    if (todayTotal + gcoinAmount > config.daily_withdraw_limit) {
      const remaining = config.daily_withdraw_limit - todayTotal;
      cb({ success: false, error: `Limite diario atingido. Restante: G$ ${remaining}` });
      return;
    }

    // Verificar saldo GCoin
    const currentGCoin = await getPlayerGCoinBalance(playerId);
    if (currentGCoin < gcoinAmount) {
      cb({ success: false, error: `Saldo insuficiente. Disponivel: G$ ${currentGCoin}` });
      return;
    }

    // Calcular R$ a receber
    const exchangeRate = parseFloat(config.exchange_rate);
    const serverAmount = Math.floor(gcoinAmount / exchangeRate);
    const newGCoin = currentGCoin - gcoinAmount;

    // Executar transacao
    await setPlayerGCoinBalance(playerId, newGCoin);
    await creditServerMoney(playerId, serverAmount);

    // Registrar transacao
    await MySQL.insert.await(
      'INSERT INTO casino_panel_transactions (player_id, type, amount_server, amount_gcoin, exchange_rate, balance_before, balance_after) VALUES (?, ?, ?, ?, ?, ?, ?)',
      [playerId, 'withdraw', serverAmount, gcoinAmount, exchangeRate, currentGCoin, newGCoin]
    );

    await logAudit(playerId, 'WITHDRAW', { gcoinAmount, serverAmount, exchangeRate, balanceBefore: currentGCoin, balanceAfter: newGCoin });

    cb({
      success: true,
      serverAmount: serverAmount,
      newBalance: newGCoin
    });
  } catch (err) {
    console.error('[PANEL] Erro em withdraw:', err);
    cb({ success: false, error: 'Erro interno do servidor' });
  }
});

// ====================================================
// ENDPOINT 4: getSaldo — atualizar saldo em tempo real
// ====================================================
RegisterNUICallback('casino:panel:getSaldo', async (data, cb) => {
  try {
    const playerId = source;
    const gcoinBalance = await getPlayerGCoinBalance(playerId);
    cb({ success: true, gcoinBalance: gcoinBalance });
  } catch (err) {
    console.error('[PANEL] Erro em getSaldo:', err);
    cb({ success: false, error: 'Erro ao buscar saldo' });
  }
});

// ====================================================
// ENDPOINT 5: getConfig — config publica (sem dados admin)
// ====================================================
RegisterNUICallback('casino:panel:getConfig', async (data, cb) => {
  try {
    const config = await getConfig();
    if (!config) {
      cb({ success: false, error: 'Config indisponivel' });
      return;
    }
    cb({
      success: true,
      config: {
        exchangeRate: parseFloat(config.exchange_rate),
        minDeposit: config.min_deposit,
        maxDeposit: config.max_deposit,
        minWithdraw: config.min_withdraw,
        maxWithdraw: config.max_withdraw,
        depositEnabled: !!config.deposit_enabled,
        withdrawEnabled: !!config.withdraw_enabled,
        maintenanceMode: !!config.maintenance_mode,
        maintenanceMessage: config.maintenance_message
      }
    });
  } catch (err) {
    console.error('[PANEL] Erro em getConfig:', err);
    cb({ success: false, error: 'Erro ao buscar config' });
  }
});

// ====================================================
// ENDPOINT 6: closePanel — jogador fechou o painel
// ====================================================
RegisterNUICallback('casino:panel:close', async (data, cb) => {
  try {
    const playerId = source;
    await logAudit(playerId, 'CLOSE_PANEL', null);
    cb({ success: true });
  } catch (err) {
    cb({ success: true }); // nao bloquear fechamento por erro de audit
  }
});
```

**Linhas:** ~280

**Por que 6 endpoints e nao mais:** O painel e uma MOLDURA — nao tem logica de jogo. Cada jogo individual (slots, poker, etc.) tem seus proprios endpoints no handler do jogo. O painel so precisa: abrir, fechar, depositar, sacar, consultar saldo, consultar config.

**Seguranca implementada:**
- `MySQL.prepare` com `?` em TODAS as queries (previne SQL injection — pesquisa X0)
- Validacao de input em CADA endpoint (isNaN, range min/max)
- Limites diarios por jogador (anti-abuse)
- Audit log em TODA acao (REGRA X12)
- Config cache com TTL 60s (evita query a cada request)
- try/catch em TODOS endpoints (nunca crash o server)

## 1.3 Criar arquivo `client/panel_client.lua`

**Arquivo NOVO — bridge Lua entre NUI React e Server FiveM.** Cada RegisterNUICallback corresponde a um endpoint no handler.

```lua
-- ====================================================
-- BLACKOUT CASINO — PAINEL FLUTUANTE — CLIENT LUA
-- client/panel_client.lua
-- Versao 1.0 - 23/03/2026
-- Bridge entre NUI (React) e Server (panel.js)
-- ====================================================

local isOpen = false

-- Comando para abrir o cassino (tecla E ou /cassino)
RegisterCommand('cassino', function()
    if isOpen then return end
    isOpen = true
    SetNuiFocus(true, true)

    -- Pedir dados ao server antes de mostrar o NUI
    TriggerServerEvent('casino:panel:requestOpen')
end, false)

-- Hotkey E para abrir (opcional — configuravel)
RegisterKeyMapping('cassino', 'Abrir Cassino Blackout', 'keyboard', 'E')

-- Server responde com dados iniciais
RegisterNetEvent('casino:panel:openData', function(payload)
    if not payload.success then
        SetNuiFocus(false, false)
        isOpen = false
        -- Mostrar notificacao de erro (ex: manutencao)
        TriggerEvent('chat:addMessage', {
            args = { '[CASSINO]', payload.error or 'Erro ao abrir cassino' }
        })
        return
    end

    SendNUIMessage({
        action = 'openCasino',
        data = payload
    })
end)

-- NUI pede para fechar
RegisterNUICallback('casino:panel:close', function(data, cb)
    SetNuiFocus(false, false)
    isOpen = false
    TriggerServerEvent('casino:panel:close')
    cb({ success = true })
end)

-- ESC para fechar
RegisterNUICallback('casino:panel:escape', function(data, cb)
    SetNuiFocus(false, false)
    isOpen = false
    TriggerServerEvent('casino:panel:close')
    cb({ success = true })
end)

-- Bridge: NUI → Server → NUI (deposito)
RegisterNUICallback('casino:panel:deposit', function(data, cb)
    TriggerServerEvent('casino:panel:deposit', data)
    -- Server responde via evento
end)

RegisterNetEvent('casino:panel:depositResult', function(result)
    SendNUIMessage({
        action = 'depositResult',
        data = result
    })
end)

-- Bridge: NUI → Server → NUI (saque)
RegisterNUICallback('casino:panel:withdraw', function(data, cb)
    TriggerServerEvent('casino:panel:withdraw', data)
    cb({}) -- cb obrigatorio (pesquisa X0 — previne JS travando)
end)

RegisterNetEvent('casino:panel:withdrawResult', function(result)
    SendNUIMessage({
        action = 'withdrawResult',
        data = result
    })
end)

-- Bridge: NUI → Server → NUI (saldo)
RegisterNUICallback('casino:panel:getSaldo', function(data, cb)
    TriggerServerEvent('casino:panel:getSaldo')
    cb({}) -- cb obrigatorio
end)

RegisterNetEvent('casino:panel:saldoResult', function(result)
    SendNUIMessage({
        action = 'saldoResult',
        data = result
    })
end)

-- Bridge: NUI → Server → NUI (config)
RegisterNUICallback('casino:panel:getConfig', function(data, cb)
    TriggerServerEvent('casino:panel:getConfig')
    cb({}) -- cb obrigatorio
end)

RegisterNetEvent('casino:panel:configResult', function(result)
    SendNUIMessage({
        action = 'configResult',
        data = result
    })
end)

-- Cleanup ao sair do servidor
AddEventHandler('onResourceStop', function(resourceName)
    if GetCurrentResourceName() ~= resourceName then return end
    if isOpen then
        SetNuiFocus(false, false)
        isOpen = false
    end
end)
```

**Linhas:** ~105

**Por que bridge ao inves de NUI direto:** No FiveM, o NUI (React) NAO pode chamar server events diretamente. O fluxo e: NUI → RegisterNUICallback (Lua client) → TriggerServerEvent → Handler (JS server) → emitNet → RegisterNetEvent (Lua client) → SendNUIMessage → React. O Lua client e o PONTE entre os dois.

**cb({}) em CADA callback:** A pesquisa X0 confirmou que fetchNui trava se o Lua nao chamar `cb()`. Mesmo quando o resultado vem via SendNUIMessage posterior, o `cb({})` DEVE ser chamado para liberar o fetch.
