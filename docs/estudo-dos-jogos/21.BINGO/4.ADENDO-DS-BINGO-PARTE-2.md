# SEÇÃO 2 — ATMOSFERAS POR ESTADO (CSS + Partículas + Sons)

## 2.1 Estado IDLE / SELECTING (Comprando Cartelas — Aguardando GIRAR)

Atmosfera convidativa, quente, premium. Foco no card selector e ball machine decorativa:

- **Fundo**: bg-casino.png + vinheta inset (§1.1). Radial gradient dourado sutil no topo.
- **Ball Machine**: bolas movem lentamente, `machineSpeed.current = 0.3`. Turbulência mínima: `grav = 0.05`, `turb = 0.08`. Glow dourado sutil no wrapper (box-shadow padrão §1.2).
- **Partículas**: GoldParticles.tsx ativo com density baixa (~12 partículas), `speedY: -0.15`, cor `#D4A843` opacity 0.2-0.4. Atmosfera de cassino sem poluir visual.
- **Spotlight**: `machineSpotlightPulse` 3s, opacity 0.6→1.0. Dourado suave atrás da machine.
- **Card Selector**: slots com stagger animation (0.1s entre slots). Hover responsivo nos slots vazios.
- **Padrão**: mini-grid visível com label do padrão sorteado. Payout exibido.
- **Som**: `bingo_open` ao abrir a tela (500ms chime ascendente, volume 50%).
- **Interatividade**: todos slots clicáveis, botão TROCAR em slots preenchidos, CTA GIRAR ativo se ≥1 cartela.

## 2.2 Estado DRAWING (Sorteio Normal — Bola a Bola)

Atmosfera intensa, urgente, foco na ball machine e auto-dab nas cartelas:

- **Ball Machine**: turbulência INTENSA. `machineSpeed.current = 1.0`. `grav = 0.15`, `turb = 0.3`. Bolas quicando rápido dentro do container circular.
- **Machine Wrapper**: box-shadow intensifica (§1.2 `.active`). Spotlight pulse mais rápido (1.5s).
- **Partículas**: density dobra (24), `speedY: -0.3`, mix 70% dourado `#D4A843` + 30% verde `#00E676`.
- **Ball Display**: bolas aparecem uma a uma, 0.8s intervalo. Spring animation na entrada (`stiffness: 400, damping: 25`).
- **Cartelas**: read-only (`pointer-events: none`). Auto-dab com `dabPulse` 0.3s a cada match.
- **Card Selector**: desaparece. Cartelas expandem para ocupar espaço disponível.
- **Botão VOLTAR**: `opacity: 0.4`, `pointer-events: none` (§1.13 `.drawing`).
- **CTA GIRAR**: escondido (sorteio ativo, não faz sentido).
- **Counter bolas**: atualiza a cada reveal. Cor muda: `#A8A8A8` → `#FFD700` (> 15/20) → `#00E676` (20/20).
- **Som**: `ball_reveal` a cada bola (volume 60%), `ball_dab` nos matches (volume 70%), tick sutil de tensão crescente.

## 2.3 Estado LUCKY HIT (Momento Lucky Number — Pausa Celebratória)

Atmosfera dourada, celebração breve. Não pausa o sorteio, mas adiciona camada visual:

- **Bola Lucky**: sai da machine com glow dourado. Flash `luckyFlash` 0.3s no Ball Display (§1.9).
- **Célula da cartela**: transiciona para `.lucky-dabbed` (§1.3) com borda dourada 2px + box-shadow intenso.
- **Badge ×N**: spring scale 0→1 (§1.4), `stiffness: 300, damping: 20`. Permanece flutuando.
- **Lucky Total**: rolling update do multiplicador (§1.4 `.updated`, `multBounce` 0.3s).
- **Partículas**: burst micro de 5 sparkles dourados ao redor da bola Lucky (Canvas, life 300ms).
- **Spotlight**: flash dourado breve no fundo — NÃO overlay fullscreen, apenas intensificação do spotlight da machine.
- **Som**: `lucky_hit` — sparkle duplo 880Hz+1100Hz, 300ms, volume 80%.
- **Sorteio**: NÃO PAUSA. Próxima bola continua no timing normal de 0.8s.

## 2.4 Estado WILD BALL (Momento Cinematográfico — Pausa Real)

Atmosfera verde, dramática. PAUSA o sorteio por ~3.5s:

- **Machine**: DESACELERA gradualmente (`machineSpeed.current` 1.0→0.15 em 500ms). Wrapper transiciona para `.wild` (§1.2 — border verde, box-shadow verde).
- **Bola Wild**: sai da machine 25% maior, overshoot dramatic (Framer Motion `scale: [0.3, 2.0, 1.5]`).
- **Flash verde**: `.bingo-flash-green` (§1.16) — overlay sutil 500ms.
- **Partículas Wild**: burst de 10 partículas verdes `#00FF88` + `#00E676` ao redor da bola (Canvas, life 1s, gravity 0.1).
- **Banner "+X EXTRAS!"**: slide-in verde (Cinzel 700, `#00E676`, glow `text-shadow`, bg `rgba(0,230,118,0.08)`).
- **Cartelas**: auto-dab PAUSA durante o efeito Wild. Retoma após reveal das extras.
- **Partículas ambiente**: CONGELAM (freeze velocity, não deletar). Retomam quando sorteio continua.
- **Machine**: após pausa 1.5s, reativa (`machineSpeed.current` sobe 0.15→1.0). Border volta dourada.
- **Extra balls**: reveladas uma a uma com mesmo interval de 0.8s, COM auto-dab e check "1 To Go".
- **Som**: `wild_reveal` — sweep ascendente 500ms + impacto grave, volume 90%. `wild_extra` — ding metálico suave a cada extra (volume 60%).

## 2.5 Estado RESULTADO (Pós-Sorteio — Vitória / Derrota / Mega)

**Vitória normal (padrão completado, prêmio ≤ 10× aposta):**
- Overlay `.bingo-overlay.win` (§1.16) — backdrop blur 4px, fade-in 0.4s.
- `.bingo-result-card` glassmorphism slide-in spring (`stiffness: 200, damping: 20`).
- `.bingo-win-text` "BINGO!" verde #00E676 glow (§1.16).
- Rolling counter 0→prêmio em 2s (easeOutCubic).
- Confetti Canvas: pool 30, 2s life, cores `#FFD700 40%, #D4A843 30%, #00E676 20%, #FFFFFF 10%`.
- Cartela vencedora: todas cells dabbed pulsam glow verde 1s.
- Saldo: flash verde + rolling increment (§1.14 `.updating`).
- Som: `win_normal` — fanfare 1.5s, volume 80%.

**Derrota (nenhum padrão completado):**
- Overlay `.bingo-overlay.lose` (§1.16) — sem blur, tom sóbrio.
- `.bingo-result-card` com shake `x: [0, -3, 3, -2, 2, -1, 1, 0]` 0.4s.
- `.bingo-lose-text` "TENTE NOVAMENTE" dourado opacity 0.7 (§1.16).
- Subtítulo italic: "A sorte pode mudar na próxima rodada!" `#A8A8A8` opacity 0.5.
- Sem partículas. Tela levemente mais escura.
- Som: `lose` — descending tone, 500ms, volume 50% (respeitoso).

**Mega Win (prêmio > 10× aposta — combinação Lucky alta + padrão difícil):**
- `.bingo-flash-white` PRIMEIRO (§1.16) — 200ms, opacity 0.3→0.
- Screen shake: container inteiro `x: [-5, 5, -4, 4, -2, 2, 0]` 500ms.
- Overlay `.bingo-overlay.mega` — blur 6px, mais escuro.
- `.bingo-win-text.mega` dourado 56px pulsante `megaPulse` (§1.16).
- Rolling counter DOURADO (não verde) 3s.
- Confetti INTENSO: pool 50, 3s life, velocidade 2-4 (vs 1-2 normal), size 4-8px.
- Ambient glow: `radial-gradient(circle, rgba(255,215,0,0.08), transparent 70%)` sobre tudo.
- Som: `win_mega` — fanfare extendida 3s + sirene metálica, volume 100%.

# SEÇÃO 3 — TIMELINES EXATAS

## 3.1 Timeline Normal Ball Reveal (1 de 20 — 0.8s cycle)

```
t=0ms       setInterval tick. currentBallIndex++.
t=0ms       Bola removida do array balls[] no Canvas machine.
            (Canvas exclui body da pool visual no próximo frame)
t=0-50ms    Elemento DOM criado no Ball Display (§1.8/1.9/1.10)
            Framer Motion: initial={{ scale: 0, opacity: 0 }}
            animate={{ scale: 1, opacity: 1 }}
            transition={{ type: "spring", stiffness: 400, damping: 25 }}
t=50ms      CHECK: ball ∈ luckyNumbers?
            SE SIM → §3.2 Lucky Hit (NÃO pausa ciclo)
t=50ms      CHECK: currentBallIndex === wildBall.index?
            SE SIM → clearInterval, §3.3 Wild Ball Phase
t=50-100ms  Auto-dab: para CADA cartela que contém o número:
            - Célula transiciona idle → .dabbed (§1.3)
            - dabPulse 0.3s
            - Som: ball_dab (volume 70%)
t=100-200ms CHECK "1 To Go": para cada cartela:
            - Simula dab de cada célula não marcada
            - Se alguma completaria padrão → .one-to-go (§1.3)
            - Se já existia 1ToGo e esse número o completa → BINGO!
t=200-800ms Pausa visual. Machine continua agitando.
t=800ms     Próxima bola (volta a t=0ms)

TOTAL 20 BOLAS NORMAIS (sem Wild): 20 × 800ms = 16s
COM Wild (+1.5s pausa + extras): ~22s total estimado
```

## 3.2 Timeline Lucky Number Hit (overlay no ciclo normal)

```
t=0ms       Ball identificada como Lucky Number.

t=0-50ms    Em CADA cartela que contém o número:
            - Célula transiciona para .lucky-dabbed (§1.3)

            - Rolling animation de old → new multiplicador

t=300ms     Sparkles terminam. Efeito visual permanece (badge + célula).

NOTA: O ciclo normal NÃO pausa. Bola seguinte aparece em t=800ms normalmente.

```

## 3.3 Timeline Wild Ball Phase (pausa real de ~3.5s + extras)

```
t=0ms       currentBallIndex === wildBall.index (detectado).
            clearInterval — sorteio PAUSA.
t=0ms       Machine começa desacelerar:
            machineSpeed.current diminui 1.0 → 0.15 via rAF (500ms)
t=0ms       Machine wrapper transiciona para .wild (§1.2)
            Border e box-shadow mudam para verde em 500ms.
t=500ms     Machine quase parada. Bola Wild sai.
t=500ms     Bola Wild DOM criada: .bingo-ball.wild (§1.10)
            Framer Motion overshoot:
            scale: [0.3, 2.0, 1.5], duration: 0.6
            ease: [0.34, 1.56, 0.64, 1]
t=500ms     Flash verde: .bingo-flash-green (§1.16) — 500ms fade.
t=500ms     Partículas verdes: 10 burst (Canvas), life 1s.
t=500ms     Som: wild_reveal — sweep + impacto, volume 90%.
t=800ms     Auto-dab da Wild Ball em cartelas (se match). dabPulse.
t=800ms     Check 1ToGo para cada cartela.
t=1100ms    Banner "+X BOLAS EXTRAS!" slide-in:
            Framer Motion y: -20→0, opacity: 0→1, duration 0.4
t=1100ms    Machine wrapper volta estado .active (border dourada)
            machineSpeed.current sobe 0.15 → 1.0
t=1500ms    Início reveal das extra balls (mesmo interval 800ms):
            Extra 1: t=1500ms (auto-dab + 1ToGo check)
            Extra 2: t=2300ms (se 2 extras)
            Extra 3: t=3100ms (se 3 extras)
t=3100ms    Extras terminadas. resumeMainDraw() — retoma intervalo normal.
t=3500ms    Sorteio normal continua de onde parou.

TOTAL WILD PHASE: ~3.5s (com 2 extras)
```

## 3.4 Timeline "1 To Go" Detection + Banner

```
t=0ms       Auto-dab de uma bola acabou de executar.
t=0-10ms    Para CADA cartela, executar check1ToGo(card, pattern):
            - Mapear grid 5×5 em boolean[][] (true = dabbed ou FREE)
            - Para cada cell NÃO dabbed:
              - Simular dab temporário
              - Executar checkPattern(grid, pattern)
              - Se TRUE: essa cell completaria o padrão!
              - Registrar: set1ToGo(cardId, { row, col, number })
              - Reverter simulação
              - BREAK (1 cell por cartela)
            - Se nenhuma cell completaria: remove1ToGo(cardId)

t=10ms      SE 1ToGo detectado pela primeira vez para esta cartela:
t=10ms      Célula ganha classe .one-to-go (§1.3)
            pulse1ToGo 1s infinite ativa.
t=10ms      Cartela ganha classe .one-to-go (§1.6)
            Border dourada, box-shadow, scale 1.02.
t=10ms      Banner .bingo-one-to-go-banner (§1.16) slide-in:
            AnimatePresence y: -20→0, opacity: 0→1, height: 0→auto
            Texto: "⚡ CARTELA X: FALTA 1! ⚡"
            Subtítulo: "Número {N} completaria {PATTERN}!"
t=10ms      Som: one_to_go — tension build 400→600Hz, 500ms, volume 70%.

            SE número seguinte É o faltante:
t=800ms     Auto-dab completa padrão → BINGO!
            Banner sai: exit y: -20, opacity: 0, height: 0
            Resultado ativa imediatamente.

            SE número seguinte NÃO é o faltante:
            Banner permanece. Célula continua pulsando.
            Próximas bolas checam novamente.
            Se novo match muda o "1 To Go" cell → atualizar banner.

NOTA: Múltiplas cartelas podem ter "1 To Go" simultâneo.
Banner mostra a de menor match count (mais próxima de ganhar).
```
