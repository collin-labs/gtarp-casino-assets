## 7.2 Segurança / Anti-Cheat (10 Itens)

| # | Aspecto | Implementação Concreta |
|---|---------|----------------------|
| 1 | **Resultado server-only** | HMAC_SHA256 via Node.js `crypto`. Client NUNCA gera números. Server gera cartelas, sorteio, Lucky, Wild, padrão — tudo determinístico via seed. Client apenas recebe e anima. |
| 2 | **Validação de aposta (5 checks)** | (1) numCards 1-4, (2) bet ≥ cardPrice, (3) saldo ≥ bet × numCards, (4) dailyRounds < maxRoundsPerDay, (5) cooldown respeitado. Cada check com mensagem de erro específica. |
| 3 | **Rate limit server-side** | `Map<playerId, timestamp>`. Cooldown 5000ms (config). Client não burla — server rejeita com "Aguarde X segundos". Botão GIRAR desabilitado com countdown visual. |
| 4 | **Daily limit** | Tabela `casino_bingo_daily` com UNIQUE(player_id, play_date). `INSERT ... ON DUPLICATE KEY UPDATE rounds_used = rounds_used + 1`. Atômico. Default 100 rodadas/dia. |
| 5 | **Provably Fair** | Hash SHA256(server_seed) publicado ANTES. Seed revelado DEPOIS. Jogador verifica client-side via Web Crypto API. Código Node.js completo (120 linhas) no bingo_provably_fair.js. Separação de domínios (:cards, :draw, :lucky, :wild, :pattern). |
| 6 | **Max payout cap** | Default 100.000 GCoin. `Math.min(calculado, config.maxPayout)`. Admin ajusta via `/bingoadmin config max_payout 200000`. Protege economia do servidor. |
| 7 | **Lucky Numbers controlados** | Multiplicadores: 2x(50%), 3x(35%), 5x(15%). Combinação máxima teórica: 5^5=3.125x — mas maxPayout cap corta. Na prática, multiplicador médio ~2-4x. |
| 8 | **Wild Ball controlado** | 20% chance (config). 60% gera apenas 1 extra ball. Impacto econômico moderado e previsível. Admin pode desativar: `wild_ball_chance = 0`. |
| 9 | **Nonce incremental** | Cada rodada incrementa nonce per-player. Impede replay attacks — mesmos seeds com nonce diferente = resultado diferente. Server tracked. |
| 10 | **Audit log completo** | TODA ação em `casino_bingo_audit`: PLAY (cartelas, padrão, resultado, prêmio), HIGH_WIN (prêmio > 10× aposta), CONFIG_CHANGE (admin, key, old→new), ERROR. JSON `details` flexível. Append-only, indexes otimizados. |
