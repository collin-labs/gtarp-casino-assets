## RELATORIO DA PESQUISA (REGRA X0)

**Rodada 1 — MySQL atomic UPDATE + oxmysql FiveM:**
- Query: "MySQL UPDATE WHERE conditional atomic lock oxmysql FiveM 2025"
- Fontes: oxmysql.com, github.com/overextended/oxmysql, coxdocs.dev/oxmysql, dev.mysql.com/doc/refman/8.4
- Descobertas: (1) OxMySQL suporta transactions via `MySQL.startTransaction()` com rollback automatico em caso de erro. (2) `UPDATE ... WHERE status='waiting' AND player2_id IS NULL` e atomico em InnoDB — row-level lock garante que apenas 1 jogador entra. (3) `affectedRows` retorna 0 se nenhuma row atendeu a condicao (room ja preenchida). (4) Prepared statements com `?` positional obrigatorios para prevenir SQL injection. (5) OxMySQL foi arquivado em Abr/2025 mas continua funcional e e o padrao para FiveM.
- Impacto: Confirma padrao do Mega Estudo (Problema 1). O UPDATE condicional e suficiente para lock atomico — NAO precisa de transaction explicita para o join. Transaction so se precisar de rollback multi-query (ex: debitar saldo + entrar na sala como operacao atomica).

**Rodada 2 — CSS perspective rotateY backfaceVisibility no CEF FiveM:**
- Query: "CSS perspective rotateY backfaceVisibility Chromium CEF FiveM NUI 3D flip coin 2025"
- Fontes: developer.mozilla.org/backface-visibility, css-tricks.com/almanac/backface-visibility, dev.to (3D coin deep dive), medium.com (3D coin flip CSS)
- Descobertas: (1) `perspective: 1000px` + `transform-style: preserve-3d` + `backface-visibility: hidden` e suportado em 98%+ dos browsers incluindo Chromium CEF. (2) Padrao comprovado para coin flip: container com perspective, inner com rotateY, 2 faces absolutas com backfaceVisibility hidden, face traseira com rotateY(180deg) inicial. (3) `-webkit-backface-visibility: hidden` OBRIGATORIO como fallback no CEF (prefixo webkit). (4) Edge stacking (60 divs com translateZ) cria ilusao de espessura — DESCARTADO por performance no NUI (muitos DOM elements). Shadow eliptica basta.
- Impacto: Confirma decisao da Pesquisa REGRA0. Tecnica funciona no CEF sem adaptacoes. Adicionar `-webkit-` prefix no CSS.

**Rodada 3 — HMAC-SHA256 PvP com 2 client seeds:**
- Query: "HMAC SHA256 provably fair PvP two client seeds coinflip gambling 2025"
- Fontes: stake.com/provably-fair/implementation, github.com/nucleare/DeepSearch_HashGames, btcgosu.com/provably-fair, gamingtec.com/provably-fair-explained, casinosblockchain.io
- Descobertas: (1) Padrao Stake: `HMAC_SHA256(serverSeed, clientSeed:nonce:cursor)`. Para coinflip PvP, adaptar: `HMAC_SHA256(serverSeed, clientSeedP1:clientSeedP2:nonce)`. (2) Coinflip precisa de apenas 1 byte (0-255) para resultado — NAO precisa de cursor (cursor e para jogos que precisam de 8+ outcomes). (3) `createHmac` do Node.js NAO e reutilizavel apos `.digest()` — criar novo objeto por operacao. (4) RustyPot (8+ anos operando) usa SHA256 com Random.org para coinflip PvP — confirma viabilidade do modelo. (5) Nonce = ID da partida (auto-increment) — simplicidade maxima.
- Impacto: Confirma decisao do Mega Estudo (Problema 5). Simplifica: nonce = room.id, sem cursor, 1 byte basta.

**O que teriamos feito sem a pesquisa:**
- Transaction explicita para o join em vez de UPDATE condicional (mais lento, mais complexo)
- Sem `-webkit-backface-visibility` no CSS (quebra no CEF de alguns servidores)
- Cursor desnecessario no HMAC (complexidade extra sem beneficio para coinflip)

-- Listar salas ativas (lobby)
SELECT id, player1_id, player1_side, bet_amount, created_at
FROM casino_coinflip_games
WHERE status = 'waiting'
ORDER BY created_at DESC;

-- Revenue por periodo (rake acumulado)
SELECT DATE(completed_at) AS dia,
       SUM(rake_amount) AS rake_total,
       COUNT(*) AS partidas,
       SUM(pot_total) AS volume_total
FROM casino_coinflip_games
WHERE status = 'completed'
  AND completed_at >= DATE_SUB(NOW(), INTERVAL 30 DAY)
GROUP BY DATE(completed_at)
ORDER BY dia DESC;

-- Leaderboard (top 20 por profit)
SELECT player_id,
       COUNT(*) AS partidas,
       SUM(CASE WHEN winner_id = player_id THEN 1 ELSE 0 END) AS wins,
       SUM(CASE WHEN winner_id = player_id THEN payout_amount - bet_amount ELSE -bet_amount END) AS profit
FROM (
  SELECT player1_id AS player_id, winner_id, payout_amount, bet_amount
  FROM casino_coinflip_games WHERE status = 'completed'
  UNION ALL
  SELECT player2_id, winner_id, payout_amount, bet_amount
  FROM casino_coinflip_games WHERE status = 'completed'
) t
GROUP BY player_id
ORDER BY profit DESC
LIMIT 20;

-- Verificar fairness (distribuicao cara vs coroa)
SELECT result,
       COUNT(*) AS total,
       ROUND(COUNT(*) * 100.0 / (SELECT COUNT(*) FROM casino_coinflip_games WHERE status = 'completed'), 2) AS pct
FROM casino_coinflip_games
WHERE status = 'completed'
GROUP BY result;

## 2.1 Criar arquivo `server/handlers/coinflip_engine.js`

**Arquivo NOVO — codigo completo executavel:**

```javascript
// ====================================================
// BLACKOUT CASINO — COINFLIP ENGINE (#12)
// Provably Fair PvP + Resultado
// Versao 1.0 - 20/03/2026
// ====================================================

const crypto = require('crypto');

// ---- PROVABLY FAIR ----

/**
 * Gera server seed aleatorio (32 bytes hex)
 * @returns {string} 64 caracteres hex
 */
function generateServerSeed() {
  return crypto.randomBytes(32).toString('hex');
}

/**
 * Calcula SHA-256 de um server seed (hash publicado ANTES do flip)
 * @param {string} serverSeed
 * @returns {string} 64 caracteres hex
 */
function hashServerSeed(serverSeed) {
  return crypto.createHash('sha256').update(serverSeed).digest('hex');
}

/**
 * Calcula resultado Provably Fair PvP
 * Formula: HMAC-SHA256(serverSeed, clientSeedP1:clientSeedP2:nonce)
 * Primeiro byte < 128 = 'cara', >= 128 = 'coroa' (exatamente 50/50)
 *
 * @param {string} serverSeed - Seed secreto do servidor
 * @param {string} clientSeedP1 - Seed do jogador 1 (criador)
 * @param {string} clientSeedP2 - Seed do jogador 2 (oponente)
 * @param {number} nonce - ID da partida (auto-increment)
 * @returns {{ result: 'cara'|'coroa', resultByte: number, hmacHex: string }}
 */
function calculateResult(serverSeed, clientSeedP1, clientSeedP2, nonce) {
  const combined = `${clientSeedP1}:${clientSeedP2}:${nonce}`;
  const hmac = crypto.createHmac('sha256', serverSeed);
  hmac.update(combined);
  const hmacHex = hmac.digest('hex');

  // Primeiro byte do HMAC (0-255)
  const resultByte = parseInt(hmacHex.substring(0, 2), 16);

  // 0-127 = cara (128 valores), 128-255 = coroa (128 valores) = exatamente 50/50
  const result = resultByte < 128 ? 'cara' : 'coroa';

  return { result, resultByte, hmacHex };
}

/**
 * Verifica se um resultado e valido (para modal Provably Fair)
 * @param {string} serverSeed
 * @param {string} serverSeedHash
 * @param {string} clientSeedP1
 * @param {string} clientSeedP2
 * @param {number} nonce
 * @param {string} claimedResult - 'cara' ou 'coroa'
 * @returns {{ valid: boolean, calculatedResult: string, hashMatch: boolean }}
 */
function verifyResult(serverSeed, serverSeedHash, clientSeedP1, clientSeedP2, nonce, claimedResult) {
  // Verificar que o hash do server seed bate com o publicado
  const calculatedHash = hashServerSeed(serverSeed);
  const hashMatch = crypto.timingSafeEqual(
    Buffer.from(calculatedHash, 'hex'),
    Buffer.from(serverSeedHash, 'hex')
  );

  // Recalcular resultado
  const { result: calculatedResult } = calculateResult(serverSeed, clientSeedP1, clientSeedP2, nonce);

  return {
    valid: hashMatch && calculatedResult === claimedResult,
    calculatedResult,
    hashMatch,
  };
}

// ---- ECONOMIA ----

/**
 * Calcula payout e rake de uma partida
 * @param {number} betAmount - Valor apostado por CADA jogador
 * @param {number} rakePercent - Taxa da casa (ex: 2.00)
 * @returns {{ potTotal: number, rakeAmount: number, payoutAmount: number }}
 */
function calculatePayout(betAmount, rakePercent) {
  const potTotal = betAmount * 2;
  const rakeAmount = Math.floor(potTotal * rakePercent / 100);
  const payoutAmount = potTotal - rakeAmount;
  return { potTotal, rakeAmount, payoutAmount };
}

/**
 * Determina o vencedor com base no resultado e nos lados escolhidos
 * @param {string} result - 'cara' ou 'coroa'
 * @param {string} player1Side - Lado do jogador 1
 * @param {string} player1Id - ID do jogador 1
 * @param {string} player2Id - ID do jogador 2
 * @returns {string} ID do vencedor
 */
function determineWinner(result, player1Side, player1Id, player2Id) {
  return result === player1Side ? player1Id : player2Id;
}

// ---- VALIDACOES ----

/**
 * Valida valor da aposta contra config
 * @param {number} amount
 * @param {object} config - { min_bet, max_bet }
 * @returns {{ valid: boolean, error: string|null }}
 */
function validateBetAmount(amount, config) {
  if (!Number.isInteger(amount) || amount <= 0) {
    return { valid: false, error: 'valor_invalido' };
  }
  if (amount < config.min_bet) {
    return { valid: false, error: 'abaixo_minimo' };
  }
  if (amount > config.max_bet) {
    return { valid: false, error: 'acima_maximo' };
  }
  return { valid: true, error: null };
}

/**
 * Valida lado escolhido
 * @param {string} side
 * @returns {boolean}
 */
function validateSide(side) {
  return side === 'cara' || side === 'coroa';
}

/**
 * Retorna lado oposto
 * @param {string} side - 'cara' ou 'coroa'
 * @returns {string}
 */
function oppositeSide(side) {
  return side === 'cara' ? 'coroa' : 'cara';
}

// ---- EXPORTS ----

module.exports = {
  generateServerSeed,
  hashServerSeed,
  calculateResult,
  verifyResult,
  calculatePayout,
  determineWinner,
  validateBetAmount,
  validateSide,
  oppositeSide,
};
```
