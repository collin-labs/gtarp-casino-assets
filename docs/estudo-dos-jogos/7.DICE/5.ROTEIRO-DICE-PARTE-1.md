## RELATORIO DA PESQUISA (REGRA X0)

**Pesquisas realizadas para o Roteiro:**
1. `FiveM oxmysql query async setImmediate emitNet callback pattern server handler 2025 2026` → 10 resultados
2. `Node.js crypto createHmac SHA256 dice roll result provably fair 4 bytes float conversion 2025` → 10 resultados

**Fontes relevantes:**
- OxMySQL Docs (coxdocs.dev/oxmysql): Prepared statements com `?` positional params. Async por default. Import via `@communityox/oxmysql` npm ou exports direto. `exports.oxmysql:query()` e `exports.oxmysql:single()` para rows. Sem necessidade de mysql-async
- FiveM OxMySQL Migration Guide (fivemx.com): Swap `@param` para `?` positional. Promise/await API. Suporta ESX, QBCore, custom frameworks. JSON column type suportado (MySQL 5.7+ / MariaDB 10.2+)
- Node.js v25.8.1 Crypto Docs (nodejs.org/api/crypto.html): `createHmac('sha256', secret).update(data).digest()` retorna Buffer de 32 bytes. Hmac NAO reutilizavel apos `.digest()` — precisa criar novo por operacao. `.digest('hex')` para string hex, `.digest()` para Buffer raw
- W3Schools Node Crypto: `timingSafeEqual(Buffer.from(a), Buffer.from(b))` OBRIGATORIO para comparacao de hashes — previne timing attacks. Ambos buffers devem ter mesmo tamanho
- Fireship.io Node Crypto Tutorial: Pattern `createHmac('sha256', key).update(message).digest('hex')` — one-liner funcional. Key pode ser string ou Buffer
- Stake.com Provably Fair (pesquisa anterior): `HMAC-SHA256(serverSeed, clientSeed:nonce:cursor)`. Para Dice: cursor=0 (apenas 12 bytes necessarios, 32 bytes do HMAC sobram). 4 bytes por dado: `buffer[0]/256 + buffer[1]/65536 + buffer[2]/16777216 + buffer[3]/4294967296` = float 0-1. `Math.floor(float * 6) + 1` = face 1-6

**O que a pesquisa trouxe de novo para o Roteiro:**
1. **OxMySQL usa `?` positional** — NAO `@param`. Queries do projeto existente (slots.js, cases.js) ja seguem este padrao
2. **Hmac NAO reutilizavel** — cada dado precisa de novo createHmac OU usar buffer unico e fatiar (mais eficiente: 1 HMAC = 32 bytes = 8 floats, 3 dados usam apenas 12 bytes)
3. **`timingSafeEqual` obrigatorio** para verificacao de hash — sem isso, timing attack pode inferir bytes
4. **JSON column** funcional no MariaDB 10.2+ (servidor GTARP usa MariaDB) — campos `bets` e `lightning_numbers` podem ser JSON nativo

**O que teriamos feito sem a pesquisa:**
- Usado `@param` nos queries → erro de sintaxe no oxmysql
- Criado 3 createHmac separados (1 por dado) em vez de fatiar 1 buffer → 3x mais lento
- Comparado hashes com `===` em vez de `timingSafeEqual` → vulneravel a timing attack
- Usado TEXT para campos JSON → sem validacao nativa do banco

-- ====================================================
-- BLACKOUT CASINO — DICE ENGINE (#7) — Schema SQL

-- ----------------------------------------------------
-- 1. CONFIG — Uma row com todos os parametros do jogo
-- Admin edita via UPDATE. Handler le no boot e cacheia.
-- ----------------------------------------------------
CREATE TABLE IF NOT EXISTS casino_dice_config (
  id INT AUTO_INCREMENT PRIMARY KEY,
  rtp_target DECIMAL(5,2) NOT NULL DEFAULT 96.00
    COMMENT 'RTP alvo em %. Admin ajusta entre 92-98. Multiplicadores calibrados para atingir este RTP',
  min_bet DECIMAL(15,2) NOT NULL DEFAULT 10.00
    COMMENT 'Aposta minima em GCoin. Impede micro-apostas que congestionam o server',
  max_bet DECIMAL(15,2) NOT NULL DEFAULT 5000.00
    COMMENT 'Aposta maxima em GCoin. Limita exposicao financeira do servidor',
  max_payout DECIMAL(15,2) NOT NULL DEFAULT 500000.00
    COMMENT 'Payout maximo por rodada. Se multiplicador x aposta > este valor, limitar. Protecao contra big wins extremos',
  lightning_count_min TINYINT UNSIGNED NOT NULL DEFAULT 2
    COMMENT 'Minimo de totais que recebem multiplicador por rodada (2-4). Menos = mais raro = mais tension',
  lightning_count_max TINYINT UNSIGNED NOT NULL DEFAULT 4
    COMMENT 'Maximo de totais com multiplicador. Mais = mais chances mas multiplicadores menores',
  max_multiplier INT UNSIGNED NOT NULL DEFAULT 1000
    COMMENT 'Multiplicador maximo permitido. Limite superior para totais 3/18. Padrao Evolution: 1000x',
  round_duration_ms INT UNSIGNED NOT NULL DEFAULT 30000
    COMMENT 'Duracao nominal da rodada em ms. Usado no auto-bet para spacing entre rodadas',
  auto_bet_max_rounds INT UNSIGNED NOT NULL DEFAULT 50
    COMMENT 'Maximo de rodadas em auto-bet. Limita sessoes automaticas para protecao do jogador',
  rate_limit_ms INT UNSIGNED NOT NULL DEFAULT 2000
    COMMENT 'Intervalo minimo entre apostas em ms. Anti-spam: impede flooding de requests',
  is_active TINYINT(1) NOT NULL DEFAULT 1
    COMMENT 'Jogo ativo? 0 = desativado pelo admin (manutencao). Handler retorna erro se 0',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
  COMMENT='Configuracao do jogo Dice — 1 row editavel pelo admin';

CREATE TABLE IF NOT EXISTS casino_dice_history (
  id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  player_id VARCHAR(50) NOT NULL
    COMMENT 'Identificador FiveM: license:xxx ou vistoId',
  dice_1 TINYINT UNSIGNED NOT NULL
    COMMENT 'Valor do dado 1 (1-6). Gerado via Provably Fair HMAC-SHA256',
  dice_2 TINYINT UNSIGNED NOT NULL
    COMMENT 'Valor do dado 2 (1-6)',
  dice_3 TINYINT UNSIGNED NOT NULL
    COMMENT 'Valor do dado 3 (1-6)',
  total TINYINT UNSIGNED NOT NULL
    COMMENT 'Soma dos 3 dados (3-18). Calculado server-side, nunca pelo client',
  bets JSON NOT NULL
    COMMENT 'Array de apostas: [{"type":"total","value":7,"amount":100},{"type":"low","amount":50}]',
  lightning_numbers JSON NOT NULL
    COMMENT 'Totais com multiplicador: [{"total":3,"multiplier":500},{"total":11,"multiplier":50}]',
  total_bet DECIMAL(15,2) NOT NULL
    COMMENT 'Soma de todas as apostas da rodada. Debitado ANTES do resultado',
  total_win DECIMAL(15,2) NOT NULL DEFAULT 0.00
    COMMENT 'Soma de todos os ganhos. Creditado APOS resultado. 0 se perdeu tudo',
  profit DECIMAL(15,2) GENERATED ALWAYS AS (total_win - total_bet) STORED
    COMMENT 'Lucro/prejuizo do jogador. Positivo = jogador ganhou. Coluna GERADA automaticamente',
  server_seed VARCHAR(64) NOT NULL
    COMMENT 'Server seed usado nesta rodada. Revelado ao jogador apos rotacao de seed',
  client_seed VARCHAR(64) NOT NULL

  nonce BIGINT UNSIGNED NOT NULL

  seed_hash VARCHAR(64) NOT NULL

  is_auto_bet TINYINT(1) NOT NULL DEFAULT 0
    COMMENT '0=manual, 1=auto-bet. Para estatisticas de uso de auto-bet',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    COMMENT 'Quando a rodada aconteceu. UTC',
  INDEX idx_player (player_id)
    COMMENT 'Busca por historico do jogador',
  INDEX idx_created (created_at)
    COMMENT 'Busca por periodo (admin dashboard)',
  INDEX idx_player_created (player_id, created_at)
    COMMENT 'Historico do jogador por periodo (query mais comum)',
  INDEX idx_total (total)
    COMMENT 'Estatisticas: frequencia de cada total (verificar RNG equilibrado)'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
  COMMENT='Historico de jogadas Dice — 1 row por rodada jogada';

-- ----------------------------------------------------
-- 3. SEEDS — Provably Fair por jogador
-- 1 row por jogador ativo. Mantem seed atual e anterior.
-- Seed anterior revelado ao jogador para verificacao.
-- ----------------------------------------------------
CREATE TABLE IF NOT EXISTS casino_dice_seeds (
  player_id VARCHAR(50) PRIMARY KEY
    COMMENT 'Identificador FiveM. PK = 1 seed pair por jogador',
  server_seed VARCHAR(64) NOT NULL
    COMMENT 'Server seed ATUAL. 64 chars hex (crypto.randomBytes(32).toString(hex)). SECRETO ate rotacao',
  server_seed_hash VARCHAR(64) NOT NULL
    COMMENT 'SHA-256(server_seed). Visivel ao jogador ANTES de qualquer aposta. Prova que seed existia',
  client_seed VARCHAR(64) NOT NULL DEFAULT 'default'
    COMMENT 'Client seed definido pelo jogador. Default: string "default" ate jogador customizar',
  nonce BIGINT UNSIGNED NOT NULL DEFAULT 0
    COMMENT 'Contador de apostas com este par de seeds. +1 a cada bet. Garante resultados unicos',
  previous_server_seed VARCHAR(64) DEFAULT NULL
    COMMENT 'Server seed ANTERIOR (revelado apos rotacao). Jogador usa para verificar rodadas passadas',
  previous_seed_hash VARCHAR(64) DEFAULT NULL
    COMMENT 'Hash do seed anterior. Jogador compara: SHA-256(previous_server_seed) == previous_seed_hash?',
  previous_nonce BIGINT UNSIGNED DEFAULT NULL
    COMMENT 'Ultimo nonce antes da rotacao. Jogador sabe quantas rodadas foram jogadas com o seed anterior',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    COMMENT 'Quando o seed pair foi criado',
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
    COMMENT 'Ultima atualizacao (rotacao de seed ou mudanca de client seed)'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
  COMMENT='Seeds Provably Fair Dice — 1 row por jogador ativo. Seed anterior guardado para verificacao';

-- ----------------------------------------------------
-- 4. PAYOUTS — Tabela de referencia de payouts base
-- Usada pelo handler para lookup rapido. Admin pode ajustar.

  total TINYINT UNSIGNED PRIMARY KEY
    COMMENT 'Total dos 3 dados (3-18)',
  combinations SMALLINT UNSIGNED NOT NULL
    COMMENT 'Quantas combinacoes de 3d6 geram este total (de 216 possiveis)',
  probability DECIMAL(8,5) NOT NULL
    COMMENT 'Probabilidade em % (combinations / 216 * 100)',
  payout_base DECIMAL(8,2) NOT NULL
    COMMENT 'Payout base sem multiplicador (ex: 150.00 para total 3)',
  lightning_min INT UNSIGNED NOT NULL
    COMMENT 'Multiplicador lightning minimo para este total',
  lightning_max INT UNSIGNED NOT NULL
    COMMENT 'Multiplicador lightning maximo para este total',
  lightning_weight TINYINT UNSIGNED NOT NULL DEFAULT 5
    COMMENT 'Peso de selecao para lightning (maior = mais chance de ser escolhido). Totais raros tem peso maior'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4
  COMMENT='Payouts base e ranges de multiplicador por total. Admin pode ajustar payouts';
