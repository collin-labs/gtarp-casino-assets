```

```

## 10.2 Cálculo de Sustentabilidade Econômica

```
CENÁRIO: Servidor com 200 jogadores simultâneos, 500 únicos/dia

RECEITA DO CASINO (GCoins drenados/dia):
├── 22 jogos × house edge 3% médio × volume 100K/jogo    = 66.000 GCoins/dia
├── Marketplace taxa 7.5% × volume 50K                    = 3.750 GCoins/dia
├── Cases: 50 aberturas/dia × 3.000 GCoins × edge 15%     = 22.500 GCoins/dia
├── Instant Sell: 20 vendas/dia × 2.000 avg × 35% spread  = 14.000 GCoins/dia
└── TOTAL SINKS:                                           ≈ 106.250 GCoins/dia

EMISSÃO (GCoins injetados/dia):
├── Jobs: 200 jogadores × 500 avg salário                  = 100.000 GCoins/dia
├── Daily Free: 200 × 100 GCoins                           = 20.000 GCoins/dia
├── Giveaways: 1/dia × 5.000                               = 5.000 GCoins/dia
└── TOTAL FAUCETS:                                         ≈ 125.000 GCoins/dia

BALANÇO: 125.000 - 106.250 = +18.750 GCoins/dia (inflação leve e controlável)
AJUSTE: Aumentar house edge de 2 jogos populares em 0.5% = -10.000/dia → equilíbrio
```

**O Marketplace com taxa é ESSENCIAL para o equilíbrio — sem ele, o gap seria +40.000/dia (inflação perigosa).**

# PARTE 12 — ANTI-EXPLOIT DETALHADO
# ═══════════════════════════════════════════════════

| # | Exploit | Como funciona | Prevenção |
|---|---------|--------------|-----------|
| 1 | **Price manipulation** | Jogador A lista item por G$ 1, jogador B (alt account) compra → transfere item de graça | Min price = 10 GCoins. Admin alerta se preço <20% da média |
| 2 | **Wash trading** | A vende para B, B vende para A, repetidamente → inflacionar preço | Rate limit: mesmo par buyer-seller max 3 transações/dia do mesmo item_type |
| 3 | **Bot sniping** | Script automático compra PECHINCHAs antes de jogadores reais | Rate limit: max 5 compras/minuto. CAPTCHA se detectar padrão |
| 4 | **Inventory duplication** | Exploit de timing: listar item + equipar simultaneamente | Server-side: check status ANTES de qualquer operação. Status AVAILABLE é pré-req para LISTED e EQUIPPED |
| 5 | **Fee evasion** | Trade direto entre jogadores fora do marketplace | Trade futuro (v2) terá taxa também. Sem trade nativo no v1 |
| 6 | **Alt account farming** | Criar alts para receber Daily Free e vender no marketplace | Min level = 5 para usar marketplace. Flag contas com mesmo IP |
| 7 | **Price oracle manipulation** | Listar items a preço absurdo para inflar a "média" | Price suggestion ignora outliers: remove top/bottom 10% antes de calcular média |
| 8 | **Spam listings** | Listar 50 items de 1 GCoin para poluir o grid | Max listings (10 default, 50 VIP). Min price = 10. Admin pode remover em batch |

# PARTE 14 — PERGUNTAS COMPLEMENTARES
# ═══════════════════════════════════════════════════

11. **Upgrade (#22) no marketplace?** O jogo "Upgrade" melhora o tier de um item (Common→Uncommon, etc.). Item upgraded pode ser vendido no marketplace com o NOVO tier? Se sim, o origin muda para "UPGRADE" e o valor de mercado sobe? Recomendação: sim, é um fluxo poderoso (jogar → dropar → upgrade → vender por mais).

12. **Itens "soulbound" (intransferíveis)?** Alguns badges de achievement poderiam ser soulbound — não podem ser vendidos. Ex: "Badge 1 Ano no Servidor". Campo `tradeable=0` no item_types já cobre isso. Quais itens devem ser soulbound?

13. **Marketplace aberto 24h ou horários?** Alguns servidores RP fecham o marketplace à noite para "imersão". Recomendação: 24h — é digital, não faz sentido ter horário.

14. **Taxa fixa ou percentual?** Atualmente é percentual (7.5%). Alternativa: taxa fixa (ex: G$ 500 por transação). Percentual é melhor porque escala com o valor e é mais justo para items baratos/caros. Manter percentual.
