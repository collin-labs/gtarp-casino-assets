# PROBLEMAS QUE TU ENFRENTOU E COMO RESOLVER
# ═══════════════════════════════════════════════════

Tu disse que o marketplace ficou: "básico, sem muita interação, filtro, precisão, informação, muito vago, conexão correta."

Cada problema tem solução concreta baseada na pesquisa de UX:

### Problema 1: "Básico"
**Causa:** Provavelmente uma lista simples de itens sem visual premium.
**Solução:** Product cards com: imagem PNG do item (do DS do casino), nome, preço em GCoins, raridade (badge colorido), vendedor (avatar + nome + rating), tempo listado, botão "COMPRAR" com feedback visual. Referência: DMarket, CSFloat — cards limpos mas informativos.

### Problema 2: "Sem muita interação"
**Causa:** Tela estática sem hover, sem animação, sem feedback.
**Solução:** Hover com glow por raridade (Common cinza, Rare azul, Epic roxo, Legendary dourado), toast notifications em tempo real ("Novo item listado!", "Seu item vendeu!"), animação de compra (GCoins saindo → item entrando no inventário), websocket ou polling para atualizações live.

### Problema 3: "Sem filtro"
**Causa:** Todos os itens numa lista só.
**Solução:** Filtros DINÂMICOS por: categoria (Armas, Veículos, Skins de Jogo, Badges, Caixas), raridade (Common→Legendary), preço (range slider GCoins min-max), jogo de origem (Crash, Slots, Pool Game, Cases...), vendedor, ordenação (mais recente, mais barato, mais caro, mais popular). Filtros combinam — ex: "Skins de Pool Game + Raridade Epic + até 10.000 GCoins". Referência: Amazon faceted search, Walmart category-specific filters.

### Problema 4: "Sem precisão"
**Causa:** Preços arbitrários, sem referência de valor.
**Solução:** Price suggestion baseado em histórico (média dos últimos 10 vendidos), badge "PREÇO JUSTO" / "ACIMA DA MÉDIA" / "PECHINCHA", gráfico de preço histórico do item (mini sparkline), tooltip com "Vendido 47 vezes · Preço médio: 3.200 GCoins". Referência: CS2 PriceEmpire, Steam Market price history.

### Problema 5: "Informação vaga"
**Causa:** Cards de item sem detalhes suficientes.
**Solução:** Cada item tem: imagem, nome, descrição, raridade, jogo de origem, data de drop, preço, vendedor (com stats: "15 vendas · 4.8★"), condição/uso (se aplicável), botão "VER DETALHES" que abre modal com histórico completo do item. Referência: Gameflip item page, DMarket item details.

### Problema 6: "Conexão correta"
**Causa:** Provavelmente problemas de integração com inventário/banco/server.
**Solução:** Transações ATÔMICAS server-side: débito do comprador + crédito do vendedor + transferência do item + cobrança de taxa + registro no audit log → tudo numa TRANSACTION SQL. Se qualquer passo falhar, ROLLBACK completo. Nenhuma transação parcial. Referência: Steam trade offers (escrow system), Gameflip guarantee.
