# 1. STATE MACHINE — CICLO DE VIDA DO ITEM
# ═══════════════════════════════════════════════════

## 1.1 O Problema: Sem State Machine, items ficam em limbo

Marketplaces sem state machine definido sofrem de:
- Item listado mas vendedor desconectou — fica "fantasma" na listagem
- Item vendido mas comprador não recebeu — transação parcial
- Item cancelado mas GCoins já debitados — rollback manual
- Item em múltiplos estados simultâneos — duplicação

## 1.2 State Machine Proposto para o Blackout Casino

```
                    ┌──────────────────┐
                    │   ITEM CRIADO    │
                    │ (drop de jogo,   │
```

## 1.3 Regras de Transição

| De | Para | Condição | Ação server |
|----|------|----------|-------------|
| — | AVAILABLE | Item criado (drop, case, reward) | INSERT com origin |
| AVAILABLE | LISTED | Jogador lista no marketplace | Remover do inventário, criar listing |
| AVAILABLE | INSTANT_SOLD | Jogador vende ao sistema (preço fixo) | Remover item, creditar GCoins |
| AVAILABLE | EQUIPPED | Jogador equipa (skin, taco, etc.) | Marcar como em uso |
| EQUIPPED | AVAILABLE | Jogador desequipa | Voltar ao inventário |
| LISTED | SOLD | Outro jogador compra | Transação atômica 5 ops |
| LISTED | CANCELLED | Vendedor cancela | Devolver ao inventário |
| LISTED | EXPIRED | Timer expirou (7d default) | Auto-devolver ao inventário |
| SOLD | AVAILABLE (buyer) | Compra concluída | Item aparece no inventário do buyer com origin="MARKETPLACE" |

**Transições PROIBIDAS (anti-exploit):**
- SOLD → AVAILABLE (do vendedor) — item já saiu, não pode voltar
- LISTED → EQUIPPED — precisa cancelar antes
- INSTANT_SOLD → qualquer — terminal, sem volta
- Qualquer estado → LISTED se já tem LISTED ativo do mesmo item type (evitar listing duplicado)

## 1.4 Campo `origin` — Origin Tracking

Descoberta baseada na análise do **Escape from Tarkov** que implementa FIR (Found in Raid) badge:

```sql
ALTER TABLE player_inventory ADD COLUMN origin ENUM(
  'GAME_DROP',           -- Ganhou jogando (Crash, Slots, Pool, etc.)
  'CASE_OPEN',           -- Abriu uma caixa (Cases)
  'MARKETPLACE_PURCHASE',-- Comprou no marketplace P2P
  'REWARD_SHOP',         -- Resgatou na loja do casino
  'DAILY_FREE',          -- Ganhou no Daily Free
  'GIVEAWAY',            -- Ganhou no Giveaway
  'ADMIN_GRANT',         -- Admin deu diretamente
  'TRADE'                -- Recebeu via trade offer (futuro)
) NOT NULL DEFAULT 'GAME_DROP' COMMENT 'Como o jogador ATUAL obteve este item';
```

**Por que isso importa:** Quando jogador A abre uma caixa e dropa um Taco Dourado, o `origin` é `CASE_OPEN`. Quando jogador A vende pro jogador B no marketplace, o `origin` do jogador B é `MARKETPLACE_PURCHASE`. O item é o mesmo, mas a HISTÓRIA de como cada jogador obteve é diferente — e isso é informação valiosa para:
- UX: badge visual no inventário ("📦 Dropado" vs "🛒 Comprado")
- Filtro: jogador pode filtrar "só meus drops" vs "compras"
- Confiança: saber que o item veio de um drop legítimo
- Admin: rastrear a cadeia de transferências

## 3.2 Abordagem: Grid unificada + Badge + Filtro

Em vez de criar abas separadas (confunde), usar:

1. **Grid unificada** com todos os itens do jogador
2. **Badge no card** indicando origem:
   - 🎰 = Drop de jogo (Crash, Slots, etc.)
   - 📦 = Aberto de caixa (Cases)

   - 🎁 = Reward Shop / Daily Free / Giveaway
   - ⚡ = Admin grant
3. **Filtro toggle** no header: `[Todos] [Drops 🎰] [Cases 📦] [Compras 🛒] [Rewards 🎁]`
4. **Tooltip** no badge: "Obtido jogando Crash em 12/03/2026"

## 4.1 Instant Sell (Venda Rápida ao Sistema)

Nem todo jogador quer esperar um comprador P2P. "Instant Sell" vende ao sistema por preço fixo (geralmente 60-70% do valor de mercado):

```javascript
async function instantSell(playerId, itemId) {
  const item = await getPlayerItem(playerId, itemId);
  const marketValue = await getPriceSuggestion(item.item_type);
  const instantPrice = Math.floor(marketValue.average * CONFIG.instant_sell_ratio); // 0.65 = 65%

  await oxmysql.transaction([
    { query: 'UPDATE player_inventory SET status = "INSTANT_SOLD" WHERE id = ?', params: [itemId] },
    { query: 'UPDATE users SET bank = bank + ? WHERE identifier = ?', params: [instantPrice, playerId] },
  ]);

  return { success: true, received: instantPrice, marketValue: marketValue.average,
           ratio: CONFIG.instant_sell_ratio };
}
```

O jogador vê: "Venda Rápida: G$ 32.500 (65% do valor de mercado G$ 50.000). Deseja vender agora?"

## 4.3 Notificação de Wishlist

```javascript
async function checkWishlistAlerts(itemType, listingPrice) {
  const watchers = await oxmysql.query(
    `SELECT player_id, notify_price FROM marketplace_favorites
     WHERE item_type = ? AND notify_price >= ?`,
    [itemType, listingPrice]
  );

  for (const watcher of watchers) {
    emitNet('casino:marketplace:price_alert', parseInt(watcher.player_id), {
      itemType, price: listingPrice,

      message: `Item que você quer está por G$ ${listingPrice.toLocaleString()} (seu alerta: G$ ${watcher.notify_price.toLocaleString()})`,
    });
  }
}
```

## 4.4 Seller Rating System

Após cada compra, comprador pode avaliar (1-5 estrelas, opcional):

```sql
ALTER TABLE marketplace_transactions ADD COLUMN rating TINYINT DEFAULT NULL COMMENT '1-5 estrelas (buyer avalia seller)';
ALTER TABLE marketplace_transactions ADD COLUMN rated_at DATETIME DEFAULT NULL;

-- View de rating agregado por vendedor
CREATE OR REPLACE VIEW marketplace_seller_ratings AS
SELECT
  seller_id,
  COUNT(*) AS total_sales,
  ROUND(AVG(rating), 1) AS avg_rating,
  COUNT(rating) AS total_ratings
FROM marketplace_transactions
WHERE rating IS NOT NULL
GROUP BY seller_id;
```
