### Queries pesquisadas

| 4 | `DMarket CSFloat skin marketplace dark UI card design CSS hover glow rarity 2025` | Referencia |

### O que a pesquisa trouxe

1. **backdrop-filter: blur() flickers no NUI FiveM** — Issue #3843 no GitHub citizenfx/fivem (marco 2026). O CEF do FiveM nao limpa o framebuffer corretamente com backdrop-filter, causando flickering visivel. MITIGACAO: usar background rgba(0,0,0,0.85) SOLIDO como fallback. Aplicar backdrop-filter SOMENTE em modais criticos (1-2 elementos, nao em cards). Nunca animar o valor de blur.
2. **Glassmorphism: max 3-5 elementos simultaneos** — Guia de performance 2026 (codeformatter.in): cada elemento com backdrop-filter dispara calculo GPU separado. 10+ elementos = lag notavel em hardware medio. Para o Marketplace, usar backdrop-filter APENAS em: modal de compra, modal de detalhes, modal de Provably Fair. Cards usam background solido com opacidade.
3. **BEM 2025-2026 confirmado padrao dominante** — getbem.com + Valorem Reply (Abr/2025): `.block__element--modifier` com pseudo-states via CSS nativo (`:hover`, `:focus`), modifiers para estados de aplicacao (`--expanded`, `--selected`, `--loading`). Neste documento: `.market-[block]__[element]--[modifier]`.
4. **Dark mode e luxury UI em alta em 2026** — Halothemes: marcas premium adotam dark backgrounds para transmitir luxo. Bento Grid (layout modular) substitui listas longas, maximiza uso de tela. Micro-interacoes sao obrigatorias para feedback.
5. **CSFloat usa cards escuros com hover sutil** — grid limpo, preco em destaque, filtros laterais colapsiveis, float value como metrica de qualidade. DMarket usa cards com imagem centralizada, badge de raridade colorido, hover com elevacao.

### O que teria feito errado sem pesquisar

- Usaria backdrop-filter em TODOS os cards do grid (causaria flickering/lag no NUI FiveM)
- Nao saberia do bug #3843 do CEF e entregaria modais com blur que nao funcionam em producao
- Usaria classes sem padrao BEM, dificultando manutencao
