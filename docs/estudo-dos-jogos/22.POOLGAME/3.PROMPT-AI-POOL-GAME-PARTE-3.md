# O QUE NÃO DEVE TER — LISTA PROIBIDO (20 ITENS)

```
1.  NUNCA fundo branco, cinza claro ou qualquer cor clara — SOMENTE dark mode
    (#0A0A0A base). Felt da mesa: SEMPRE #0D3B1E (verde escuro),
    NUNCA verde brilhante. Cushions: gradiente dourado metálico, NUNCA madeira flat.

2.  NUNCA usar Three.js, react-three-fiber ou WebGL para o Pool Game —
    Canvas 2D + Matter.js é suficiente e seguro no CEF do FiveM.
    WebGL pode causar crash, blank screen ou FPS inaceitável no NUI.

3.  NUNCA font sans-serif em títulos do jogo — SEMPRE Cinzel Decorative para
    "POOL GAME", nomes de cues, "VOCÊ VENCEU!", "DERROTA", nomes dos tiers.

4.  NUNCA px fixo para fontes, padding ou tamanhos de layout —
    SEMPRE clamp(min, vw, max). Errado: font-size: 16px.
    Correto: font-size: clamp(11px, 1.4vw, 16px).

5.  NUNCA border-radius > 18px em containers — Cards: 10-14px,
    Botões: 8-10px, Bolas Canvas: 50% (círculo perfeito).

6.  NUNCA single-layer box-shadow — SEMPRE multi-layer (mínimo 2-3 camadas):
    ambient + contact shadow + glow colorido para elementos ativos.

7.  NUNCA título de jogo sem text-shadow glow dourado:
    text-shadow: 0 0 12px rgba(255,215,0,0.6), 0 2px 6px rgba(0,0,0,1).

8.  NUNCA botão sem 4 estados: normal, hover, active, disabled.
    Disabled: opacity 0.35, cursor not-allowed, pointer-events: none.

9.  NUNCA elemento interativo sem tooltip — 18 tooltips mapeados acima.
    Incluindo: todos os botões, spin selector, power bar, guideline,
    ghost ball, timer, POT display, todos os tiers no lobby, stat bars dos cues.

10. NUNCA valores GCoin em cor diferente de #00E676 (verde neon).
    text-shadow: 0 0 10px rgba(0,230,118,0.5) obrigatório.
    Valores negativos (derrota): #FF3B3B com shadow vermelho sutil.

11. NUNCA shadowBlur no Canvas loop de animação — usar radialGradient fake
    para glow de bolas e caçapas. shadowBlur é O(n*pixels) e destrói
    FPS no NUI. Proibido em drawBall(), drawTable(), drawGuideline(),
    ou qualquer função chamada dentro de requestAnimationFrame.

12. NUNCA calcular physics definitiva no client — client ANIMA o resultado
    recebido do server via lerp (interpolação suave). Exception permitida:
    guideline visual (aiming preview) que não afeta resultado real.

13. NUNCA mostrar guideline de mira do oponente — revelar estratégia do
    adversário quebra o fair play. Mostrar apenas cue stick sutil (opcional,
    atualização a 4fps, sem guideline).

14. NUNCA scroll vertical para navegar entre cues no Cue Shop em mobile —
    SEMPRE scroll-snap horizontal (overflow-x: auto + scroll-snap-type: x mandatory).

15. NUNCA rolling counter sem easing — o prêmio deve desacelerar nos
    últimos 20% para criar tensão: ease-out quint (1 - Math.pow(1 - t, 5)).
    Counter linear sem easing parece placeholder.

16. NUNCA exibir rake sem contexto — sempre mostrar:
    "Rake: G$100 (5% do pot)" ou "Você recebe: G$1.900 (pot - rake)".
    Jogador deve entender a taxa antes de entrar na partida.

17. NUNCA foul sem feedback triplo simultâneo:
    (1) flash vermelho Canvas na mesa, (2) banner slide-in "FOUL",
    (3) som foul.mp3. Qualquer elemento faltando = experiência incompleta.

18. NUNCA pool game sem landscape lock em mobile —
    Canvas de mesa precisa de landscape para proporção 2:1.
    Sempre tentar screen.orientation.lock('landscape') e mostrar
    rotate-warning animado se não suportado.

19. NUNCA reescrever componentes inteiros ao adicionar features — edições
    cirúrgicas str_replace (REGRA X6). Novas features são camadas adicionadas
    sobre componentes existentes, nunca substituição total.

20. NUNCA atualizar saldo visível antes da animação terminar —
    saldo atualiza DEPOIS de: rolling counter terminar + confetti aparecer
    (vitória) ou shake terminar (derrota). Nunca imediatamente ao receber
    o response do server — o jogador deve viver o momento antes de ver o número.
```
