## RELATORIO DA PESQUISA (REGRA X0)

**Rodada 1 — oxmysql FiveM query/insert/update async 2025-2026:**
- Query: "oxmysql FiveM query_async insert_async transaction 2025 2026"
- Fontes: coxdocs.dev/oxmysql, oxmysql.com, github.com/overextended/oxmysql, forum.cfx.re/oxmysql, vertexmods.com/fivem-database-guide, fivemx.com/mysql-async-to-oxmysql
- Descobertas: (1) OxMySQL usa exports: `exports['oxmysql']['query_async']`, `insert_async`, `update_async`, `scalar_async`. (2) Prepared statements com `?` positional obrigatorios. (3) MariaDB recomendado sobre MySQL 8 (reserved keywords). (4) OxMySQL foi arquivado Abr/2025 mas continua funcional e padrao FiveM. (5) Transaction via `exports['oxmysql']['transaction_async']` para operacoes atomicas multi-query. (6) Connection pooling nativo, async por default.
- Impacto: Confirma padrao do Mega Estudo. Usar `exports['oxmysql']` para todas queries. UNIQUE KEY (player_identifier, claim_date) dispensa transaction para anti-abuse.

**Rodada 2 — HMAC-SHA256 Node.js crypto provably fair 2025:**
- Query: "HMAC SHA256 Node.js crypto createHmac server seed provably fair casino 2025"
- Fontes: nodejs.org/api/crypto.html, geeksforgeeks.org/node-js-crypto-createhmac, stake.com/provably-fair/implementation, github.com/nucleare/DeepSearch_HashGames, btcgosu.com/provably-fair, mojoauth.com/hashing/hmac-sha256-in-nodejs
- Descobertas: (1) `crypto.createHmac('sha256', serverSeed).update(clientSeed + ':' + nonce).digest('hex')` e o padrao. (2) Stake.com usa cursor para jogos que precisam de 8+ outcomes — Daily-Free NAO precisa (1 segmento = 1 outcome, sem cursor). (3) Hmac NAO e reutilizavel apos `.digest()` — criar novo por operacao. (4) Primeiro 4 bytes do hash hex → parseInt → modulo total_peso → selecao ponderada. (5) `crypto.randomBytes(32).toString('hex')` para gerar serverSeed aleatorio.
- Impacto: Simplifica PF do Mega Estudo. Sem cursor, nonce = auto-increment do claim_id.

**O que teriamos feito sem a pesquisa:**
- Usado mysql-async em vez de oxmysql (deprecated, menos performatico)
- Adicionado cursor desnecessario no HMAC (complexidade extra sem beneficio para roda de 8 segmentos)
- Criado Hmac reutilizavel causando erro silencioso (digest so funciona 1x)

  INDEX idx_claim_date (claim_date),
  INDEX idx_player (player_identifier),
  INDEX idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
