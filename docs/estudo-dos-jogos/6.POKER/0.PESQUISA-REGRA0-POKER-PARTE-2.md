## 8.1 Adaptacao do Sistema (mesmo padrao Slots/Roulette)

```
ANTES DA MAO:
1. Server gera server_seed (random 32 bytes)
2. Server calcula hash = SHA-256(server_seed)
3. Hash enviado ao client (NUI) ANTES do deal
4. Client gera client_seed (automatico ou input manual)
5. Nonce incrementa a cada mao

NO DEAL:
6. Combined = HMAC-SHA256(server_seed, client_seed + ":" + nonce)
7. Hex result convertido para sequence de indices
8. Indices mapeiam para posicoes no deck shuffled
9. Deck shuffled determina: cartas do jogador, cartas do dealer,
   community cards (UTH)

APOS A MAO:
10. Server revela server_seed ao fim da sessao (ou apos N maos)
11. Client verifica: SHA-256(revealed_server_seed) == hash_recebido
12. Se match: resultado foi justo. Se nao: manipulacao detectada
```

**Diferenca vs Slots:** No poker, o resultado e um deck shuffled (52 posicoes) ao inves de simbolos em grid. O HMAC gera a sequencia de shuffle, nao o resultado individual.

**Modal Provably Fair no NUI:** Mesma estrutura dos outros jogos — campos para server_seed_hash, client_seed, nonce, botao "Verificar".
