## Endpoint 1: `casino:poker:ante`

**Funcao:** Iniciar uma nova mao. Jogador coloca ante (e blind/trips no UTH).

**Parametros de entrada:**
```
{
  source: number          // Player server ID (injetado pelo FiveM)
  bet: number             // Valor do ante (min 10, max 10000)
  mode: 'caribbean'|'uth' // Modo escolhido
  clientSeed: string      // Seed do jogador para Provably Fair (max 64 chars)
  jackpotBet: boolean     // Caribbean: side bet $1 para progressive
  tripsBet: number|null   // UTH: valor da trips side bet (null se nao apostar)
}
```

**Validacoes (server-side, nesta ordem):**
1. `mode` deve ser 'caribbean' ou 'uth'. Rejeitar qualquer outro valor.
2. `bet` deve ser numerico, >= min_ante, <= max_ante. Rejeitar NaN, negativo, zero.
3. Cooldown: verificar se ultima mao do jogador foi ha mais de `rate_limit_ms` (config). Rejeitar se cooldown ativo.
4. Saldo: verificar se jogador tem saldo >= bet total (ante + blind + trips + jackpot). Debitar ANTES de distribuir cartas.
5. `clientSeed`: string, max 64 caracteres, sanitizar contra SQL injection.
6. `tripsBet`: se mode='uth', validar numerico >= 0. Se mode='caribbean', ignorar.
7. `jackpotBet`: se true, verificar se jackpot_enabled=true na config. Debitar $1 adicional.

**Logica interna:**
1. Gerar server_seed (crypto.randomBytes(32).toString('hex'))
2. Calcular server_seed_hash = SHA-256(server_seed)
3. Gerar deck shuffled via HMAC-SHA256(server_seed, clientSeed + ':' + nonce)
4. Distribuir cartas:
   - Caribbean: 5 para jogador (face up), 5 para dealer (4 down + 1 up)
   - UTH: 2 hole para jogador (face up), 2 hole para dealer (down)
5. Salvar sessao Provably Fair no MySQL
6. Criar sessionId (UUID v4)

**Retorno (callback):**
```
{
  success: true,
  sessionId: string,        // UUID para identificar esta mao
  playerCards: string[],     // Ex: ['Ad','Kh','Qs','Jd','9c']
  dealerUpCard: string|null, // Caribbean: 1 carta. UTH: null
  serverSeedHash: string,    // Hash para verificacao futura
  jackpotPool: number|null,  // Valor atual do pool (se jackpotBet=true)
  balance: number            // Saldo atualizado apos debito
}
```

**Retorno erro:**
```
{ success: false, error: 'INSUFFICIENT_BALANCE'|'COOLDOWN_ACTIVE'|'INVALID_MODE'|'INVALID_BET' }
```

## Endpoint 2: `casino:poker:decision` (Caribbean Stud)

**Funcao:** Jogador decide fold ou raise no Caribbean Stud.

**Parametros de entrada:**
```
{
  source: number
  sessionId: string     // UUID da mao atual
  action: 'fold'|'raise' // Decisao do jogador
}
```

**Validacoes:**
1. `sessionId` deve existir, pertencer ao source, estar no estado 'decision_pending'.
2. `action` deve ser 'fold' ou 'raise'.
3. Se 'raise': verificar saldo >= 2x ante (raise = 2x ante). Debitar.

**Logica interna (se raise):**
1. Revelar cartas do dealer (todas 5)
2. Verificar dealer qualifica (AK ou melhor) usando pokersolver
3. Se dealer NAO qualifica: ante paga 1:1, raise push (retorna)
4. Se dealer qualifica: comparar maos via Hand.winners()
   - Jogador ganha: ante 1:1 + raise pela paytable (lookup hand name -> multiplier)
   - Dealer ganha: jogador perde ante + raise
   - Empate: ante + raise push (retornam)
5. Se jackpotBet=true: verificar mao do jogador contra jackpot paytable
   - Royal Flush: 100% pool
   - Straight Flush: 10% pool
   - Four Kind: $500
   - Full House: $100
   - Flush: $50
   - Jackpot paga INDEPENDENTE de vitoria vs dealer
6. Atualizar saldo, registrar mao no historico, atualizar jackpot pool

**Logica interna (se fold):**
1. Jogador perde ante. Registrar resultado 'fold'.
2. Se jackpotBet=true: ainda verificar jackpot (jogador pode ter flush+ e ter foldado por engano — mas no Caribbean Stud real, fold cancela tudo incluindo jackpot). Decisao: jackpot NAO paga em fold (padrao industria).

**Retorno (callback):**
```
{
  success: true,
  dealerCards: string[],      // Todas cartas do dealer reveladas
  dealerHandName: string,     // Ex: 'Two Pair'
  dealerQualified: boolean,
  playerHandName: string,     // Ex: 'Flush'
  result: 'win'|'lose'|'push'|'fold',
  antePayout: number,
  raisePayout: number,
  jackpotPayout: number,      // 0 se nao ganhou
  totalPayout: number,
  balance: number,
  serverSeed: string|null     // Revelado se sessao encerrada
}
```

## Endpoint 3: `casino:poker:preflop` (UTH)

**Funcao:** Decisao pre-flop no Ultimate Texas Hold'em.

**Parametros:**
```
{
  source: number
  sessionId: string
  action: 'bet4x'|'bet3x'|'check'
}
```

**Validacoes:**
1. `sessionId` estado deve ser 'preflop_pending'.
2. `action` deve ser um dos 3 valores.
3. Se 'bet4x': verificar saldo >= 4x ante. Debitar. Marcar playBetPlaced=true.
4. Se 'bet3x': verificar saldo >= 3x ante. Debitar. Marcar playBetPlaced=true.
5. Se 'check': nenhum debito. Avancar para flop.

**Logica:** Se bet4x/3x: marcar que o jogador JA apostou (nao pode apostar de novo no flop/river). Revelar flop (3 community cards).

**Retorno:**
```
{
  success: true,
  communityCards: string[],  // 3 cartas do flop
  playBetPlaced: boolean,    // true se apostou
  canBetFlop: boolean        // false se ja apostou
}
```

## Endpoint 4: `casino:poker:flop` (UTH)

**Funcao:** Decisao pos-flop no UTH.

**Parametros:**
```
{
  source: number
  sessionId: string
  action: 'bet2x'|'check'
}
```

**Validacoes:**
1. `sessionId` estado deve ser 'flop_pending'.
2. Se `playBetPlaced` ja e true (apostou no preflop): action DEVE ser 'check' (automatico). Rejeitar bet2x.
3. Se 'bet2x': verificar saldo >= 2x ante. Debitar.

**Logica:** Revelar turn + river (2 community cards). Se ninguem apostou ate aqui, avancar para river_decision.

**Retorno:**
```
{
  success: true,
  communityCards: string[],  // Agora 5 cartas
  playBetPlaced: boolean,
  canBetRiver: boolean       // false se ja apostou antes
}
```

## Endpoint 5: `casino:poker:river` (UTH)

**Funcao:** Decisao final no UTH. Se nao apostou antes, DEVE apostar 1x ou fold.

**Parametros:**
```
{
  source: number
  sessionId: string
  action: 'bet1x'|'fold'
}
```

**Validacoes:**
1. `sessionId` estado deve ser 'river_pending'.
2. Se `playBetPlaced` ja e true: pular para showdown (automatico).
3. Se 'bet1x': verificar saldo >= 1x ante. Debitar.
4. Se 'fold': jogador perde ante + blind. Trips ainda paga se aplicavel.

**Logica (showdown):**
1. Revelar cartas do dealer (2 hole cards)
2. Avaliar melhor mao de 5 cartas (de 7) para jogador e dealer usando pokersolver
3. Dealer qualifica com pair+
4. Se dealer NAO qualifica: Play paga 1:1, Ante push
5. Se dealer qualifica e jogador ganha: Play 1:1 + Ante 1:1
6. Blind: paga por paytable se jogador tem straight+ (independente de dealer)
7. Trips: paga por paytable se jogador tem three kind+ (independente de vitoria)

**Retorno:**
```
{
  success: true,
  dealerCards: string[],
  dealerHandName: string,
  dealerQualified: boolean,
  playerHandName: string,
  result: 'win'|'lose'|'push'|'fold',
  antePayout: number,
  playPayout: number,
  blindPayout: number,
  tripsPayout: number,
  totalPayout: number,
  balance: number,
  serverSeed: string|null
}
```
