## 5.2 ALTER Necessario

```sql
-- INVENTORY #17 - ALTER para campo equipped_at
-- Adiciona timestamp de quando item foi equipado
-- Copiar-colar funciona - MySQL 5.7+/8.0

ALTER TABLE casino_cases_inventory
    ADD COLUMN equipped_at TIMESTAMP NULL DEFAULT NULL AFTER sold_at;

-- Index para consultas de itens equipados por jogador
ALTER TABLE casino_cases_inventory
    ADD INDEX idx_user_equipped (user_id, status, equipped_at);
```

**Motivo:** O campo `sold_at` existe mas `equipped_at` nao. Para historico e audit, precisamos saber QUANDO o item foi equipado, nao apenas que foi. O index otimiza a query "quais itens estao equipados por este jogador".

## 5.3 Queries do Inventory (Usadas nos Handlers)

```sql
-- Q1: Listar inventario do jogador (handler my_inventory - JA EXISTE)
SELECT inv.id as inv_id, inv.status, inv.acquired_at, inv.equipped_at,
       i.name, i.rarity, i.rarity_color, i.value, i.sell_back_value, 
       i.image_url, i.item_type, i.description, i.game_item_id,
       o.id as opening_id, o.case_id,
       c.name as case_name
FROM casino_cases_inventory inv
JOIN casino_cases_openings o ON inv.opening_id = o.id
JOIN casino_case_items i ON inv.item_id = i.id
JOIN casino_cases c ON o.case_id = c.id
WHERE inv.user_id = ? AND inv.status IN ('owned','equipped')
ORDER BY inv.acquired_at DESC;

-- Q2: Equipar item (handler novo)
UPDATE casino_cases_inventory 
SET status = 'equipped', equipped_at = NOW() 
WHERE id = ? AND user_id = ? AND status = 'owned';

-- Q3: Desequipar item (handler novo)
UPDATE casino_cases_inventory 
SET status = 'owned', equipped_at = NULL 
WHERE id = ? AND user_id = ? AND status = 'equipped';

-- Q4: Vender item (handler JA EXISTE - sem alteracao)

-- UPDATE casino_cases_inventory SET status='sold', sold_at=NOW()

-- Q5: Contar itens por jogador (header)
SELECT COUNT(*) as total, 
       SUM(i.sell_back_value) as valor_total
FROM casino_cases_inventory inv
JOIN casino_case_items i ON inv.item_id = i.id
WHERE inv.user_id = ? AND inv.status IN ('owned','equipped');

-- Q6: Historico de vendas do jogador
SELECT i.name, i.rarity, i.sell_back_value, inv.sold_at
FROM casino_cases_inventory inv
JOIN casino_case_items i ON inv.item_id = i.id
WHERE inv.user_id = ? AND inv.status = 'sold'
ORDER BY inv.sold_at DESC LIMIT 20;
```

## 5.4 Queries Admin

```sql
-- Total de itens em circulacao por status
SELECT status, COUNT(*) as qtd FROM casino_cases_inventory GROUP BY status;

-- Jogadores com inventario mais valioso
SELECT inv.user_id, COUNT(*) as itens, SUM(i.sell_back_value) as valor_total
FROM casino_cases_inventory inv
JOIN casino_case_items i ON inv.item_id = i.id
WHERE inv.status IN ('owned','equipped')
GROUP BY inv.user_id ORDER BY valor_total DESC LIMIT 20;

-- Itens mais vendidos (sell-back)
SELECT i.name, i.rarity, COUNT(*) as vendas, SUM(i.sell_back_value) as total_pago
FROM casino_cases_inventory inv
JOIN casino_case_items i ON inv.item_id = i.id
WHERE inv.status = 'sold'
GROUP BY i.id ORDER BY vendas DESC;

-- Audit de equip/unequip
SELECT * FROM casino_cases_audit 
WHERE action IN ('equip','unequip') 
ORDER BY created_at DESC LIMIT 50;

-- Revenue total de sell-back (GCoins saindo do casino para jogadores)
SELECT SUM(i.sell_back_value) as total_pago_sellback
FROM casino_cases_inventory inv
JOIN casino_case_items i ON inv.item_id = i.id
WHERE inv.status = 'sold';
```
