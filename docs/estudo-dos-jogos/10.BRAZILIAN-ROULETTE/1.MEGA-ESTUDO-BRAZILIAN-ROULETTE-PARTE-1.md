# 4. RISCOS DE SEGURANÇA

| # | Risco | Sintoma | Causa | Solução |
|:-:|-------|---------|-------|---------|
| 1 | Manipulação de apostas | Jogador envia aposta após saber resultado | Client enviando bets depois do spin | Server tem `BETTING_PHASE` boolean. Rejeita bets quando `false`. Timer server-side, não client |
| 2 | Multiplicadores fraudados | Jogador força multiplicador no número apostado | Client tentando definir Lucky Numbers | Lucky Numbers gerados APENAS no server. Client recebe resultado final, nunca escolhe |
| 3 | Payout inflado | Jogador recebe mais do que deveria | Bug no cálculo de payout com multiplicador | Validação server: `payout = basePayout × multiplier`. Verificação dupla. Audit log com cálculo completo |
| 4 | Duplicação de apostas | Jogador aposta várias vezes na mesma rodada | Falta de lock por rodada | Cada rodada tem `roundId`. Server mantém `Map<playerId, Set<roundId>>`. Rejeita duplicatas |
| 5 | Saldo negativo | Jogador aposta mais do que tem | Race condition no débito | Débito atômico: `UPDATE users SET balance = balance - ? WHERE balance >= ? AND id = ?`. Se affected_rows = 0, rejeita |
