# 8. PROVABLY FAIR — 8 PASSOS

## 8.1 Algoritmo para Roleta com Multiplicadores

A Brazilian Roulette #10 usa Provably Fair com DOIS resultados por rodada: (1) posicao da bola (0-36) e (2) quais numeros recebem multiplicador e qual valor.

| Passo | Acao | Detalhe |
|:-----:|------|---------|
| 1 | Servidor gera serverSeed (32 bytes hex aleatorio) | `crypto.randomBytes(32).toString('hex')` |
| 2 | Servidor calcula serverSeedHash | `SHA256(serverSeed)` — hash publicado ao jogador ANTES da aposta |
| 3 | Jogador fornece clientSeed (ou browser gera automatico) | Input opcional no modal PF. Default: `Math.random().toString(36).slice(2)` |
| 4 | Nonce incrementa automaticamente a cada rodada | Inteiro, comeca em 0, incrementa +1 |
| 5 | Servidor calcula HMAC para posicao da bola | `HMAC_SHA256(serverSeed, clientSeed + ':' + nonce + ':0')` → primeiros 8 hex → parseInt → modulo 37 = numero resultado (0-36) |
| 6 | Servidor calcula HMAC para multiplicadores (modo Relampago) | `HMAC_SHA256(serverSeed, clientSeed + ':' + nonce + ':1')` → primeiros 2 hex = quantidade (1-5). Proximos bytes em pares = quais numeros (modulo 37, sem repetir). Ultimos bytes = valor do multiplicador por numero (mapeado por faixa: 0-127=50x, 128-191=100x, 192-223=200x, 224-243=250x, 244-251=300x, 252-254=400x, 255=500x) |
| 7 | Jogador pode verificar apos rodada | Servidor revela serverSeed. Jogador recalcula: SHA256(serverSeed) deve == serverSeedHash publicado no passo 2. Recalcula HMAC para confirmar resultado |
| 8 | Badge JUSTO (verificado) aparece no historico | Se verificacao OK, badge verde "JUSTO" ao lado da rodada. Se fail, badge vermelho "INVALIDO" + alerta admin |

## 8.2 Variaveis do Algoritmo
