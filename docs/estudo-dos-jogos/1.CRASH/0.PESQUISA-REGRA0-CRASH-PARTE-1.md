## 2.1 FORMATO A — Grafico Puro (Stake.com Crash)

**Mecanica:** Grafico com eixo X (tempo) e Y (multiplicador). Uma linha/ponto sobe diagonalmente. Multiplicador comeca em 1.00x e cresce exponencialmente. O jogador faz cashout antes do crash. Sem avatar, sem foguete — apenas a curva matematica.

**Detalhamento completo do Stake Crash:**
- Multiplicador: inicia 1.00x, maximo teorico 1.000.000x (rounds especiais ja atingiram 25.000x)
- Cashout minimo: 1.01x
- Tempo entre rounds: 5 segundos
- Duracao do round: 5-30 segundos (depende de quando crasha)
- RTP: 99% (house edge 1% — um dos mais baixos do mercado)
- Modos: Manual (define bet + clica cashout manualmente) e Auto (define bet + target multiplier + regras de increase/decrease)
- Auto Bet: permite configurar aumento de bet apos win ou loss (Martingale automatizado)
- Auto Cashout: define multiplicador alvo, sistema saca automaticamente
- Social: jogo live com centenas de jogadores simultaneos. Ve quando outros sacam. Chat integrado. Leaderboard
- Historico: barra no topo mostra resultados dos ultimos N rounds (ex: 1.23x, 45.67x, 1.01x...)
- Visual: fundo escuro, curva verde neon subindo, multiplicador grande central, crash em vermelho
- Provably Fair: SHA-256 chain de hashes. Server seed (64 char hex) hashado antes das apostas. Client seed do jogador. Nonce incrementa. Byterator gera resultado

**UX critica — o que torna o Stake Crash viciante:**
1. Velocidade: rounds de 5-30s = dopamina constante
2. Controle: o jogador DECIDE quando sair (diferente de slots que sao passivos)
3. Social: ver outros jogadores sacando/perdendo cria pressao e comparacao
4. Historico visivel: barra de resultados anteriores cria ilusao de padrao (gambler's fallacy)
5. Simplicidade: 2 inputs (bet + cashout) = curva de aprendizado zero

**Para RP:** Modelo perfeito para o Blackout Casino. Visual minimalista se adapta bem ao Design System dourado/verde. A curva neon e o multiplicador gigante sao os elementos visuais centrais. O aspecto social funciona em FiveM pois multiplos jogadores estao no mesmo servidor. House edge de 1% (como Stake) permite economia equilibrada no RP.
