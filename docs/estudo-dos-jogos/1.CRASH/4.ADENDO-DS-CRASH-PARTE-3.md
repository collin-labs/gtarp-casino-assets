# SECAO 3 — TIMELINES EXATAS

## 3.1 Timeline Round Completo (Waiting → Betting → Rising → Crash → Result)

```
t=0ms        Round anterior termina. Server envia: { phase: 'waiting', nextRoundIn: 5000 }
             Client recebe via NUI message.

t=0ms        WAITING inicia:
             - Canvas limpo (clearRect).
             - Grid desenhado (§1.3 drawGrid).
             - Multiplicador muda para .crash-canvas__multiplier--waiting (§1.6).
             - Texto: "PROXIMO ROUND" Cinzel 700 #D4A843.
             - Countdown numerico aparece: "5s" JetBrains Mono 800 #FFD700.
             - Barra de progresso inicia preenchimento linear 5s.
             - Canvas wrapper: .crash-canvas--waiting (§1.2).

t=0-5000ms   COUNTDOWN (5 segundos):
             A cada segundo (1000ms intervals):
             - Numero atualiza com Framer Motion key change:
               initial={{ scale: 1.4, opacity: 0 }}
               animate={{ scale: 1, opacity: 1 }}
               transition={{ duration: 0.3 }}
             - Barra fill width: (elapsed / 5000) * 100%.
             - t=2000ms: Fase muda para BETTING (server envia phase: 'betting').

t=2000ms     BETTING inicia (3 segundos restantes de apostas):
             - Multiplicador muda para .crash-canvas__multiplier--betting (§1.6).
             - Texto: "APOSTE AGORA" Cinzel 800 #FFD700 com bettingPulse.
             - Botao APOSTAR habilitado (.crash-btn--bet ativo).
             - Controles: inputs ativos, atalhos ativos.
             - Som: nenhum (silencio de concentracao).

t=2000-5000ms Jogadores enviam apostas:
             - Client: fetchNui('casino:crash:bet', { amount })
             - Lua: TriggerServerEvent('casino:crash:bet', amount)
             - Server valida saldo, registra bet, retorna confirmacao.
             - Feed atualiza: "Jogador_X: 500 GC → Apostou" fade-in 200ms.
             - Input muda para --disabled apos aposta confirmada.

t=5000ms     RISING inicia. Server envia: { phase: 'rising', startTime: now }
             - Canvas: rAF loop inicia.
             - Frame 0: curva comeca no canto inferior esquerdo (0, h).
             - Multiplicador muda para .crash-canvas__multiplier--rising (§1.6).
             - Canvas wrapper: .crash-canvas--rising (§1.2) glow verde.
             - Tom ascendente inicia: OscillatorNode sine 200Hz, gain 0.04.
             - Botao CASHOUT aparece (AnimatePresence, scale 0.85→1, opacity 0→1, 250ms).
             - Botao APOSTAR oculto (display: none).
             - Inputs: --disabled.

t=5000ms+    RISING loop (duracao variavel — depende do crash point):
             A cada frame (~16ms a 60fps):
             1. clearRect(0, 0, w, h)
             2. drawGrid(ctx, w, h)
             3. Calcular multiplier: Math.pow(Math.E, GROWTH_RATE * elapsed)
             4. Calcular novo ponto: { x: progressX, y: h - log(mult)/log(maxScale) * h }
             5. Adicionar ponto ao array points[]
             6. drawCurve(ctx, w, h, points, currentX, currentY, maxScale)
             7. Atualizar texto multiplicador: "{mult.toFixed(2)}x"
             8. Atualizar valor cashout no botao: "SACAR {bet*mult} GC"
             9. Atualizar frequencia do oscillator:
                freq = 200 + (log(mult)/log(100)) * 1800
                osc.frequency.setTargetAtTime(min(freq,2000), ctx.currentTime, 0.1)

             A cada CASHOUT de outro jogador:
             - Feed entry aparece: "Jogador_X: sacou 2.10x" #00E676, fade-in 200ms.

             Quando JOGADOR LOCAL faz cashout:
             - Client: fetchNui('casino:crash:cashout')
             - Server valida, retorna: { success: true, cashoutAt: mult, payout: amount }
             - Botao CASHOUT some (AnimatePresence exit).
             - Som: crash_cashout (523→659→784Hz, 100ms cada).
             - Feed: "Voce: sacou {mult}x" #00E676.

t=CRASH      Server detecta: multiplier >= crashPoint.
             Server envia: { phase: 'crashed', crashPoint: X.XX, results: [...] }

t=CRASH+0ms  Client recebe crash:
             1. rAF loop PARA (cancelAnimationFrame).
             2. Tom ascendente: gain.exponentialRampToValueAtTime(0.001, ctx.currentTime + 0.15).
             3. Curva redesenhada VERMELHA: drawCurveCrashed() (§1.4).
             4. Flash vermelho: drawCrashFlash(ctx, w, h, 0.12) — decai 0.01/frame.

t=CRASH+50ms Screen shake: container Framer Motion x: [-3,4,-3,2,-1,0] 500ms.

t=CRASH+100ms Particulas: createCrashParticles(curveEndX, curveEndY) — 20 circulos.
             updateAndDrawParticles() chamado por 60 frames separado do loop principal.
             Som: crash_explode — 110Hz sawtooth 300ms volume 0.15.

t=CRASH+150ms Oscillator para: osc.stop().

t=CRASH+200ms Multiplicador: .crash-canvas__multiplier--crashed (§1.6).
             crashedFlash keyframe: scale 1→1.35→1.1 em 400ms.
             Canvas wrapper: .crash-canvas--crashed (§1.2) glow vermelho.

t=CRASH+300ms Badge novo na barra: .crash-badge--new (§1.7) spring 350ms.

t=CRASH+500ms Overlay resultado:
             SE jogador SACOU: overlay vitoria (§2.4 vitoria).
             SE jogador PERDEU: overlay derrota (§2.4 derrota).
             SE jogador NAO APOSTOU: nenhum overlay, apenas visual do crash.

t=CRASH+3500ms Overlay auto-dismiss (ou clique antes).
             Saldo atualizado no header.
             Feed finalizado: todas entradas "active" → "Perdeu" #FF4444.

t=CRASH+5000ms WAITING do proximo round inicia (volta para t=0ms).

TOTAL round cycle: ~15-25 segundos (5s waiting + variavel rising + 5s cooldown)
```

## 3.2 Timeline Cashout do Jogador (desde clique ate feedback)

```
t=0ms        Jogador clica/toca botao CASHOUT.
             Botao fica --disabled temporariamente (previne double-click).

t=0-20ms     fetchNui('casino:crash:cashout') →
             Lua Client → TriggerServerEvent('casino:crash:cashout')

t=20-80ms    Server valida:
             - Jogador tem aposta ativa? (sim → continua, nao → rejeita)
             - Phase ainda e 'rising'? (sim → continua, nao → rejeita)
             - Registra cashout com multiplicador ATUAL do server.
             - Calcula payout: bet * cashoutMultiplier.
             - Credita saldo.
             - Retorna: { success, cashoutAt, payout }

t=80ms       Client recebe resposta positiva:
             1. Botao CASHOUT some: AnimatePresence exit (opacity 0, scale 0.85, y -10, 250ms).
             2. Som cashout: 523Hz→659Hz→784Hz (Do-Mi-Sol), 100ms cada, delay 80ms.
             3. Feed atualiza: "Voce: sacou {mult}x → +{payout} GC" #00E676.
             4. Status do bet muda para 'cashed_out'.

t=330ms      Ultimo tom (Sol 784Hz) termina.

t=400ms      Overlay de vitoria NAO aparece ainda — espera o crash acontecer.
             Jogador observa a curva continuar subindo (outros ainda jogando).
             Se a curva sobe muito alem do cashout, jogador sente FOMO.
             Se a curva crashar logo, jogador sente alivio.

t=CRASH      Quando crash acontece:
             Overlay vitoria aparece (§2.4) mostrando ganho do jogador.

TOTAL cashout flow: ~80ms ate confirmacao visual.
```
