## 1.5 Fluxo Completo de um Round (18 passos)

```
1.  Server inicia round: gera serverSeed (crypto.randomBytes(32).toString('hex'))
2.  Server calcula crashPoint via Provably Fair (HMAC_SHA256)
3.  Server envia hash do serverSeed para todos NUIs (compromisso criptografico)
4.  Estado: BETTING (5 segundos). NUIs mostram "APOSTE AGORA" + countdown

5.  Jogador A: abre CrashControls, define bet 500 GC, auto cashout 2.50x
6.  casino:crash:bet { amount: 500, autoCashout: 2.50 }
7.  Server valida 5 checks: saldo, limites, estado do round, cooldown, duplicidade
8.  Server: vRP.removeMoney(playerA, 500). INSERT casino_crash_bets
9.  Server broadcast: nova aposta para todos NUIs (CrashBetFeed atualiza)

10. Estado: RISING. Server envia sinal de inicio. Todos NUIs iniciam animacao
11. Client: Canvas desenha curva subindo. Multiplicador incrementa. Som ascendente inicia
12. Multiplicador atinge 2.50x: server detecta autoCashout do Jogador A
    -> vRP.addMoney(playerA, 500 * 2.50 = 1250). UPDATE casino_crash_bets
    -> Broadcast: "Jogador A sacou em 2.50x" para todos NUIs
13. NUI do Jogador A: overlay verde "VOCE SACOU EM 2.50x! +750 GC"
14. Jogador B (sem autoCashout): continua assistindo. Multiplicador sobe...

15. Multiplicador atinge crashPoint (ex: 3.71x). Server envia CRASHED
16. Todos NUIs: curva para, fica vermelha. Multiplicador final "CRASHED @ 3.71x"
17. Jogador B (nao sacou): "VOCE PERDEU 200 GC". Shake. Tom grave
18. Server revela serverSeed. Apos 5s cooldown, volta ao passo 1
```
