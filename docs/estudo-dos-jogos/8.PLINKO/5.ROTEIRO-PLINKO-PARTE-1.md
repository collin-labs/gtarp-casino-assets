## RELATORIO DA PESQUISA (REGRA X0)

**Pesquisas realizadas para o Roteiro:**
1. `plinko provably fair HMAC SHA256 implementation server-side code` → 10 resultados
2. `canvas 2D plinko ball animation bezier curve requestAnimationFrame code` → 10 resultados

**Fontes relevantes:**
- Skincade Plinko Fairness (skincade.com/fairness/plinko): Codigo REAL funcional do byteGenerator — `createHmac('sha256', seed).update(entropy + ':' + nonce + ':' + currentRound)`. Loop `while(true)` que yield 32 bytes por round. generateFloats usa 4 bytes por float (range 0 a 4294967295 / 2^32). getPlinkoOutcome fatia o array de floats em grupos de rowAmount para cada bola
- Stake.com Provably Fair Implementation (stake.com/provably-fair/implementation): Confirma padrao HMAC_SHA256 com cursor multi-round. Server seed + client seed + nonce + cursor. Cursor avanca 32 bytes por round de HMAC
- NSoft Medium (medium.com/nsoft/provably-fair): `crypto.randomBytes(32).toString('hex')` para gerar server seed. `crypto.createHash('sha256').update(seed).digest('hex')` para hash publico. Seed hash publicado ANTES da rodada, seed revelado DEPOIS da troca
- GitHub uday6703/plinko_app: Implementacao funcional com `SHA256(serverSeed + ':' + clientSeed + ':' + nonce)` como combined seed. Test vectors confirmados: seed `b2a5f3...`, nonce 42, resultado reproduzivel
- MDN Advanced Animations (developer.mozilla.org): Padrao `requestAnimationFrame(draw)` com `clearRect` a cada frame. Gravity simulation: `ball.vy *= 0.99; ball.vy += 0.25`. Boundary detection com `ball.y + ball.vy > canvas.height - ball.radius`
- Versionfeatures Canvas Bezier 2026: `willReadFrequently: true` no getContext para apps que leem pixels frequentemente. Batch paths com `beginPath()` unico. HiDPI scaling: `ctx.scale(devicePixelRatio, devicePixelRatio)`. OffscreenCanvas para workers em 2026

**O que a pesquisa trouxe de novo para o Roteiro:**
1. **Skincade byteGenerator e o padrao correto** — yield de 32 bytes por round com incremento de currentRound. Exatamente o multi-round HMAC definido na pesquisa Doc 0. Codigo confirmado e reproduzivel
2. **4 bytes por float (nao 2)** — granularidade maxima (4.294.967.296 valores). Para 16 rows, precisa de 64 bytes = 2 rounds de HMAC. Para 8 rows, 32 bytes = 1 round
3. **Cursor pattern do Stake** — cursor global que avanca, nao nonce separado por float. Adaptaremos para simplicidade: nonce por drop (cada drop reseta cursor a 0, incrementa nonce global)
4. **Canvas willReadFrequently** — nao aplicavel (nao lemos pixels), mas batch beginPath e HiDPI scaling sim
5. **timingSafeEqual obrigatorio** — para verificacao de hash no endpoint verify

**O que teriamos feito sem a pesquisa:**
- Implementado byteGenerator com logica propria em vez do padrao Skincade testado — risco de incompatibilidade com verificadores third-party
- Usado 2 bytes por float (65536 valores) em vez de 4 bytes (4 bilhoes) — granularidade 65.000x menor
- Ignorado HiDPI scaling no Canvas — pegs borrados em telas Retina
- Comparado hashes com `===` em vez de `timingSafeEqual` — vulneravel a timing attack

-- ====================================================
-- BLACKOUT CASINO — PLINKO ENGINE (#8) — Schema SQL
-- Copiar e colar no MySQL/MariaDB. Executar em sequencia.
-- Refs: Mega Estudo P2 (5.2), Pesquisa P2 (economia)

-- ====================================================

-- ====================================================
-- TABELA 1: HISTORICO DE DROPS
-- Registra cada drop jogado com TODAS as variaveis
-- para auditoria, Provably Fair, e analytics admin
-- ====================================================
CREATE TABLE IF NOT EXISTS casino_plinko_drops (
  id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  player_id       VARCHAR(64)       NOT NULL   COMMENT 'Identifier FiveM (license:xxxxx)',
  bet_amount      DECIMAL(15,2)     NOT NULL   COMMENT 'Valor apostado em GCoin',
  risk_level      ENUM('low','medium','high') NOT NULL DEFAULT 'medium' COMMENT 'Nivel de risco escolhido',
  rows_count      TINYINT UNSIGNED  NOT NULL   DEFAULT 12 COMMENT 'Quantidade de linhas no board (8-16)',
  result_slot     TINYINT UNSIGNED  NOT NULL   COMMENT 'Indice do slot onde a bola aterrisou (0 a rows_count)',
  multiplier      DECIMAL(10,4)     NOT NULL   COMMENT 'Multiplicador do slot aterrisado (ex: 0.2, 1.0, 110.0, 1000.0)',
  payout          DECIMAL(15,2)     NOT NULL   COMMENT 'Valor pago ao jogador (bet_amount * multiplier)',
  profit          DECIMAL(15,2)     NOT NULL   COMMENT 'Lucro do jogador (payout - bet_amount, pode ser negativo)',
  server_seed_hash VARCHAR(64)      NOT NULL   COMMENT 'SHA-256 do server seed (exibido ANTES do drop)',
  client_seed     VARCHAR(64)       NOT NULL   COMMENT 'Seed fornecido ou aceito pelo jogador',
  nonce           BIGINT UNSIGNED   NOT NULL   DEFAULT 0 COMMENT 'Contador incrementado a cada drop com este par de seeds',
  server_seed     VARCHAR(64)       DEFAULT NULL COMMENT 'Server seed REVELADO apos troca de seeds (NULL enquanto ativo)',
  path            JSON              NOT NULL   COMMENT 'Array de direcoes por row. Ex: ["R","L","R","R","L"]',
  created_at      TIMESTAMP         DEFAULT CURRENT_TIMESTAMP,

  INDEX idx_plinko_player       (player_id),
  INDEX idx_plinko_created      (created_at),
  INDEX idx_plinko_multiplier   (multiplier),
  INDEX idx_plinko_risk         (risk_level),
  INDEX idx_plinko_player_date  (player_id, created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;


-- ====================================================
-- TABELA 2: CONFIGURACAO ADMIN
-- Parametros do jogo configuraveis via painel admin
-- Lidos pelo handler no mount e cacheados por 60s
-- ====================================================
CREATE TABLE IF NOT EXISTS casino_plinko_config (
  id              INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  config_key      VARCHAR(64)   NOT NULL UNIQUE COMMENT 'Chave do parametro',
  config_value    VARCHAR(255)  NOT NULL        COMMENT 'Valor do parametro',
  description     VARCHAR(255)  DEFAULT NULL    COMMENT 'Descricao para o admin',
  updated_at      TIMESTAMP     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;


-- ====================================================
-- TABELA 3: SEEDS PROVABLY FAIR (por jogador)
-- Armazena pares de seeds ativos e revelados
-- Um jogador tem 1 par ativo por vez (is_active = 1)
-- Ao rotacionar: is_active = 0, revealed_at = NOW(),
-- server_seed preenchido, novo par criado com is_active = 1
-- ====================================================
CREATE TABLE IF NOT EXISTS casino_plinko_seeds (
  id              BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  player_id       VARCHAR(64)   NOT NULL        COMMENT 'Identifier FiveM',
  server_seed     VARCHAR(64)   NOT NULL        COMMENT 'Seed secreto do servidor (64 hex chars)',
  server_seed_hash VARCHAR(64)  NOT NULL        COMMENT 'SHA-256 do server seed (publico)',
  client_seed     VARCHAR(64)   NOT NULL        COMMENT 'Seed fornecido pelo jogador',
  nonce           BIGINT UNSIGNED NOT NULL DEFAULT 0 COMMENT 'Contador de drops com este par',
  is_active       TINYINT(1)    NOT NULL DEFAULT 1 COMMENT '1 = par ativo, 0 = revelado',
  created_at      TIMESTAMP     DEFAULT CURRENT_TIMESTAMP,
  revealed_at     TIMESTAMP     NULL DEFAULT NULL COMMENT 'Quando o server seed foi revelado',

  INDEX idx_seeds_player_active (player_id, is_active),
  INDEX idx_seeds_player        (player_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;


-- ====================================================
-- SEEDS DE CONFIGURACAO — Executar apos CREATE TABLE
-- Estes INSERT definem os valores DEFAULT do jogo
-- Admin pode alterar via UPDATE sem recompilar
-- ====================================================
INSERT INTO casino_plinko_config (config_key, config_value, description) VALUES
  ('min_bet',               '10',             'Aposta minima em GCoin'),
  ('max_bet',               '50000',          'Aposta maxima em GCoin'),
  ('min_rows',              '8',              'Minimo de linhas no board'),
  ('max_rows',              '16',             'Maximo de linhas no board'),
  ('default_rows',          '12',             'Rows padrao ao abrir o jogo'),
  ('default_risk',          'medium',         'Risco padrao ao abrir o jogo'),
  ('risk_levels_enabled',   'low,medium,high','Niveis de risco habilitados (separados por virgula)'),
  ('autobet_max_drops',     '100',            'Maximo de drops por sessao de autobet'),
  ('autobet_delay_ms',      '1000',           'Intervalo minimo entre drops em autobet (ms)'),
  ('max_payout_per_drop',   '500000',         'Payout maximo por drop unico (seguranca anti-exploit)'),
  ('provably_fair_enabled', 'true',           'Habilitar sistema Provably Fair'),
  ('history_visible_count', '20',             'Quantidade de drops visiveis no historico horizontal'),
  ('rate_limit_per_second', '3',              'Maximo de drops por segundo por jogador'),
  ('maintenance_mode',      'false',          'Modo manutencao (bloqueia novos drops)'),
  ('house_edge_percent',    '1.00',           'House edge em porcentagem (referencia, nao altera logica)');


-- ====================================================
-- QUERIES ADMIN UTEIS (nao executar automaticamente)
-- Copiar e colar no cliente MySQL quando necessario
-- ====================================================

-- Total wagered e payout nas ultimas 24h
SELECT
  COUNT(*)            AS total_drops,
  SUM(bet_amount)     AS total_wagered,
  SUM(payout)         AS total_payout,
  SUM(profit) * -1    AS house_profit,
  AVG(multiplier)     AS avg_multiplier,
  MAX(multiplier)     AS max_multiplier_hit
FROM casino_plinko_drops
WHERE created_at >= NOW() - INTERVAL 24 HOUR;

-- Top 10 maiores ganhos de todos os tempos
SELECT player_id, bet_amount, multiplier, payout, risk_level, rows_count, created_at
FROM casino_plinko_drops
ORDER BY payout DESC
LIMIT 10;

-- RTP real calculado (todas as bets)
SELECT
  ROUND(SUM(payout) / SUM(bet_amount) * 100, 4) AS rtp_real_percent,
  COUNT(*) AS total_drops,
  SUM(bet_amount) AS total_wagered
FROM casino_plinko_drops;

-- RTP por nivel de risco
SELECT
  risk_level,
  COUNT(*) AS drops,
  ROUND(SUM(payout) / SUM(bet_amount) * 100, 4) AS rtp_percent,
  ROUND(AVG(multiplier), 4) AS avg_mult,
  MAX(multiplier) AS max_mult
FROM casino_plinko_drops
GROUP BY risk_level;

-- RTP por quantidade de rows
SELECT
  rows_count,
  COUNT(*) AS drops,
  ROUND(SUM(payout) / SUM(bet_amount) * 100, 4) AS rtp_percent,
  ROUND(AVG(multiplier), 4) AS avg_mult
FROM casino_plinko_drops
GROUP BY rows_count
ORDER BY rows_count;

-- Jogadores mais ativos (por volume de drops)
SELECT
  player_id,
  COUNT(*)        AS drops,
  SUM(bet_amount) AS total_wagered,
  SUM(payout)     AS total_payout,
  SUM(profit)     AS total_profit,
  MAX(multiplier) AS best_mult
FROM casino_plinko_drops
GROUP BY player_id
ORDER BY drops DESC
LIMIT 20;

-- Drops com multiplicador extreme (>= 50x) nas ultimas 7 dias
SELECT player_id, bet_amount, multiplier, payout, risk_level, rows_count, created_at
FROM casino_plinko_drops
WHERE multiplier >= 50
  AND created_at >= NOW() - INTERVAL 7 DAY
ORDER BY multiplier DESC;

-- Verificar se algum jogador tem mais de 1 seed ativo (erro de logica)
SELECT player_id, COUNT(*) AS active_seeds
FROM casino_plinko_seeds
WHERE is_active = 1
GROUP BY player_id
HAVING active_seeds > 1;
