# SECAO 3 — TIMELINES DE ANIMACAO (ms)

Sequencia completa de um drop do Plinko, do clique no APOSTAR ate o retorno ao estado IDLE.

## 3.1 Timeline — Drop Normal (multiplicador < 50x)

```
t=0ms        Jogador clica APOSTAR
             → fetchNui('plinko:bet') enviado ao server
             → Estado muda para DROPPING
             → Controles desabilitam (opacity 0.4, grayscale)
             → Botao muda texto para "AGUARDE..."
             → Bola aparece no topo do board com pulse "preparing"
             → Som: button_click.mp3 (50ms)

t=50ms       Bola inicia animacao "preparing" (pulsacao scale 1→1.1→1)
             → Esperando resposta do server (50-200ms de latencia)

t=150ms      Resposta do server chega (media)
             → Path recebido: ["R","L","R","R","L","R","L","L","R","R","L","R"]
             → Bola para de pulsar, inicia queda
             → Som: ball_drop.mp3 (300ms, whoosh suave)

t=270ms      Row 1: bola toca primeiro peg
             → Peg flash #FFD700 (150ms alpha decay)
             → Som: peg_hit_soft.mp3 (playbackRate 1.00)
             → Trail comeca a registrar posicoes

t=390ms      Row 2: bola toca segundo peg
             → Peg flash
             → Som: peg_hit_soft.mp3 (playbackRate 1.01)

t=510ms      Row 3: peg_hit_hard.mp3 (playbackRate 1.02)
             ...
             (cada row: ~120ms + jitter 40ms = 120-160ms por segmento)

t=1590ms     Row 12 (ultima para 12 rows): bola toca ultimo peg
             → Peg flash
             → Som: peg_hit_hard.mp3 (playbackRate 1.12 — pitch mais agudo)
             → Progressive pitch cria sensacao de aceleracao e tensao crescente

t=1710ms     Bola aterrisa no slot
             → Estado muda para LANDED
             → Slot pulsa com cor do tier (animation slotPulse)
             → Som: slot_land_low/medium/high.mp3 (conforme tier do multiplicador)

t=1900ms     Overlay de resultado aparece
             → Multiplicador: scale 0→1.6→1 (spring 500ms)
             → Payout: float-up fade "+G$X" (2s)
             → Contagem animada: 0→valor (rAF, 1.5s, ease-out cubic)
             → Som: win_counter_tick.mp3 (throttled cada 50ms durante contagem)

t=4400ms     Resultado persiste (2.5s de visibilidade)

t=4400ms     Overlay dismiss (AnimatePresence exit)
             → Estado muda para IDLE
             → Controles reativam (opacity 1, interativeis)
             → Botao volta ao texto "APOSTAR" com pulse
             → Historico badges atualiza (novo badge entra com stagger)
             → Board slot para de pulsar

TOTAL DROP NORMAL (12 rows): ~4.4 segundos
TOTAL DROP NORMAL (16 rows): ~5.2 segundos (4 rows extras × ~160ms)
TOTAL DROP NORMAL (8 rows): ~3.6 segundos
```
