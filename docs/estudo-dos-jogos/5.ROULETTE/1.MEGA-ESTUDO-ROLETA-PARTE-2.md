## Problema 1: Sincronizacao Animacao vs Resultado Server-Side

**Sintoma:** O resultado e determinado server-side ANTES da animacao. O client precisa animar a roda parando EXATAMENTE no numero correto. Se o calculo do angulo estiver errado, a roda para num numero diferente do resultado real.

**Causa tecnica:** A roda tem 37 pockets em posicoes angulares fixas. O angulo de cada numero depende da sequencia EXATA na roda European (0,32,15,19,4,21,2,25,17,34,6,27,13,36,11,30,8,23,10,5,24,16,33,1,20,14,31,9,22,18,29,7,28,12,35,3,26). O calculo precisa mapear numero -> indice na sequencia -> angulo.

**Solucao com codigo real:**

```javascript
// Sequencia European (sentido horario a partir do 0)
const EUROPEAN_SEQUENCE = [0,32,15,19,4,21,2,25,17,34,6,27,13,36,11,30,8,23,10,5,24,16,33,1,20,14,31,9,22,18,29,7,28,12,35,3,26];

const POCKET_ANGLE = 360 / 37; // ~9.73 graus por pocket

function calcularAnguloParada(numero) {
  const indice = EUROPEAN_SEQUENCE.indexOf(numero);
  if (indice === -1) return 0;
  // Angulo base do pocket (centro do pocket)
  const anguloBase = indice * POCKET_ANGLE;
  // Variacao aleatoria dentro do pocket (+/- 3 graus) para parecer natural
  const variacao = (Math.random() - 0.5) * (POCKET_ANGLE * 0.6);
  return anguloBase + variacao;
}

function calcularRotacaoTotal(numero) {
  const anguloParada = calcularAnguloParada(numero);
  // 5-8 voltas completas + angulo final
  const voltasCompletas = (5 + Math.floor(Math.random() * 4)) * 360;
  // Roda gira sentido horario, bola sentido oposto
  // O angulo final e onde a bola PARA (fixo no topo)
  // Entao a roda precisa rotacionar ate o pocket ficar no topo
  return voltasCompletas + (360 - anguloParada);
}
```

**Impacto:** ~20 linhas. Critico acertar a sequencia. Testar com TODOS os 37 numeros antes de deploy.

## Problema 2: Performance de Animacao no NUI (GPU Compartilhada)

**Sintoma:** FiveM compartilha GPU entre o jogo 3D e o NUI (Chromium). Animacoes pesadas podem causar stutter.

**Causa tecnica:** CSS transform rotate e GPU-accelerated (composited layer). Porem, se combinado com backdrop-filter blur + particulas Canvas + multiplos box-shadow, pode saturar a GPU em maquinas mais fracas.

**Solucao multi-camada:**
1. Roda: APENAS `transform: rotate()` na imagem PNG. GPU-accelerated, performante.
2. Bola: APENAS `transform: rotate() + translateX()`. Sem shadow durante animacao.
3. Lightning effects: `will-change: opacity, transform` nos elementos animados. Remover apos animacao (evitar composited layers permanentes).
4. Backdrop-filter: usar APENAS em modais (resultado, PF). NAO na mesa durante jogo.
5. Particulas: limitar a 30 particulas max. Canvas `requestAnimationFrame` com throttle a 30fps durante spin (economiza GPU).
6. Box-shadow: durante spin ativo, simplificar shadows (1 camada em vez de 5). Restaurar apos spin.

**Codigo de otimizacao:**
```javascript
// Hook para detectar se esta no FiveM NUI
function useIsNUI() {
  return typeof window !== 'undefined' && window.invokeNative !== undefined;
}

// Reduzir efeitos se NUI
const isNUI = useIsNUI();
const particleCount = isNUI ? 20 : 40;
const shadowLayers = isNUI ? 1 : 5;
const blurAmount = isNUI ? '8px' : '16px';
```

**Impacto:** ~15 linhas. Melhoria significativa em GPUs fracas (GTX 1050 tier).

## Problema 3: Race Condition entre Apostas e Spin

**Sintoma:** Jogador tenta apostar DEPOIS do spin ser iniciado. Ou dois spins sao disparados simultaneamente.

**Solucao:**
```javascript
// Server handler
let jogadoresEmSpin = new Set();

function handleSpin(jogadorId, apostas) {
  if (jogadoresEmSpin.has(jogadorId)) {
    return { erro: 'Spin ja em andamento' };
  }
  jogadoresEmSpin.add(jogadorId);

  // ... processar spin ...

  // Apos resultado calculado e enviado
  setTimeout(() => jogadoresEmSpin.delete(jogadorId), 1000);
}
```

Client-side: desabilitar botao GIRAR + todos os bet areas durante spin. Re-habilitar apos resultado.

**Impacto:** ~10 linhas server + ~5 linhas client.

## Problema 4: Workaround .png.png no goldPath

**Sintoma:** Arquivo `5.IMAGEM-DOURADA-ROULETTE.png.png` tem extensao dupla. O `lib/assets.ts` linha ~70 tem condicional especial para IDs 5 e 13.

**Solucao:** Renomear o arquivo para `5.IMAGEM-DOURADA-ROULETTE.png` (remover extensao extra) e remover o workaround no codigo. Edicao cirurgica em `lib/assets.ts`:

```javascript
// ANTES (linha ~70 de lib/assets.ts):
const ext = (id === 5 || id === 13) ? ".png.png" : ".png";

// DEPOIS:
const ext = ".png";
```

E renomear o arquivo no filesystem. **Impacto:** 1 linha de codigo + 1 rename de arquivo.
