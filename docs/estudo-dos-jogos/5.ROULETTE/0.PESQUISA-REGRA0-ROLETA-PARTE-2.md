## 3.2 Analise de Abordagens Tecnicas

### ABORDAGEM A - Imagem PNG da Roda + CSS Transform Rotate

**Como funciona:** A roda e uma IMAGEM estatica (PNG/WebP). A animacao e feita rotacionando a imagem via CSS transform: rotate(Xdeg). O numero final determina o angulo de parada. A bola e um elemento separado com animacao oposta.

**Vantagens:**
- Visual PERFEITO - a imagem e criada por designer/AI com qualidade ilimitada (metalico, reflexos, texturas)
- Performance excelente - GPU-accelerated CSS transform
- Simples de implementar - sem calculo de geometria de pockets
- Funciona em QUALQUER browser/NUI sem problemas
- A roda do Blackout Casino pode ter estetica premium (ouro, verde neon, brilhos) impossivel de replicar com Canvas/SVG

**Desvantagens:**
- Imagem grande se alta resolucao (~500KB-2MB)
- Nao e dinamicamente customizavel (precisa nova imagem para layout diferente)

**Para o Blackout Casino:** RECOMENDADA. A identidade visual premium exige uma roda com aparencia de casino real - metalica, dourada, com profundidade 3D. Isso so se consegue com imagem de designer. CSS rotate e trivial e performante.

### ABORDAGEM B - SVG Gerado Programaticamente

**Como funciona:** A roda e construida via SVG com arcs (caminhos curvos), textos ao longo de paths, e cores programaticas. Rotacao via CSS transform ou SVG animate.

**Vantagens:**
- Totalmente customizavel via codigo (cores, numeros, tamanhos)
- Leve em tamanho de arquivo
- Escalavel sem perda de qualidade

**Desvantagens:**
- Dificil atingir visual premium (metalico, reflexos, profundidade 3D)
- Safari tem PROBLEMAS conhecidos com SVG animado (fonte: missingdice DEV.to) - fix: animar div wrapper, nao SVG
- Complexo de implementar corretamente (textPath, arc calculations)

**Para o Blackout Casino:** NAO recomendada como abordagem primaria. Pode ser usada como fallback ou para a mesa de apostas, mas a RODA deve ser imagem premium.

### ABORDAGEM C - Canvas 2D

**Como funciona:** Desenho da roda via Canvas API. Arcs, fills, strokes. Animacao via requestAnimationFrame.

**Vantagens:**
- Controle total pixel-a-pixel
- Pode combinar com imagem de fundo (drawImage)

**Desvantagens:**
- NAO e GPU-accelerated nativamente (pode ser lento em NUI)
- Complexidade alta para visual premium
- Nao escala bem responsivamente (precisa redraw manual)

**Para o Blackout Casino:** NAO recomendada. CSS transform rotate sobre imagem PNG e mais simples e performante.

### ABORDAGEM D - Framer Motion Rotate

**Como funciona:** Usar `motion.div` com `animate={{ rotate: finalAngle }}` e `transition={{ duration, ease }}`. A roda (imagem ou SVG) e filha do motion.div.

**Vantagens:**
- Ja esta no projeto (Framer Motion 11)
- Easing sofisticado nativo (cubic-bezier, spring, tween)
- Controle programatico do angulo final (server determina)
- Integracao perfeita com AnimatePresence, whileHover, etc.
- Pode animar bola e roda em paralelo com timings diferentes

**Desvantagens:**
- Nenhuma significativa para este caso

**Para o Blackout Casino:** RECOMENDADA como metodo de animacao. Framer Motion rotate sobre imagem PNG da roda. Bola animada separadamente com timing offset.

## 3.3 Decisao Tecnologica: Stack para a Roleta

| Componente | Tecnologia | Justificativa |
|------------|-----------|---------------|
| **Roda visual** | Imagem PNG/WebP (gerada por AI/designer) | Visual premium impossivel via SVG/Canvas |
| **Animacao da roda** | Framer Motion `animate={{ rotate }}` | Ja no projeto. Easing nativo. Controle do angulo final |
| **Bola** | Div/imagem com Framer Motion animate | Orbita reversa a roda + queda final no pocket |
| **Mesa de apostas** | Grid CSS (DOM puro) | Tabela de numeros com click-to-bet. Responsive |
| **Chips** | Imagens PNG posicionadas via absolute | Visual premium. Nao precisa ser dinamico |
| **Racetrack** | SVG estatico com areas clicaveis | Voisins/Tiers/Orphelins como sections |
| **Historico** | DOM list com scroll | Ultimos 50-100 resultados |
| **Lightning effects** | CSS @keyframes + Framer Motion | Raios, flashes, multiplicadores animados |
| **Resultado server-side** | Provably Fair (HMAC-SHA256) | Numero determinado ANTES da animacao |
| **Comunicacao** | fetchNui (client) -> TriggerServerEvent (Lua) -> handler.js (server) | Padrao do projeto |

## 3.4 Fluxo Tecnico de um Spin

```
1. Jogador posiciona apostas (client-side, state React)
2. Clica GIRAR
3. Client envia apostas via fetchNui('roulette:spin', { bets, totalBet })
4. Lua client repassa: TriggerServerEvent('casino:roulette:spin', bets)
5. Server handler:
   a. Valida saldo (totalBet <= saldo do jogador)
   b. Debita saldo
   c. Gera resultado Provably Fair:
      - Combina serverSeed + clientSeed + ':' + nonce
      - HMAC-SHA256 -> hex string
      - Primeiro 8 chars -> int -> modulo 37 -> numero 0-36
   d. [Lightning] Gera 1-5 Lucky Numbers + multiplicadores
   e. Calcula ganhos com base nas apostas vs numero vencedor
   f. Credita ganhos
   g. Salva no MySQL (historico, audit)
   h. Retorna: { number, color, luckyNumbers?, winAmount, newBalance }
6. Client recebe resultado:
   a. Calcula angulo da roda para parar no numero certo
   b. [Lightning] Anima Lucky Numbers com efeitos de raio
   c. Inicia animacao Framer Motion: roda gira N voltas + para no angulo
   d. Bola anima em orbita reversa + queda
   e. Apos parada: overlay de resultado (ganho/perda)
   f. Atualiza historico, hot/cold, saldo
```
