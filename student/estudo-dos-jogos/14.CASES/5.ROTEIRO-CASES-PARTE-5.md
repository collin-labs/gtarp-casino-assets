**DEPOIS (substituir com preload + volume control + mute toggle):**
```tsx
export function useCasesSounds() {
  const soundsRef = useRef<Record<CasesSoundKey, Howl> | null>(null);
  const mutedRef = useRef(false);

  useEffect(() => {
    soundsRef.current = preloadSounds();
    return () => {
      if (soundsRef.current) {
        Object.values(soundsRef.current).forEach(howl => howl.unload());
      }
    };
  }, []);

  const play = useCallback((key: CasesSoundKey) => {
    if (mutedRef.current || !soundsRef.current) return;
    const howl = soundsRef.current[key];
    if (howl) {
      howl.stop(); // parar instancia anterior se estiver tocando
      howl.play();
    }
  }, []);

  const stop = useCallback((key: CasesSoundKey) => {
    if (!soundsRef.current) return;
    soundsRef.current[key]?.stop();
  }, []);

  const setVolume = useCallback((volume: number) => {
    if (!soundsRef.current) return;
    const clamped = Math.max(0, Math.min(1, volume));
    Object.values(soundsRef.current).forEach(howl => howl.volume(clamped));
  }, []);

  const toggleMute = useCallback(() => {
    mutedRef.current = !mutedRef.current;
    if (soundsRef.current) {
      Object.values(soundsRef.current).forEach(howl => howl.mute(mutedRef.current));
    }
    return mutedRef.current;
  }, []);

  return { play, stop, setVolume, toggleMute, isMuted: () => mutedRef.current };
}
```

**Motivo:** SoundManagerCases original tem apenas 3 sons mapeados e carrega sob demanda (lazy). O novo precarrega todos 18 sons no mount (evita delay no primeiro play), tem volume control global, mute toggle, e tipagem forte via CasesSoundKey.

```typescript
// ====================================================

import { useEffect } from 'react';

export function useEscClose(isOpen: boolean, onClose: () => void) {
  useEffect(() => {
    if (!isOpen) return;

    const handler = (e: KeyboardEvent) => {
      if (e.key === 'Escape') {
        e.preventDefault();
        onClose();
      }
    };

    window.addEventListener('keydown', handler);
    return () => window.removeEventListener('keydown', handler);
  }, [isOpen, onClose]);
}
```

## Provably Fair — Resumo Tecnico Final (5.13)

```
ALGORITMO: HMAC-SHA256 (padrao Stake.com, confirmado pesquisa X0)
FORMULA:   createHmac('sha256', serverSeed).update(clientSeed:nonce:0).digest()
           4 primeiros bytes → unsigned int32 → roll 0-999999

FLUXO:
1. Jogador abre Cases → init_session cria sessao com server_seed + server_seed_hash
2. Server retorna server_seed_hash (publicado ANTES de qualquer aposta)
3. Jogador pode editar client_seed (update_client_seed)
4. Cada abertura: roll = HMAC(serverSeed, clientSeed:nonce:0), nonce++
5. Jogador pode revelar seed (reveal_seed) → seed revelado, sessao encerrada, nova criada
6. Jogador pode verificar (verify) → recalcula roll com seeds, compara hash

SEGURANCA:
- Server seed: crypto.randomBytes(32) — 256 bits entropia
- Hash publicado ANTES: SHA-256 do server seed
- Nonce incrementado atomicamente no DB (sem Math.random)
- Race condition prevenida por transacao atomica no oxmysql
- Seed revelado SOMENTE apos jogador solicitar — nao automatico
```
