## 2.1 Formatos de Case Opening Analisados

### Formato 1: Abertura Solo Classica (CS2/Valve)
O jogador compra ou recebe uma caixa, paga uma "chave" para abrir, e uma animacao de strip horizontal revela o item sorteado. A strip mostra itens passando rapidamente da direita para a esquerda, desacelerando ate parar no item vencedor. O item e revelado com efeito de glow baseado na raridade — azul para Common, roxo para Uncommon, rosa para Classified, vermelho para Covert, dourado para Special (knife/gloves). A emocao vem da desaceleracao: o jogador ve itens raros passando e quase parando, criando tensao maxima antes do reveal final.

No CS2, as probabilidades sao fixas e publicadas: Mil-Spec (azul) 79.92%, Restricted (roxo) 15.98%, Classified (rosa) 3.2%, Covert (vermelho) 0.64%, Special (dourado) 0.26%. Cada abertura e um evento independente — nao ha acumulacao de sorte. A animacao visual da strip NAO determina o resultado; o resultado e calculado server-side no momento da abertura, e a strip e apenas uma representacao visual que sempre para no item ja determinado.

### Formato 2: Fast Open / Quick Open
Alternativa oferecida por sites third-party (CSGOEmpire, KeyDrop, Clash.gg): pula a animacao de strip e revela o item imediatamente com um flash de cor da raridade. Atende jogadores que abrem muitas caixas em sequencia e preferem velocidade. Implementacao simples: mesma logica server-side, apenas sem a animacao de desaceleracao.

### Formato 3: Case Battle (PvP)
Modo competitivo onde 2-4 jogadores abrem caixas simultaneamente. Cada jogador abre a mesma quantidade de caixas, e quem acumular o maior valor total de itens ganha TODOS os itens de todos os participantes. No empate, uma roleta (wheel spin) determina o vencedor. Variante "Split Mode": os ganhos sao distribuidos igualmente entre os participantes. O formato PvP transforma case opening de experiencia solo em espetaculo social com torcida e rivalidade.

### Formato 4: Caixa Diaria Gratuita (Daily Free)
Mecanica de retencao/engajamento: o jogador ganha 1 caixa gratis por dia (ou por periodo). Os itens sao de menor valor (GCoin pequeno, itens cosmeticos baratos), mas a mecanica cria habito de login diario. CSGOEmpire oferece free daily case com recompensas de ate $1.445 (extremamente raro), o que gera esperanca e retorno constante.

### Formato 5: Genesis Terminal (Inovacao CS2 Armory 2024)
Mecanica alternativa onde um "Arms Dealer" apresenta 5 ofertas sequenciais a precos diferentes. O jogador pode aceitar uma oferta ou recusar todas. Nao e case opening classico, mas mostra inovacao de UX na area — o elemento de escolha humana misturado com randomizacao.

# 4. EIXO 2 — TECNOLOGIA

## 4.1 Opcoes Tecnicas Comparadas

### Opcao A: Framer Motion (animate x + translateX)
- Strip como div longa com itens posicionados inline
- `animate={{ x: -targetPosition }}` com `transition={{ duration: 4, ease: [0.15, 0, 0.05, 1] }}`
- Desaceleracao customizavel via cubic-bezier
- JA INSTALADO no projeto (framer-motion 11.18.x)
- Performance: CSS transform via compositor thread, 60fps em NUI Chromium
- Prioridade: alta (padrao do projeto, consistente com outros jogos)

### Opcao B: CSS @keyframes com translateX
- @keyframes slideStrip que vai de translateX(0) ate translateX(-target)
- animation-timing-function: cubic-bezier(0.15, 0, 0.05, 1)
- Sem dependencia adicional, CSS puro
- Desvantagem: menos controle dinamico (target position calculado server-side, precisa injectar no CSS)
- Desvantagem: sem onAnimationComplete callback confiavel cross-browser em NUI

### Opcao C: GSAP ScrollTrigger
- gsap.to(strip, { x: -target, duration: 4, ease: "power4.out" })
- GSAP nao esta instalado no projeto
- Overkill para uma unica animacao de strip
- Adicionaria ~30KB de bundle

## 4.2 Decisao Tecnica

**Framer Motion (Opcao A) escolhido.** Razoes:

1. JA INSTALADO — zero dependencia nova. Import direto de "framer-motion"
2. Consistente com o resto do projeto (Slots, Roulette, Cases existente usam Framer Motion)
3. Controle dinamico: targetPosition calculado server-side, passado como prop, animate responde
