# O QUE NAO DEVE TER — LISTA PROIBIDO (18 ITENS)

```
1.  NUNCA fundo branco, cinza claro ou qualquer cor clara — SOMENTE dark mode
    (#0A0A0A base). Curva neon SEMPRE sobre fundo escuro. Grid NUNCA visivel
    demais — max rgba(212,168,67,0.05).

2.  NUNCA usar Three.js, react-three-fiber ou WebGL para o Crash —
    Canvas 2D nativo e suficiente para 1 curva + grid + particulas.
    WebGL e overkill e pode causar problemas no CEF do FiveM.

3.  NUNCA font sans-serif em titulos do jogo — SEMPRE Cinzel para
    "CRASH", "APOSTAR", "CASHOUT", "VOCE SACOU", "CRASHED", "PROVABLY FAIR".

4.  NUNCA px fixo para fontes, padding ou tamanhos de layout —
    SEMPRE clamp(min, vw, max). Errado: font-size: 16px.
    Correto: font-size: clamp(11px, 1.4vw, 16px).

5.  NUNCA border-radius > 18px em containers — Cards: 10-14px,
    Botoes: 8-10px, Badges historico: 3-5px.

6.  NUNCA single-layer box-shadow — SEMPRE multi-layer (minimo 2 camadas):
    ambient + glow colorido. Botao CASHOUT: 3 camadas no pulse.

7.  NUNCA titulo sem text-shadow glow dourado:
    text-shadow: 0 0 12px rgba(255,215,0,0.6), 0 2px 6px rgba(0,0,0,1).

8.  NUNCA botao sem 4 estados: normal, hover, active, disabled.
    Disabled: opacity 0.35, cursor not-allowed, pointer-events none.

9.  NUNCA elemento interativo sem tooltip — 14 tooltips mapeados acima.
    Incluindo: APOSTAR, CASHOUT, inputs, atalhos, toggles, icones, saldo.

10. NUNCA valores GCoin em cor diferente de #00E676 (verde neon).
    text-shadow: 0 0 8px rgba(0,230,118,0.4) obrigatorio.
    Valores negativos (derrota): cor fica rgba(212,168,67,0.6), NAO vermelho
    (derrota e discreta, nao agressiva — ver Tela 4B).

11. NUNCA shadowBlur no Canvas dentro do requestAnimationFrame loop —
    shadowBlur e O(n*pixels) e destroi FPS no NUI. Glow da curva feito via
    CSS box-shadow no container do canvas, NAO ctx.shadowBlur.
    Glow do ponto final: radialGradient fake (circulo semi-transparente maior).

12. NUNCA enviar crash point ao client ANTES do crash acontecer —
    client conhece apenas o hash do server seed. O crash point real
    so e revelado DEPOIS que o round termina. Se enviado antes, jogador
    poderia calcular e sacar exatamente antes do crash.

13. NUNCA mostrar overlay de resultado SEM a animacao completa —
    sequencia obrigatoria: (1) curva muda para vermelho, (2) flash canvas,
    (3) shake container, (4) particulas disparam, (5) SO ENTAO overlay aparece.
    Overlay antes da animacao quebra a dramaticidade.

14. NUNCA rolling counter sem easing — o premio deve desacelerar nos
    ultimos 20%: ease-out quint (1 - Math.pow(1 - t, 5)).
    Counter linear parece placeholder.

15. NUNCA atualizar saldo visivel antes da animacao terminar —
    saldo atualiza DEPOIS de: rolling counter terminar + confetti
    (vitoria) ou shake terminar (derrota). Nunca no momento do response.

16. NUNCA botao CASHOUT visivel quando jogador NAO apostou ou JA sacou —
    CASHOUT so aparece durante RISING quando status da aposta e "active".
    Nos outros estados: oculto (display none, NAO opacity 0).

17. NUNCA tom ascendente continuo apos o crash — o oscillator DEVE ser
    parado com exponentialRampToValueAtTime ANTES da explosao soar.
    Sequencia: (1) gain ramp para 0.001 em 150ms, (2) osc.stop() apos 200ms,
    (3) crash_explode toca. Se tom e explosao sobrepem, o audio fica sujo.

18. NUNCA reescrever componentes inteiros ao adicionar features — edicoes
    cirurgicas str_replace (REGRA X6). Novas features sao camadas adicionadas
    sobre componentes existentes, nunca substituicao total.
```
