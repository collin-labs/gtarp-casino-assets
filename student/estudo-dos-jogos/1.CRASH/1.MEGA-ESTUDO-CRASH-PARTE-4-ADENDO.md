# 1. PROBLEMAS TECNICOS COMPLEMENTARES

## Problema 6: Canvas 2D Performance em NUI Chromium 93

O NUI do FiveM usa Chromium 93 (2021) com limitacoes em GPU compositing.
Canvas 2D com requestAnimationFrame funciona bem para a curva do Crash,
mas precisa de cuidado com:
- clearRect a cada frame (evitar ghosting na curva)
- lineWidth responsivo via devicePixelRatio
- Anti-aliasing nativo do Canvas (ctx.imageSmoothingEnabled = true)
- Fallback: se performance < 30fps, reduzir pontos no path (sampling)

**Solucao:** Limitar o path a max 500 pontos (1 ponto por frame a 60fps = 8.3s
de curva visivel). Apos 500 pontos, remover os mais antigos (sliding window).

## Problema 7: Hash Chain Provably Fair — Integridade

O Crash usa hash chain (nao single hash como outros jogos). Cada rodada
revela o hash da rodada ANTERIOR, formando cadeia verificavel:

```
hash_1000000 = SHA256("server_seed")
hash_999999 = SHA256(hash_1000000)
...
hash_1 = SHA256(hash_2)
```

O crash point eh calculado do hash + salt:
```
hmac = HMAC_SHA256(hash_n, salt)
crash_point = max(1, (2^52 / (h + 1)) * (1 - house_edge))
```

**Risco:** Se a hash chain for comprometida, TODOS os crash points futuros
sao previsiveis. Mitigacao: salt publico (block hash Bitcoin/Ethereum) que
nao pode ser previsto pelo operador.

## Problema 8: Concorrencia Multi-jogador no FiveM

Crash eh o unico jogo onde TODOS os jogadores compartilham a MESMA rodada.
No FiveM, isso significa:
- Server emite evento de inicio para TODOS os NUIs simultaneamente
- Cada jogador tem latencia diferente (NUI local vs server remoto)
- Cashout deve ser validado server-side com timestamp (nao client-side)
- Se jogador faz cashout a 2.50x mas server ja esta em 2.51x, usar server value

**Solucao:** Timestamp server em CADA bet e cashout. Delta max 200ms de tolerancia.

# 2. FEATURES COMPLEMENTARES

## Feature 25: Multiplier Tier Visual System

5 tiers visuais baseados no multiplicador atual:
- Tier 1 (1.00x - 1.99x): Verde #00E676, curva fina, som neutro
- Tier 2 (2.00x - 4.99x): Amarelo #FFD700, curva media, som tension
- Tier 3 (5.00x - 9.99x): Laranja #FF6D00, curva grossa, som high tension
- Tier 4 (10.00x - 99.99x): Vermelho #FF1744, curva pulsante, som epic
- Tier 5 (100.00x+): Roxo #AA00FF, curva com particulas, som legendary

Cada tier muda: cor da curva Canvas, cor do multiplicador, intensidade do glow,
velocidade do oscillator no som, cor do fundo (gradient sutil).

## Feature 26: Feed Lateral com Animacao

Feed lateral mostra apostas de outros jogadores em tempo real:
- Novo bet: slide-in da direita com opacity 0→1
- Cashout: badge verde com multiplicador, flash sutil
- Bust: badge vermelho com X, opacity reduzida
- Max 50 entries, auto-scroll com smooth behavior
- Cada entry com player name (truncado 12 chars), bet amount, resultado

## Feature 27: Sound Design por Estado

Mapeamento de sons por estado do jogo:
- WAITING: tick metronomo a cada segundo do countdown
- RISING: tom ascendente continuo (oscillator frequency sobe com multiplicador)
- CASHOUT: coin cascade + "ka-ching" metalico
- CRASHED: explosao grave + vidro quebrando + reverb
- BIG WIN (10x+): fanfare curta + aplausos
- MEGA WIN (100x+): fanfare longa + bass drop
