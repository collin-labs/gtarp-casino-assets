## 6.1 str_replace BlackoutCasino.tsx — LazyMotion wrapper

O Framer Motion importa ~30KB por default. Com `LazyMotion` + `domAnimation`, reduz para ~15KB carregando apenas features usadas (no layout animations, no drag — so animate/exit/whileHover).

**ANTES (no topo do JSX do BlackoutCasinoInner, logo apos o return):**
```tsx
    <>
      {/* CAMADA 0+1: OVERLAY ESCURO (fullscreen) */}
      <motion.div
```

**DEPOIS (wrapper LazyMotion envolvendo tudo):**
```tsx
    <LazyMotion features={domAnimation}>
      {/* CAMADA 0+1: OVERLAY ESCURO (fullscreen) */}
      <m.div
```

**Import adicional necessario:**
```tsx
import { LazyMotion, domAnimation, m } from "framer-motion";
// Nota: substituir <motion.div> por <m.div> em TODOS os componentes
// OU manter <motion.div> e usar LazyMotion features={domMax} (30kb, mas sem tree-shaking)
// Recomendacao: manter <motion.div> com domMax por compatibilidade com layoutId
```

**Decisao tecnica:** O Dock usa `layoutId` que requer `domMax` (nao `domAnimation`). Portanto, usar `LazyMotion features={domMax}` para manter compatibilidade. A economia e menor (~5KB) mas a arquitetura fica pronta para otimizar quando layoutId nao for mais necessario.

```tsx

```

**DEPOIS (adicionar useEffect de ESC logo antes do getSaldo):**
```tsx
  // handler ESC: fecha jogo → modal → painel (nessa ordem de prioridade)
  useEffect(() => {
    const handleKeyDown = (e: KeyboardEvent) => {
      if (e.key === "Escape") {
        if (activeGame) {
          closeGame();
          sound.play("game_back");
        } else if (selectedGame) {
          setSelectedGame(null);
        } else {
          // fechar painel (notificar Lua)
          api.fecharPainel();
          sound.play("panel_close");
        }
      }
    };
    window.addEventListener("keydown", handleKeyDown);
    return () => window.removeEventListener("keydown", handleKeyDown);
  }, [activeGame, selectedGame, closeGame, setSelectedGame, api, sound]);

  useEffect(() => {
    api.getSaldo().then((res) => {
```

**Motivo:** No FiveM, ESC e o botao universal de "fechar NUI". O handler respeita hierarquia: primeiro fecha jogo ativo, depois modal, depois o painel inteiro. Sem isso, ESC faz NADA e jogador fica preso.
