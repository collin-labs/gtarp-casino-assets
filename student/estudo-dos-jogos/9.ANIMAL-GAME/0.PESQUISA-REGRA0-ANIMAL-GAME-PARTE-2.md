## 3.1 Componente A - Grid de 25 Animais (Selecao de Apostas)

### Opcao A1 - Grid CSS com Framer Motion (Recomendada)

**Como funciona:**
- Grid 5x5 usando CSS Grid (`grid-template-columns: repeat(5, 1fr)`)
- Cada animal e um card clicavel com imagem, nome e numero do grupo
- Framer Motion para: entrada staggered, hover scale+glow, selecao com borda neon, deselecao com fade
- Imagens ja existem: `public/assets/bicho/1.png` a `25.png`

**Implementacao no Blackout Casino:**
- Container: CSS Grid responsivo com `clamp()` para gap e card size
- Card normal: borda dourada rgba(212,168,67,0.3), fundo rgba(5,5,5,0.95)
- Card hover: borda intensifica, scale 1.05, spotlight verde segue mouse
- Card selecionado: borda verde neon #00E676, glow pulsante, badge com numero
- Card desabilitado: opacity 0.4, grayscale, cursor not-allowed (quando max selecao atingido)
- Stagger: delay = index * 0.03s na entrada

**Codigo conceitual (Framer Motion):**
```
<motion.div
  whileHover={{ scale: 1.05, borderColor: "rgba(0,230,118,0.6)" }}

```

**Complexidade:** Baixa. Grid CSS + Framer Motion ja sao padrao no projeto.

## 3.2 Componente B - Animacao de Sorteio (Reveal dos 5 Premios)

### Opcao B1 - Box Reveal com Framer Motion (Recomendada)

Inspirado na BGaming: 5 caixas/esferas douradas que abrem sequencialmente, revelando o animal sorteado dentro.

**Sequencia de animacao:**
1. 5 caixas douradas fechadas aparecem em linha horizontal
2. Caixa 1 treme levemente (anticipation: 0.5s)
3. Caixa 1 abre: tampa sobe + gira (Framer Motion rotateX + y), animal surge de dentro com scale 0 -> 1 + glow
4. Pausa dramatica (0.8s) — nome do animal aparece abaixo com text glow
5. Repete para caixas 2, 3, 4, 5 com delay crescente
6. Se jogador acertou: confetti dourado + flash verde + som de vitoria
7. Se errou: cards selecionados ficam com X vermelho sutil

**Implementacao tecnica:**
- AnimatePresence para cada caixa (mount/unmount)
- Variantes Framer Motion: closed -> shaking -> opening -> revealed
- useAnimate hook para sequencia imperativa (await entre cada caixa)
- Timing total: ~6-8 segundos para os 5 premios (nao pode ser rapido demais)

**Codigo conceitual:**
```
const [scope, animate] = useAnimate();

async function revealSequence(results) {
  for (let i = 0; i < 5; i++) {
    // Shake
    await animate(`#box-${i}`, { x: [-3, 3, -3, 0] }, { duration: 0.4 });
    // Open lid
    await animate(`#lid-${i}`, { rotateX: -120, y: -30, opacity: 0 }, { duration: 0.5 });
    // Reveal animal
    await animate(`#animal-${i}`, { scale: [0, 1.2, 1], opacity: 1 }, { duration: 0.4 });
    // Show name
    await animate(`#name-${i}`, { opacity: 1, y: [10, 0] }, { duration: 0.3 });
    // Pause between boxes
    await new Promise(r => setTimeout(r, 800));
  }
}
```

**Complexidade:** Media. A sequencia precisa de timing cuidadoso para criar tensao.

## 3.4 Componente D - Provably Fair (HMAC_SHA256)

**Descoberta da pesquisa:** O padrao da industria em 2025-2026 para jogos verificaveis e:
```
Result = HMAC_SHA256(server_seed, client_seed + ":" + nonce)
```

**Para o Jogo do Bicho especificamente:**
- server_seed: string aleatoria gerada pelo server antes do jogo
- client_seed: string fornecida pelo jogador (ou gerada automaticamente)
- nonce: contador incrementado a cada rodada
- O HMAC gera um hash de 64 chars hex
- Desse hash, extraimos 5 numeros de 0-9999 (os 5 milhares do sorteio)
- De cada milhar, derivamos: dezena, centena, grupo (animal)

**Algoritmo para extrair 5 resultados de um hash:**
```
const hash = HMAC_SHA256(serverSeed, clientSeed + ":" + nonce);
// hash tem 64 chars hex = 32 bytes
// Usamos 5 segmentos de 4 chars (4 hex = 16 bits = 0-65535)
for (let i = 0; i < 5; i++) {
  const segment = hash.substring(i * 8, i * 8 + 4);
  const value = parseInt(segment, 16);
  const milhar = value % 10000; // 0000-9999
  results[i] = milhar;
}
```

**Verificacao pelo jogador:**
1. Antes do jogo: server envia hash do server_seed (SHA256)
2. Jogador define client_seed (ou aceita o default)
3. Jogo acontece, resultado e revelado
4. Apos o jogo: server revela server_seed original
5. Jogador pode recalcular HMAC_SHA256 e confirmar que o resultado bate

**Complexidade:** Media. O algoritmo e simples, mas a UX de verificacao precisa ser clara.
