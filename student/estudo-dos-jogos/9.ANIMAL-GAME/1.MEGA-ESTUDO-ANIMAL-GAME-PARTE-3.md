## 7.2 Seguranca / Anti-Cheat (10 Itens)

| # | Aspecto | Implementacao Concreta |
|---|---------|----------------------|
| 1 | **Provably Fair** | HMAC_SHA256(server_seed, client_seed:nonce). Hash commitado ANTES da aposta. Server seed revelado APOS. Jogador recalcula e verifica. |
| 2 | **Validacao de input** | modo e enum valido, animals.length == expectedForMode, bet >= min e <= maxForMode, saldo suficiente. Rejeitar qualquer coisa fora de range. |
| 3 | **Rate limiting** | Cooldown 3s entre rodadas (config). Max 60 rodadas/hora por jogador. Map<playerId, timestamp> server-side. |
| 4 | **Max payout cap** | G$3.000.000 por rodada (config). Validado no server ANTES de creditar. Impossivel bypass pelo client. |
| 5 | **Max bet por modo** | Simple G$5.000, Double G$2.000, Triple G$1.000, Quadra G$500, Quina G$200. Validado server-side. |
| 6 | **Resultado server-only** | Client envia APENAS: mode, animals, bet, clientSeed. Server calcula TUDO: hash, resultados, payout. Client so anima. |
| 7 | **Audit log completo** | TODA rodada em casino_bicho_audit: PLAY, WIN, LOSE, CONFIG_CHANGE. JSON details com seeds, resultados, payout. Append-only. |
| 8 | **Atomicidade financeira** | Debito ANTES do draw. Credito APOS confirmacao de vitoria. Se server crashar entre debito e credito: jogador perde bet (worst case). Nao ha estado inconsistente onde jogador ganha sem debito. |
| 9 | **Config protegida** | Endpoint admin:config exige isAdmin. Alteracoes registradas no audit com valor anterior e novo. Nao acessivel por jogadores. |
| 10 | **Client seed customizavel** | Jogador pode definir proprio client_seed ou aceitar automatico. Isso garante que o server NAO pode manipular resultado unilateralmente. |
