## Problema 1: Sincronizacao Estado Server vs Client durante Rodada Ativa

**Sintoma:** O jogador clica uma tile. O client precisa saber se e gema ou bomba ANTES de animar. Porem, o client NAO pode conhecer o grid inteiro (anti-cheat). Isso cria latencia entre click e feedback visual.

**Causa tecnica:** Em Mines, o grid e gerado server-side com Provably Fair HMAC-SHA256. O client envia a posicao clicada ao server via fetchNui. O server verifica se aquela posicao e gema ou bomba e retorna o resultado. So entao o client anima. Essa round-trip adiciona ~50-150ms de latencia no NUI do FiveM.

**Dados concretos:**
- fetchNui round-trip tipico em FiveM: ~30-80ms (rede local, server no mesmo processo)
- Tempo de animacao de reveal: ~400ms (gema) ou ~600ms (bomba)
- Total percebido pelo jogador: ~430-680ms (aceitavel — cassinos reais tem delay similar)

**Solucao implementada:**
```javascript
// No MinesGame.tsx — feedback otimista com confirmacao server
async function handleTileClick(position) {
  if (gameState !== 'playing' || revealedTiles.has(position)) return;

  // Feedback imediato: tile entra em estado "loading" (pulse sutil)
  setLoadingTile(position);
  playSound('tile_click');

  // Pedir resultado ao server
  const resultado = await fetchNui('mines:revealTile', {
    roundId: currentRound.id,
    position,
  });

  setLoadingTile(null);

  if (resultado.type === 'gem') {
    setRevealedTiles(prev => new Set([...prev, position]));
    setMultiplier(resultado.multiplier);
    setNextProfit(resultado.nextProfit);
    setTotalProfit(resultado.totalProfit);
    playSound('gem_reveal');
  } else if (resultado.type === 'bomb') {
    setRevealedTiles(prev => new Set([...prev, position]));
    setBustPosition(position);
    setGameState('bust');
    playSound('bomb_explode');
    // Apos 500ms, revelar todas as posicoes
    setTimeout(() => {
      setFullGrid(resultado.fullGrid);
      playSound('tiles_reveal_stagger');
    }, 500);
  }
}
```

**Impacto:** ~40 linhas no MinesGame.tsx. Latencia imperceptivel para o jogador porque o feedback "loading" e instantaneo e a animacao de reveal mascara o delay.

## Problema 2: Calculo de Multiplicador — Precisao de Ponto Flutuante

**Sintoma:** O multiplicador envolve divisoes repetidas que acumulam erro de ponto flutuante. Com 24 picks e 1 mina, o multiplicador teorico e 24.75x mas JS pode calcular 24.749999... ou 24.750001.

**Causa tecnica:** JavaScript usa IEEE 754 double-precision. Multiplicacoes e divisoes repetidas acumulam epsilon (~2.2e-16 por operacao). Apos 24 multiplicacoes encadeadas, o erro pode ser ~5e-15 — insignificante para display mas problematico se comparado com tabela fixa.

**Solucao: tabela de multiplicadores pre-calculada + arredondamento**
```javascript
// Pre-calcular multiplicadores para todas combinacoes possiveis
// Executar 1x na inicializacao do server
function gerarTabelaMultiplicadores(houseEdge = 0.01) {
  const tabela = {};
  for (let minas = 1; minas <= 24; minas++) {
    tabela[minas] = {};
    let mult = 1;
    const totalGemas = 25 - minas;
    for (let picks = 1; picks <= totalGemas; picks++) {
      const tilesRestantes = 25 - (picks - 1);
      const gemasRestantes = totalGemas - (picks - 1);
      mult *= (tilesRestantes / gemasRestantes);
      // Arredondar para 2 casas decimais (padrao cassino)
      tabela[minas][picks] = Math.floor(mult * (1 - houseEdge) * 100) / 100;
    }
  }
  return tabela;
}

// Uso no handler: multiplicador = TABELA[qtdMinas][gemasReveladas]
// Sem calculo em runtime, sem erro de ponto flutuante acumulado
```

**Impacto:** ~25 linhas no handler. Tabela calculada 1x no boot do server. Lookup O(1) por pick. Precisao garantida. Padrao Stake/BC.Game (todos usam tabela pre-calculada, nao calculo runtime).

## Problema 3: Rate Limiting e Abuso de Auto Mode

**Sintoma:** Jogador malicioso usa script para enviar centenas de fetchNui por segundo, tentando exploitar timing ou overload do server.

**Causa tecnica:** fetchNui nao tem rate limit nativo. Sem protecao, cada RegisterNUICallback aceita qualquer frequencia de chamadas.

**Solucao: rate limiter por jogador no handler**
```javascript
// No mines.js handler — rate limiter simples
const cooldowns = new Map();
const COOLDOWN_MS = 200; // minimo 200ms entre acoes

function checkRateLimit(playerId) {
  const agora = Date.now();
  const ultimaAcao = cooldowns.get(playerId) || 0;
  if (agora - ultimaAcao < COOLDOWN_MS) {
    return false; // bloqueado
  }
  cooldowns.set(playerId, agora);
  return true; // permitido
}

// Limpar cooldowns de jogadores que sairam (a cada 5min)
setInterval(() => {
  const agora = Date.now();
  for (const [id, tempo] of cooldowns) {
    if (agora - tempo > 300000) cooldowns.delete(id);
  }
}, 300000);
```

**Impacto:** ~20 linhas no handler. Protege contra flood sem afetar gameplay normal (jogador real nunca clica 5x por segundo).

```javascript
// server/handlers/mines.js — endpoint principal
async function handleBet(playerId, betAmount, minesCount, callback) {
  if (!checkRateLimit(playerId)) {
    return callback({ error: 'rate_limit', message: 'Aguarde antes de apostar novamente.' });
  }

  // Validacoes
  const config = await oxmysql.query('SELECT * FROM casino_mines_config WHERE id = 1');
  if (!config[0]?.enabled) {
    return callback({ error: 'disabled', message: 'Mines esta desativado.' });
  }
  if (minesCount < config[0].min_mines || minesCount > config[0].max_mines) {
    return callback({ error: 'invalid_mines', message: 'Quantidade de minas invalida.' });
  }
  if (betAmount < config[0].min_bet || betAmount > config[0].max_bet) {
    return callback({ error: 'invalid_bet', message: 'Valor de aposta invalido.' });
  }

  // Verificar saldo
  const saldo = await getSaldo(playerId);
  if (saldo < betAmount) {
    return callback({ error: 'insufficient_funds', message: 'Saldo insuficiente.' });
  }

  // Verificar se ja tem rodada ativa
  const rodadaAtiva = await oxmysql.query(
    'SELECT id FROM casino_mines_rounds WHERE player_id = ? AND result = "ACTIVE"',
    [playerId]
  );
  if (rodadaAtiva.length > 0) {
    return callback({ error: 'active_round', message: 'Finalize a rodada atual antes de apostar.' });
  }

  // Gerar seeds e grid (Provably Fair)
  const seeds = await getOrCreateSeeds(playerId);
  const grid = gerarPosicoesMinas(seeds.server_seed, seeds.client_seed, seeds.nonce, minesCount);

  // Debitar saldo
  await debitarSaldo(playerId, betAmount);

  // Criar rodada
  const rodada = await oxmysql.query(
    `INSERT INTO casino_mines_rounds
     (player_id, bet_amount, mines_count, grid_data, server_seed, server_seed_hash, client_seed, nonce)
     VALUES (?, ?, ?, ?, ?, ?, ?, ?)`,
    [playerId, betAmount, minesCount, JSON.stringify(grid.positions),
     seeds.server_seed, seeds.server_seed_hash, seeds.client_seed, seeds.nonce]
  );

  // Incrementar nonce
  await oxmysql.query(
    'UPDATE casino_mines_seeds SET nonce = nonce + 1 WHERE player_id = ?',
    [playerId]
  );

  callback({
    roundId: rodada.insertId,
    serverSeedHash: seeds.server_seed_hash,
    minesCount,
    gemsAvailable: 25 - minesCount,
  });
}
```

**Codigo do endpoint mines:revealTile:**

```javascript
async function handleRevealTile(playerId, roundId, position, callback) {
  if (!checkRateLimit(playerId)) {
    return callback({ error: 'rate_limit' });
  }
  if (position < 0 || position > 24 || !Number.isInteger(position)) {
    return callback({ error: 'invalid_position' });
  }

  const rodada = await oxmysql.query(
    'SELECT * FROM casino_mines_rounds WHERE id = ? AND player_id = ? AND result = "ACTIVE"',
    [roundId, playerId]
  );
  if (rodada.length === 0) {
    return callback({ error: 'no_active_round' });
  }

  const round = rodada[0];
  const gridData = JSON.parse(round.grid_data);
  const tilesRevealed = round.tiles_revealed ? JSON.parse(round.tiles_revealed) : [];

  if (tilesRevealed.includes(position)) {
    return callback({ error: 'already_revealed' });
  }

  const tileType = gridData[position]; // 'gem' ou 'bomb'
  tilesRevealed.push(position);

  if (tileType === 'bomb') {
    // BUST — jogador perdeu
    await oxmysql.query(
      `UPDATE casino_mines_rounds
       SET result = 'BUST', tiles_revealed = ?, finished_at = NOW()
       WHERE id = ?`,
      [JSON.stringify(tilesRevealed), roundId]
    );

    return callback({
      type: 'bomb',
      position,
      multiplier: 0,
      payout: 0,
      fullGrid: gridData, // revelar todas posicoes
    });
  }

  // GEM — calcular multiplicador
  const gemsRevealed = tilesRevealed.filter(pos => gridData[pos] === 'gem').length;
  const multiplicador = TABELA_MULTIPLICADORES[round.mines_count][gemsRevealed];
  const totalProfit = Math.floor(round.bet_amount * multiplicador) - round.bet_amount;
  const gemasRestantes = (25 - round.mines_count) - gemsRevealed;
  const nextMultiplier = gemasRestantes > 0
    ? TABELA_MULTIPLICADORES[round.mines_count][gemsRevealed + 1]
    : null;
  const nextProfit = nextMultiplier
    ? Math.floor(round.bet_amount * nextMultiplier) - round.bet_amount
    : null;

  await oxmysql.query(
    `UPDATE casino_mines_rounds
     SET gems_revealed = ?, multiplier = ?, tiles_revealed = ?
     WHERE id = ?`,
    [gemsRevealed, multiplicador, JSON.stringify(tilesRevealed), roundId]
  );

  callback({
    type: 'gem',
    position,
    multiplier: multiplicador,
    totalProfit,
    nextMultiplier,
    nextProfit,
    gemsRevealed,
    gemsRemaining: gemasRestantes,
  });
}
```
