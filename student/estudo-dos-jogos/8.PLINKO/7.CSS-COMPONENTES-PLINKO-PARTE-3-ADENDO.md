# BALL STATES CSS (item 7.21 parcial)

NOTA: Ball eh renderizada no Canvas 2D, mas estes estados definem parametros
visuais que o Canvas rendering engine aplica por frame.

```css
/* Ball state: idle (antes do drop, invisivel) */
--ball-opacity: 0;
--ball-scale: 0;
--ball-trail: none;
/* Ball nao existe no Canvas ate o drop ser acionado */

/* Ball state: dropping (descendo, visivel + trail) */
--ball-opacity: 1;
--ball-scale: 1;
--ball-color: #D4A843;
--ball-shadow: 0 0 8px rgba(212, 168, 67, 0.5);
--ball-trail-alpha: 0.3;
--ball-trail-length: 5; /* frames de trail */

/* Ball state: bouncing (contato com peg — trigger flash) */
--ball-opacity: 1;
--ball-scale: 1.1; /* micro-pulse ao contato */
--peg-flash-color: rgba(255, 255, 255, 0.8);
--peg-flash-duration: 150ms;
--peg-flash-radius: 8px; /* glow ao redor do peg */

/* Ball state: landing (entrou no slot — slot glow) */
--ball-opacity: 0; /* ball desaparece ao entrar no slot */
--slot-glow-color: var(--slot-tier-color);
--slot-glow-spread: 15px;
--slot-scale: 1.15; /* bounce scale do slot */
--slot-scale-duration: 300ms;
```

## Resumo

| Estado | Visibilidade | Scale | Trail | Trigger |
|--------|:-----------:|:-----:|:-----:|---------|
| idle | hidden | 0 | none | antes do drop |
| dropping | visible | 1 | alpha 0.3, 5 frames | apos click DROP |
| bouncing | visible | 1.1 (pulse) | ativo | contato peg |
| landing | fade out | 0 | none | entrou no slot |
