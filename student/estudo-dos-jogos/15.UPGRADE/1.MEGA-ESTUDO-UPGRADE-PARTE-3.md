## 7.2 Seguranca / Anti-Cheat (10 Itens)

| # | Aspecto | Implementacao Concreta |
|---|---------|----------------------|
| 1 | **Calculo server-only** | Chance% e calculado no SERVER, nao no client. Client exibe preview, mas server recalcula independentemente. Se client enviar chance manipulada, server ignora. |
| 2 | **Provably Fair HMAC_SHA256** | Resultado determinado por HMAC_SHA256(serverSeed, clientSeed + ':' + nonce). Hash publicado ANTES do upgrade. Seed revelado APOS. Jogador pode verificar independentemente. Codigo baseado no Stake.com (publicado). |
| 3 | **Transacao atomica MySQL** | Remover item + adicionar item alvo (ou apenas remover) dentro de BEGIN/COMMIT. Se qualquer step falhar, ROLLBACK. Impossivel duplicar item ou perder sem registrar. |
| 4 | **Rate limit 3 camadas** | Cooldown em memoria (3s entre upgrades), lock anti-race (1 upgrade por vez por jogador), daily limit em MySQL (100/dia persistente). |
| 5 | **Validacao de input (5 checks)** | (1) Jogador possui o item? (2) Item alvo existe e esta ativo? (3) Valor alvo > valor meu? (4) Chance >= minChance config? (5) Chance <= maxChance config? |
| 6 | **Seed rotation** | Ao trocar client_seed, server revela server_seed anterior. Novo par gerado. Nonce reseta. Previne pre-computacao de resultados. |
| 7 | **Anti-manipulation de valor** | Valores dos itens vem do banco (casino_case_items.value), NAO do client. Client nao pode inflar valor do item proprio para aumentar chance. |
| 8 | **Audit log completo** | TODA acao em casino_upgrade_audit: UPGRADE_ITEM, UPGRADE_GCOIN, CONFIG_CHANGE, SEED_ROTATE, ERROR. JSON details com payload completo. Append-only, sem DELETE/UPDATE. |
| 9 | **Daily limit persistente** | Tabela casino_upgrade_daily com UNIQUE(player_id, play_date). INSERT ON DUPLICATE KEY UPDATE atomico. Sobrevive a restart do server. |
| 10 | **Config admin auditada** | Toda mudanca de config (house_edge, limites) gera registro em audit com old_value e new_value. Admin nao pode mudar edge silenciosamente. |

## 7.3 Perguntas Pre-Implementacao para o BC

| # | Pergunta | Default Sugerido | Impacto |
|---|----------|:----------------:|---------|
| 1 | House edge? | 8% | Revenue da casa. 8% e competitivo com mercado (CSGORoll ~8%, RustClash 10%) |
| 2 | Chance minima permitida? | 1% | Evita upgrades "impossiveis" que frustram. 1% ainda permite x100 |
| 3 | Chance maxima permitida? | 75% | Evita "upgrades seguros" que nao sao jogo de azar. 75% = multiplicador ~1.2x |
| 4 | Modo GCoin ativo? | Sim | Permite jogar sem ter itens. Mais acessivel. Pode desativar se preferir so item |
| 5 | Modo Item ativo? | Sim | Core do jogo. Reutiliza itens do Cases #14 |
| 6 | Limite diario? | 100 upgrades | Protecao contra vicio. Pode ajustar por admin |
| 7 | Cooldown? | 3000ms (3s) | Evita spam. 3s e padrao no mercado |
| 8 | Provably Fair visivel? | Sim | Transparencia. Botao "?" discreto no canto |
| 9 | Auto-upgrade? | Sim (max 20) | "Tentar N vezes automaticamente". Conveniente mas pode ser polemico |
| 10 | Som habilitado por padrao? | Sim | Feedback sonoro melhora experiencia. Toggle no canto para desativar |
