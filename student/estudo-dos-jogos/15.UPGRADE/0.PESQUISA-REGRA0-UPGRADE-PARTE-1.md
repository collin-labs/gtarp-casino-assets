## 2.1 MECANICA UNIVERSAL DO UPGRADER (padrao do mercado)

Apos analisar 6+ plataformas, a mecanica e consistente em todas:

**Fluxo completo do jogador:**

1. O jogador seleciona um ITEM DO INVENTARIO (ou usa saldo/moeda in-game)
2. O jogador seleciona um ITEM ALVO de valor superior (do catalogo da plataforma)
3. O sistema calcula a PORCENTAGEM DE CHANCE baseado na razao de valor:
   - Chance = (valor_depositado / valor_alvo) * (1 - house_edge)
   - Exemplo: Item de $5, alvo de $10, house edge 8% -> Chance = ($5/$10) * 0.92 = 46%
   - Exemplo: Item de $0.50, alvo de $25, house edge 8% -> Chance = ($0.50/$25) * 0.92 = 1.84%
4. A porcentagem e exibida ANTES do jogador confirmar (transparencia obrigatoria)
5. O jogador clica "UPGRADE" para confirmar
6. A ANIMACAO de resultado e executada (wheel spinner, ticker, circle)
7. Se GANHOU: recebe o item alvo no inventario
8. Se PERDEU: perde o item depositado (ou saldo equivalente)

**Variacao encontrada no Clash.gg:** O jogador pode escolher o PONTO DE INICIO do wheel (onde o ticker comeca), adicionando uma ilusao de controle estrategico. O resultado ja esta determinado pelo provably fair antes da animacao.

### Tipo B - Barra Linear com Ticker (segundo mais comum)
**Plataformas:** CSGORoll, algumas variantes

**Implementacao:** div com width proporcional + CSS transition no marcador
**Vantagem:** Mais simples visualmente, funciona bem em mobile
**Referencia:** "Hit upgrade button, wheel spins, marker lands in that tiny slice"

**Plataformas:** Algumas plataformas menores
**Visual:** Exibe um numero rolando de 0 a 100. Se o numero final for <= chance%, ganhou.
**Implementacao:** Counter animado com CSS, resultado pre-determinado
**Vantagem:** Ultra simples, zero dependencia visual
**Desvantagem:** Menos dramatico, menos engajamento
