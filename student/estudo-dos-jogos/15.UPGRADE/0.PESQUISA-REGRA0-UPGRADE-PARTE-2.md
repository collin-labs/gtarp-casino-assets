## 3.1 Componente A - Animacao do Wheel (o coracao visual do jogo)

A pesquisa revelou **4 abordagens tecnicas** para implementar a animacao circular do upgrader:

### Opcao A1 - SVG Circle + Framer Motion Rotate (RECOMENDADO)

**Fontes:** motion.dev/docs/react-svg-animation, logrocket.com (SVG circular progress React), github.com/tomickigrzegorz/react-circular-progress-bar, blog.noelcserepy.com (SVG path animation Framer Motion)

**Como funciona em detalhe:**
O wheel e composto por 2 camadas SVG:

Camada 1 - Arcos do wheel (zona verde + zona vermelha):
```
<svg viewBox="0 0 200 200">
  <!-- Arco VERMELHO (derrota) = circunferencia completa como background -->
  <circle cx="100" cy="100" r="90" fill="none"
    stroke="#FF1744" strokeWidth="18"
    strokeDasharray="565.48"  <!-- 2*PI*90 = circunferencia -->
    strokeDashoffset="0" />
  <!-- Arco VERDE (vitoria) = fatia proporcional a chance -->
  <circle cx="100" cy="100" r="90" fill="none"
    stroke="#00E676" strokeWidth="18"
    strokeDasharray="565.48"
    strokeDashoffset={565.48 * (1 - chancePercent/100)} />
</svg>
```
- `strokeDasharray` = circunferencia total (2*PI*raio)
- `strokeDashoffset` = quanto "esconder" do arco. Para 30% de chance: offset = 565.48 * 0.70 = 395.84 (mostra 30% verde)
- A zona verde COMECA no topo (12h) e cresce no sentido horario

Camada 2 - Ticker/Ponteiro (elemento que gira):
```tsx
<motion.line
  x1="100" y1="100" x2="100" y2="20"
  stroke="#FFD700" strokeWidth="3"
  animate={{ rotate: anguloFinal }}
  transition={{
    duration: 3.5,
    ease: [0.12, 0.8, 0.3, 1], // cubic-bezier: rapido no inicio, desacelera no fim
  }}
  style={{ transformOrigin: "100px 100px" }}
/>
```
- O Framer Motion anima `rotate` de 0 a N graus (varias voltas + angulo final)
- anguloFinal = (random via provably fair) * 360. Se caiu na faixa 0 a chance*3.6, ganhou
- A animacao roda 3-5 voltas completas (1080-1800 graus) + angulo final, dando sensacao de spin real
- Easing cubic-bezier `[0.12, 0.8, 0.3, 1]` cria desaceleracao natural (rapido -> lento -> para)
- Alternativa: `type: "spring"` com `visualDuration: 3.5, bounce: 0.05` para parada suave com micro-bounce

**Descoberta na pesquisa Motion.dev (marco 2026):**
- Framer Motion foi renomeado para "Motion" (motion.dev) desde 2025
- SVG transforms no Motion: transformOrigin e relativo ao CENTRO do elemento (nao ao viewBox) -- diferente do SVG nativo
- `motion.circle`, `motion.line`, `motion.path` suportam todas as props de animacao
- `useMotionValue` + `animate()` permitem animar sem re-renders React (performance critica para NUI)
- Custom easing via array `[a, b, c, d]` ou funcao JS `(t) => ...`

**Pros:**
- Zero dependencias extras (Motion/Framer Motion ja esta no projeto como `framer-motion@11.18.x`)
- SVG e responsivo por natureza com `viewBox`
- Performance: animacao de `rotate` roda no compositor thread (GPU-accelerated)
- Glow/brilho via SVG `<filter>` ou CSS `filter: drop-shadow()`
- Integra com sistema de cores do DS via CSS variables
- `useMotionValue` evita re-renders durante spin (critico para 60fps no NUI)

**Contras:**
- Requer calculos de circunferencia para posicionar arcos (math de circulo)
- SVG filter para glow pode ter custo no NUI (usar com moderacao)

**Veredicto:** MELHOR OPCAO. Reutiliza stack existente, performance otima, visual premium possivel.

### Opcao A2 - Canvas 2D Arc + requestAnimationFrame

**Fontes:** dev.to/sababg/custom-wheel-of-prize-with-canvas (codigo funcional completo), petecorey.com (React hooks + Canvas animation), medium.com/@pdx.lucasm (Canvas with React custom hook)

**Como funciona:**
- useRef para canvas element + useEffect para setup
- `ctx.arc(centerX, centerY, radius, startAngle, endAngle)` para segmentos verde/vermelho
- `ctx.fillStyle` com `createRadialGradient()` para visual premium
- requestAnimationFrame loop: `angleDelta *= friction` a cada frame para desaceleracao
- Ticker como linha desenhada com rotacao incrementada a cada frame

**Codigo de referencia encontrado (DEV.to WheelComponent):**
```
// Spin logic simplificada
let angleDelta = maxSpeed; // velocidade maxima inicial
const upTime = segments.length * upDuration; // tempo de aceleracao
const downTime = segments.length * downDuration; // tempo de desaceleracao

function onTimerTick() {
  const elapsed = Date.now() - spinStart;
  const progress = Math.min(elapsed / (upTime + downTime), 1);
  if (progress < upTime/(upTime+downTime)) {
    // acelerando
    angleDelta = maxSpeed * Math.sin(progress * Math.PI * 0.5);
  } else {
    // desacelerando
    angleDelta = maxSpeed * Math.sin(0.5 + progress * Math.PI * 0.5);
  }
  angleCurrent += angleDelta;
  draw(); // redesenha canvas com nova rotacao
}
```

**Pros:**
- Controle pixel-perfect de cada frame
- Pode adicionar particulas e trail effects DENTRO do canvas (mesmo contexto)
- Performance excelente para animacao simples
- Nenhuma dependencia extra

**Contras:**
- NAO e responsivo nativamente (precisa handler de resize + recalcular dimensoes)
- NAO integra com Framer Motion (dois sistemas de animacao separados)
- Texto dentro do canvas requer fillText() com posicionamento manual
- Acessibilidade zero (canvas e opaco)
- Re-render do canvas em cada frame vs GPU-accelerated transform do SVG

**Veredicto:** VIAVEL como fallback. Mais complexo que SVG para este caso de uso. Melhor para jogos que ja usam Canvas (Crash, Plinko). Nao recomendado para Upgrader.

### Opcao A3 - CSS Conic Gradient + Transform Rotate

**Como funciona:**
```css
.wheel {
  width: 200px;
  height: 200px;
  border-radius: 50%;
  background: conic-gradient(
    #00E676 0% 30%,    /* verde = chance 30% */
    #FF1744 30% 100%   /* vermelho = resto */
  );
}
.ticker {
  position: absolute;
  transform: rotate(var(--angle));
  transition: transform 3.5s cubic-bezier(0.12, 0.8, 0.3, 1);
}
```

**Pros:**
- Ultra simples (5 linhas CSS para o wheel visual)
- conic-gradient suportado em todos browsers modernos
- CSS transition para animacao do ticker (sem JS)

**Contras:**
- Sem controle granular por segmento (sem borda entre zonas, sem glow individual)
- conic-gradient nao suporta stroke ou outline entre fatias
- Dificil adicionar efeitos premium (particulas, brilho na zona de parada)
- Menos dramatico visualmente

**Veredicto:** BOM para prototipo rapido ou versao simplificada. Para producao premium, SVG (A1) e superior.

### DECISAO TECNICA FINAL: SVG Arc + Framer Motion (Opcao A1)

**Justificativa com 7 pontos:**
1. Framer Motion (agora "Motion") ja esta no projeto v11.18.x (zero bundle adicional)
2. SVG escala responsivamente via viewBox + clamp() no container (DS P1 Responsividade)
3. Cores via CSS custom properties (integra com Design System)
4. `rotate` e GPU-accelerated no NUI/CEF (Chromium compositor thread)
5. useMotionValue evita re-renders React durante spin (performance critica)
6. SVG filters permitem glow dourado no ponteiro e nas zonas
7. Easing customizavel via cubic-bezier ou spring com bounce

## 3.5 Componente E - Provably Fair

**Fontes:** stake.com/provably-fair/implementation (codigo JS oficial), btcgosu.com/provably-fair/, 99rekt.com/learn/how-provably-fair-works, bgaming.com/provably-fair

**Algoritmo padrao para Upgrader (baseado no Stake.com):**

```
ANTES DO UPGRADE:
1. Server gera server_seed (random 64-char hex string)
2. Server publica hash = SHA256(server_seed) -- jogador ve o hash
3. Jogador confirma o upgrade (client_seed gerado automaticamente pelo browser)

CALCULO DO RESULTADO:
4. combined = HMAC_SHA256(server_seed, client_seed + ":" + nonce)
5. Extrair 4 bytes do hash: hex = combined.substring(0, 8)
6. decimal = parseInt(hex, 16)
7. resultado = (decimal % 10000) / 100 -- numero de 0.00 a 99.99
8. Se resultado < chance% => GANHOU
   Se resultado >= chance% => PERDEU

APOS O UPGRADE:
9. Server revela server_seed
10. Jogador pode verificar: SHA256(server_seed_revelado) === hash_publicado
11. Jogador pode recalcular o resultado com os seeds

CONVERSAO PARA ANGULO DO WHEEL:
12. angulo_visual = resultado * 3.6 -- converte 0-100 para 0-360 graus
13. O ticker gira ate parar nesse angulo
14. Se angulo_visual cai dentro da zona verde (0 a chance*3.6), vitoria exibida
```

**Dados concretos da pesquisa Stake.com (codigo real):**
- `createHmac('sha256', serverSeed)` -- Node.js crypto nativo
- `hmac.update(clientSeed + ':' + nonce + ':' + currentRound)`
- `const buffer = hmac.digest()` -- retorna 32 bytes
- 4 bytes = 1 resultado de jogo, 32 bytes = ate 8 resultados
- Nonce incrementa a cada aposta (garante unicidade sem mudar seeds)
- Client seed gerado pelo browser do jogador (pode customizar)
