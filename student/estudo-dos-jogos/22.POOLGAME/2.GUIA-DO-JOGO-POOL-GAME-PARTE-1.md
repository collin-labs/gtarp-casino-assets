## MECÂNICAS PROFUNDAS — POR QUE CADA DECISÃO

### Por que 8-Ball como modo principal e não 9-Ball?
8-ball é o formato de pool mais jogado no mundo — é o que o jogador casual CONHECE. "Encaçape seus solids ou stripes, depois a 8-ball." Qualquer brasileiro que já jogou sinuca no bar entende em 10 segundos. 9-ball exige "sempre acertar a menor primeiro" — confuso para quem não joga competitivamente. O Miniclip 8 Ball Pool (70M+ usuários) prova: 8-ball é universal. 9-ball existe como modo secundário para jogadores avançados que querem partidas mais curtas e dinâmicas.

### Por que RAKE e não house edge?
Pool é skill-based — adicionar RNG seria destruir a essência. Em jogos RNG (slots, loteria), a casa tem vantagem matemática. Em pool, os dois jogadores têm chance igual — a casa não pode "fazer o jogador perder." A solução histórica (desde 1800s) é o rake: a casa cobra comissão sobre o pot. 5% de rake em pot de 2.000 GCoins = 100 GCoins para o casino. O jogador ACEITA pagar rake porque sabe que pode ganhar pela habilidade. É psicologicamente diferente: "perdi porque ele jogou melhor" vs "perdi porque a máquina decidiu." O primeiro gera respeito e vontade de melhorar; o segundo gera frustração.

### Por que SPIN e não apenas aim+power?
Sem spin, pool digital é "aponte e acerte" — simplista, repetitivo, sem profundidade. Spin transforma cada tacada em uma decisão multidimensional: "Se eu der follow, a cue ball continua e encaçapa a próxima. Se eu der draw, ela volta para posição." O Miniclip 8 Ball Pool tem spin como feature central — jogadores profissionais usam em 60%+ das tacadas. A pesquisa no World of Pool confirma: side spin tem 3 efeitos reais (squirt, swerve, throw). Nós simplificamos para 2D (post-collision velocity adjustment) que captura 90% do feel sem physics 3D.

### Por que guideline CONFIGURÁVEL por cue?
Guideline longa = fácil mirar. Sem guideline = expert mode. O sistema de cues com stat "aim" que controla o comprimento da guideline (100-300px) cria um skill ceiling natural: jogador com Basic Cue (guideline curta) precisa de mais habilidade para mirar; jogador com Diamond Cue (guideline longa) tem ajuda visual. Isso incentiva compra de cues (monetização) SEM ser pay-to-win — a guideline ajuda a mirar, não mira sozinha. O admin pode configurar `guidelineMode: "none"` para torneios hardcore.

### Por que Matter.js e não Canvas 2D puro?
O CodePen do Dmujt prova: pool funcional em Matter.js pode ser feito em ~200 linhas. Gravity=0 é nativo, sensors para pockets são elegantes, frictionAir simula felt perfeitamente, colisão ball-ball é automática com restitution configurável. Canvas 2D puro exigiria 400-600 linhas de colisão manual com bugs de tunneling. Matter.js resolve 80% da physics automaticamente, e o custom spin layer resolve os 20% restantes. Já foi considerada para a Loteria #18 — reuso de dependência.

### Por que server-side physics?
Se o client calcular physics, um hacker pode manipular posições de bolas ("minha bola encaçapou, juro!"). Com server-side physics, o server é a ÚNICA fonte de verdade: client envia input (angle, power, spin), server simula via Matter.js, e retorna resultado (posições finais) para AMBOS clients. Impossível hackear. Trade-off: ~200ms de latência — perfeitamente aceitável para turn-based.

## VALOR NA ECONOMIA DO SERVIDOR RP

| Impacto | Detalhamento |
|---------|-------------|
| **Rake = receita passiva** | 5% de cada pot. 50 partidas/dia × pot médio 1.000 GCoins × 5% = 2.500 GCoins/dia de receita pura para o casino. Sem risco — a casa ganha independente de quem vence. |
| **Cues = sink econômico** | 5 cues: 0 + 2k + 5k + 15k + 50k = 72.000 GCoins para comprar tudo. Multiplicado por 100 jogadores = 7.200.000 GCoins removidos da economia permanentemente. Combate inflação. |
| **Skill-based = retenção diferente** | Jogador que perde no pool não culpa "a máquina" — culpa a si mesmo. Isso gera: (1) vontade de melhorar, (2) treino no modo practice, (3) compra de cue melhor, (4) volta para jogar de novo. Loop de retenção mais saudável que RNG. |
| **Social = RP orgânico** | Pool é 1v1 presencial. Jogadores interagem: trash talk, aposta, torcida. Spectators assistem e comentam. É conteúdo RP natural — diferente de slots onde o jogador está sozinho. |
| **Gateway competitivo** | Pool é porta de entrada para torneios futuros. Admin pode organizar campeonatos com bracket, premiação, streaming. Cria eventos RP únicos. |
| **Economia circular** | Dinheiro NÃO é destruído — circula de perdedor para vencedor. Apenas o rake é drenado. Isso é saudável: jogadores habilidosos acumulam GCoins, jogadores medianos perdem gradualmente, todos gastam em cues. |
| **Practice mode = inclusão** | Mesa sem aposta permite que QUALQUER jogador aprenda sem risco. Converte jogador casual em apostador futuro. Sem barreira de entrada. |
| **Tiers = segmentação** | Tier 250 para casual, tier 5.000 para high-roller. Cada segmento econômico tem sua mesa. Admin adiciona/remove tiers via config. |
