```javascript
/* COPIAR — drawBallInHand() — função Canvas durante ball-in-hand */
function drawBallInHand(ctx, mouseX, mouseY, balls, tableRect) {
  const { left, top, right, bottom } = tableRect;
  const R = BALL_RADIUS;

  // Verificar posição válida
  const inBounds = mouseX > left + R && mouseX < right - R &&
                   mouseY > top + R && mouseY < bottom - R;
  const noOverlap = !balls.some(b => b.id !== 0 &&
    Math.hypot(mouseX - b.x, mouseY - b.y) < R * 2.2);
  const isValid = inBounds && noOverlap;

  // Ghost cue ball
  ctx.save();
  ctx.globalAlpha = isValid ? 0.65 : 0.3;

  // Gradiente esférico igual à cue ball real
  const grad = ctx.createRadialGradient(
    mouseX - R * 0.4, mouseY - R * 0.4, 0,
    mouseX, mouseY, R
  );
  grad.addColorStop(0, 'rgba(255,255,255,0.9)');
  grad.addColorStop(0.5, 'rgba(220,220,220,0.8)');
  grad.addColorStop(1, 'rgba(160,160,160,0.5)');
  ctx.beginPath();
  ctx.arc(mouseX, mouseY, R, 0, Math.PI * 2);
  ctx.fillStyle = grad;
  ctx.fill();

  // Anel de validação (verde=ok, vermelho=inválido)
  ctx.globalAlpha = 0.7;
  ctx.strokeStyle = isValid ? 'rgba(0,230,118,0.7)' : 'rgba(255,23,68,0.7)';
  ctx.lineWidth = 2;
  ctx.stroke();

  // X vermelho se inválido
  if (!isValid) {
    ctx.globalAlpha = 0.6;
    ctx.strokeStyle = 'rgba(255,23,68,0.8)';
    ctx.lineWidth = 1.5;
    const r = R * 0.45;
    ctx.beginPath();
    ctx.moveTo(mouseX - r, mouseY - r); ctx.lineTo(mouseX + r, mouseY + r);
    ctx.moveTo(mouseX + r, mouseY - r); ctx.lineTo(mouseX - r, mouseY + r);
    ctx.stroke();
  }

  // Zona válida sutil (toda a mesa em verde muito leve)
  if (isValid) {
    ctx.globalAlpha = 0.03;
    ctx.fillStyle = '#00E676';
    ctx.fillRect(left, top, right - left, bottom - top);
  }

  ctx.restore();
}
```

# SEÇÃO 3 — TIMELINES EXATAS

## 3.1 Timeline Shot Dispatch (desde SHOOT até início de animação)

```
t=0ms       Jogador clica SHOOT (ou toca no mobile).
            Client coleta: { angle, power, spinTop, spinSide }

t=0ms       SHOOT button transiciona para --locked.
            Guideline e taco: Framer Motion exit 0.2s.

t=0-20ms    fetchNui('casino:pool:shoot', shotData) →
            Lua Client → TriggerServerEvent('casino:pool:shoot', shotData)

t=20-100ms  Server recebe. Valida turno. Calcula physics (Matter.js async).
            Server roda simulação: ~30-90ms por shot (26 bodies).

t=100-200ms Server retorna via callback:
            { ballPositions, pocketed, foul, nextTurn, gameOver, winner }

t=200ms     Client inicia animação:
            currentPositions → targetPositions via lerp() 2s total.
            play('cue_strike') — t=200ms.

t=200-400ms Bolas acelerando (ease-in primeiros 200ms do lerp).
            Colisões ball-ball: play('ball_hit_ball') a cada detecção.
            Colisões cushion: play('ball_hit_cushion').

t=800-2200ms Fase de desaceleração (ease-out cubic).
            Bolas aproximam posições finais.

t=2200ms    Animação base termina (lerp progress = 1).
            Verificar pocketed[].

t=2200ms+   Para cada bola encaçapada:
            triggerPocketAnimation() — shrink 0→scale 0 em 300ms.
            Stagger: cada pocket +100ms delay.
            play('ball_pocket') por bola, play('ball_pocket_8') se 8-ball.

t=2700ms    Todas pocket animations terminadas.
            Verificar: foul? gameOver? nextTurn?

t=2700ms    SE foul:
            → pool-foul-flash ativa (600ms @keyframes).
            → pool-foul-banner slide-in spring.
            → play('foul') — buzz 400ms.
            → SE scratch: iniciar ball-in-hand (§3.2)
            → SE não scratch: turno muda após 1.5s

t=2700ms    SE gameOver:
            → delay 500ms
            → overlay resultado (§2.4)

t=2700ms    SE nextTurn (jogada normal):
            → delay 300ms
            → turn switch banner (§2.3)
            → avatar pulse troca de jogador

TOTAL shot cycle (sem gameOver): ~3-4s
TOTAL shot cycle (com pocket 8-ball + vitória): ~3.5-4.5s
```
