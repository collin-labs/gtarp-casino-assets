## 7.2 Segurança / Anti-Cheat (10 Itens)

| # | Aspecto | Implementação Concreta |
|---|---------|----------------------|
| 1 | **Physics server-only** | Server roda Matter.js para CADA shot. Client envia apenas input (angle, power, spin). Server retorna resultado (posições finais). Client APENAS anima interpolando para posições do server. Impossível hackear posições de bolas. |
| 2 | **Validação de input (5 checks)** | (1) É o turno do jogador? (2) angle é número finito [-π, π]? (3) power é 0-1? (4) spinTop é -1 a +1? (5) spinSide é -1 a +1? Inputs fora de range = rejeitar. |
| 3 | **Rate limit por turno** | Server só aceita 1 shot por turno. Flag `shotPending = true` enquanto simula. Segundo shot = ignorar. Reset flag quando turno encerra. |
| 4 | **Timer com auto-foul** | Server mantém timer por turno. Se `maxTurnTime` excedido: auto-foul + ball-in-hand para oponente. Client não controla timer — server é autoritativo. |
| 5 | **Disconnect = forfeit** | `Events.on('playerDropped')` no Lua server. Timeout 30s para reconexão. Se não reconecta: auto-forfeit, oponente vence, pot distribuído. |
| 6 | **Rake atômico** | `pot = tierA + tierB; rake = pot * config.rakePercent / 100; prize = pot - rake`. Calculado no server antes de creditar. Audit log com valores exatos. |
| 7 | **Daily limit** | Tabela `casino_pool_daily` com UNIQUE(player_id, play_date). `INSERT ... ON DUPLICATE KEY UPDATE matches_played = matches_played + 1`. Atômico. Default 50/dia. |
| 8 | **Cooldown entre partidas** | `Map<playerId, timestamp>`. Cooldown 5000ms (config). Evita spam de join. |
| 9 | **Simulation budget** | Máximo 300 steps por shot simulation. Force stop após. Previne loops infinitos ou shots maliciosos que tentam overload o server. |
| 10 | **Audit log completo** | TODA ação em `casino_pool_audit`: MATCH_START, MATCH_END (winner, reason, prize, rake), SHOT (angle, power, spin, pocketed), FOUL (type), FORFEIT, CUE_BUY, CONFIG_CHANGE, ERROR. JSON `details` flexível. Append-only. |
