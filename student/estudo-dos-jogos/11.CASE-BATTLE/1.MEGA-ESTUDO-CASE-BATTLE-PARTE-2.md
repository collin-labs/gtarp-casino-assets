# 6. DETALHAMENTO TÉCNICO — REVEAL STRIP (Peça Central)

O RevealStrip é o componente mais importante do Case Battle. É a animação que transforma uma simples "abertura de caixa" em experiência cinematográfica.

## 6.1 Arquitetura do RevealStrip

```
RevealStrip.tsx
├── Props: { items: Item[], winnerIndex: number, onComplete: () => void }
├── State: { isSpinning, revealedItem }
├── GSAP Timeline:
│   ├── Phase 1 (0-0.5s): Strip acelera de 0 → velocidade máxima
│   ├── Phase 2 (0.5-3s): Strip mantém velocidade (itens passando rápido)
│   ├── Phase 3 (3-4.5s): Strip desacelera (ease: power3.out)
│   └── Phase 4 (4.5-5s): Item vencedor para no centro com bounce sutil
└── Callback: onComplete() → atualiza score, avança para próximo round
```

## 6.3 GSAP Animation Code (Pseudocódigo)

```typescript
// RevealStrip — lógica GSAP
function spinStrip(items: Item[], winnerIndex: number) {
  const itemWidth = 83; // width + margin
  const stripWidth = items.length * itemWidth;
  
  // Posição do item vencedor centralizado
  const containerCenter = containerRef.current.offsetWidth / 2;
  const targetX = -(winnerIndex * itemWidth) + containerCenter - (itemWidth / 2);
  
  // Adicionar overshoot (várias voltas antes de parar)
  const fullLoops = 3; // 3 voltas completas antes de parar
  const totalDistance = (stripWidth * fullLoops) + Math.abs(targetX);
  
  const tl = gsap.timeline();
  
  // Start: posição 0
  tl.set(innerRef.current, { x: 0 });
  
  // Spin rápido (ease-in) → constante → desacelera (ease-wheel do DS)
  tl.to(innerRef.current, {
    x: -totalDistance,
    duration: 5,
    ease: "power3.out", // DS P3A §18: --ease-wheel
    onUpdate: () => {
      // Som de tick quando item cruza o centro
      const currentX = gsap.getProperty(innerRef.current, "x") as number;
      const itemIndex = Math.floor(Math.abs(currentX) / itemWidth);
      if (itemIndex !== lastTickIndex.current) {
        lastTickIndex.current = itemIndex;
        soundManager.play('battle_tick', { volume: 0.3 });
      }
    },
    onComplete: () => {
      // Highlight do item vencedor
      soundManager.play('battle_reveal');
      setRevealedItem(items[winnerIndex]);
      onComplete();
    }
  });
  
  return tl;
}
```

## 6.4 Geração dos Itens no Strip

O strip NÃO mostra apenas os itens possíveis da caixa. Ele gera uma lista VISUAL de ~60-80 itens aleatórios (respeitando probabilidades) com o item vencedor posicionado no índice calculado. Isso cria a ilusão de "quase ganhar" itens raros.

```typescript
function generateStripItems(caseItems: CaseItem[], winnerItem: CaseItem, count = 70): StripItem[] {
  const strip: StripItem[] = [];
  const winnerPosition = Math.floor(count * 0.7) + Math.floor(Math.random() * 10); // ~70% do caminho
  
  for (let i = 0; i < count; i++) {
    if (i === winnerPosition) {
      strip.push(winnerItem);
    } else {
      // Item aleatório respeitando probabilidades
      const roll = Math.random() * 100;
      let cum = 0;
      for (const item of caseItems) {
        cum += item.probability;
        if (roll <= cum) { strip.push(item); break; }
      }
    }
  }
  
  return strip;
}
```

## 8.1 Provably Fair — Implementação Completa

### Fluxo Criptográfico

```
ANTES DA BATALHA:
1. Server gera: serverSeed = crypto.randomBytes(32).toString('hex')
2. Server calcula: serverSeedHash = SHA256(serverSeed)
3. Server armazena serverSeed (oculto) e serverSeedHash (público) no DB
4. Jogador vê serverSeedHash na sala de espera
5. Jogador pode definir clientSeed (default: gerado pelo browser)

DURANTE A BATALHA (por jogador, por round):
6. Para cada round, calcular:
   result = HMAC_SHA256(serverSeed, `${clientSeed}-${nonce}`)
   ticket = parseInt(result.substring(0, 8), 16) % totalWeight
7. Nonce incrementa +1 por round por jogador

APÓS A BATALHA:
8. Server revela serverSeed (não-hasheado)
9. Jogador pode verificar: SHA256(serverSeed) === serverSeedHash (mostrado antes)
10. Jogador pode reproduzir todos os resultados com os 4 valores
```

### Código de Verificação (exibido no modal)

```javascript
// Código para o jogador verificar — copiar e colar
const crypto = require('crypto');

const serverSeed = 'REVELADO_APOS_BATALHA';
const clientSeed = 'SEU_CLIENT_SEED';
const nonce = 0; // incrementa por round

// Verificar hash
const hash = crypto.createHash('sha256').update(serverSeed).digest('hex');
console.log('Hash bate?', hash === 'HASH_MOSTRADO_ANTES');

// Reproduzir resultado
const hmac = crypto.createHmac('sha256', serverSeed)
  .update(`${clientSeed}-${nonce}`).digest('hex');
const ticket = parseInt(hmac.substring(0, 8), 16) % TOTAL_WEIGHT;
console.log('Ticket:', ticket); // deve bater com o resultado
```

## 9.1 Modo Underdog (Fase 1 — SQL + Backend)

No modo Underdog, o jogador com MENOR valor total vence. A lógica é idêntica ao Classic, apenas a ordenação inverte:

```javascript
// Classic: maior vence
battleResults.sort((a, b) => b.totalValue - a.totalValue);

// Underdog: menor vence
if (battle.mode === 'underdog') {
  battleResults.sort((a, b) => a.totalValue - b.totalValue);
}
```

**Impacto visual:** Badge "UNDERDOG" em vermelho na lista de batalhas. Na tela de jogo, os valores menores ficam destacados em verde (inversão).

## 9.2 Batalha Privada (Fase 1 — SQL + Backend)

Campo `is_private` + `private_link` (UUID de 32 chars). Batalha privada não aparece na lista pública (`battle_list` filtra `is_private = 0`). Criador recebe link para compartilhar.

## 9.3 Cashback para Perdedores (Fase 1 — SQL + Backend)

Campo `cashback_pct` (0-10%). Ao finalizar, perdedores recebem `entry_fee * (cashback_pct / 100)` de volta:

## 9.4 Handler de Cancelamento (Fase 1 — Backend novo)

```javascript
// HANDLER 13: Cancelar batalha
RegisterNuiCallback('casino:cases:battle_cancel', async (data, cb) => {
  const userId = source;
  const { battleId } = data;
  
  // Verificar: é o criador? Batalha está em waiting? Está sozinho?
  const [[battle]] = await conn.execute(
    'SELECT * FROM casino_cases_battles WHERE id = ? AND creator_id = ? AND status = ?',
    [battleId, userId, 'waiting']
  );
  if (!battle) return cb({ ok: false, error: 'Não pode cancelar' });
  
  const [[{ cnt }]] = await conn.execute(
    'SELECT COUNT(*) as cnt FROM casino_cases_battle_players WHERE battle_id = ?',
    [battleId]
  );
  if (cnt > 1) return cb({ ok: false, error: 'Batalha já tem outros jogadores' });
  
  // Devolver entry fee
  vRP.addMoney(userId, parseFloat(battle.entry_fee));
  
  // Cancelar
  await conn.execute(
    `UPDATE casino_cases_battles SET status = 'cancelled' WHERE id = ?`, [battleId]
  );
  
  // Audit
  await conn.execute(
    `INSERT INTO casino_cases_audit (user_id, action, details) VALUES (?, 'battle_cancel', ?)`,
    [userId, JSON.stringify({ battle_id: battleId, refund: battle.entry_fee })]
  );
  
  cb({ ok: true, refund: parseFloat(battle.entry_fee), newBalance: vRP.getMoney(userId) });
});
```
