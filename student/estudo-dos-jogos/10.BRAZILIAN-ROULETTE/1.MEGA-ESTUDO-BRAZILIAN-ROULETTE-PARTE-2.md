# RELATÓRIO DA PESQUISA COMPLEMENTAR (REGRA X0)

## O que foi pesquisado
- Provably Fair implementation para roulette (Stake.com docs, BTCGOSU, NSoft/Medium, Gamingtec)
- `provably-fair-framework` npm (matthewlilley) — suporta `EuropeanRoulette` strategy nativa
- SHA-256 + HMAC para roleta: `hash → hex → parseInt(hex.substring(0,8), 16) % 37` = número 0-36
- Padrão Stake.com 2026: `byteGenerator` com cursor (JÁ IMPLEMENTADO no slots handler do projeto)

## O que trouxe de novo
1. **O projeto JÁ TEM byteGenerator Provably Fair implementado** no `server/handlers/slots.js` (padrão Stake.com 2026). Podemos reutilizar diretamente a mesma função `byteGenerator` + `generateFloats` para gerar o número da roleta.
2. **Para roleta europeia**: 1 float → `Math.floor(float × 37)` = número 0-36. Para Lucky Numbers: 5 floats adicionais → `Math.floor(float × 37)` sem repetição.
3. **Para multiplicadores**: 1 float por Lucky Number → distribuição ponderada: 50x (50%), 100x (30%), 200x (15%), 500x (5%).

## O que teríamos feito SEM a pesquisa
Implementaríamos um RNG separado com `Math.random()` — inseguro e não verificável. A pesquisa mostrou que o byteGenerator do Stake.com JÁ EXISTE no projeto e pode ser reutilizado. Economia de ~100 linhas de código.

## 8.2 Handlers Detalhados

### `br_roulette:config` — Carregar configuração
```
Parâmetros: { mode: 'relampago' | 'classica' | 'mini' }
Validações: modo existe na tabela config
Retorno: { min_bet, max_bet, straight_up_payout, round_timer, ... }
```

### `br_roulette:bet` — Registrar aposta
```
Parâmetros: { mode, bet_type, numbers: number[], amount }
Validações:
  1. Fase = 'betting' (server controla timer)
  2. amount >= min_bet E amount <= max_bet
  3. Saldo suficiente (SELECT balance WHERE id = user_id)
  4. bet_type válido para o modo
  5. numbers[] válidos (0-36 para europeia, 0-14 para mini)
  6. Total de apostas do jogador nesta rodada <= max_bet
Ação: Débito atômico → INSERT bet → retorna { ok, newBalance, betId }
Retorno: { ok: true, balance: 4500.00, betId: 12345 }
```

### `br_roulette:spin` — Executar rodada (trigger automático ou manual)
```
Parâmetros: (nenhum — server controla)
Validações: Fase = 'betting' terminada (timer expirou)
Lógica:
  1. Gerar resultado via Provably Fair:
     float = generateFloats({ serverSeed, clientSeed, nonce, cursor: 0, count: 1 })
     resultNumber = Math.floor(float[0] * 37)  // 0-36

  2. Se modo 'relampago':
     luckyCount = random(config.lightning_min_lucky, config.lightning_max_lucky)
     luckyFloats = generateFloats({ ..., cursor: 4, count: luckyCount * 2 })
     Para cada lucky:
       number = Math.floor(luckyFloats[i*2] * 37) // sem repetição com resultado
       multiplierFloat = luckyFloats[i*2 + 1]
       multiplier = weightedPick(multiplierFloat, {50: 0.50, 100: 0.30, 200: 0.15, 500: 0.05})

  3. Calcular payouts por bet:
     Para cada bet da rodada:
       se resultNumber ∈ bet.numbers:
         basePayout = bet.amount * bet.payout_multiplier
         se modo relâmpago E bet_type = straight E resultNumber ∈ luckyNumbers:
           finalPayout = bet.amount * luckyNumbers[resultNumber]
         senão:
           finalPayout = basePayout
         creditar jogador (UPDATE balance SET balance = balance + finalPayout)

  4. INSERT round + UPDATE bets com won/win_amount
  5. UPDATE stats
  6. INSERT audit log
  7. Incrementar nonce
  8. Broadcast resultado para todos os clientes

Retorno (broadcast): {
  roundId, resultNumber, resultColor,
  luckyNumbers: { "17": 200, "5": 50 },
  winners: [{ userId, betType, amount, winAmount, multiplier }],
  nextPhase: 'betting', nextRoundIn: 5000
}
```

### `br_roulette:seed_change` — Trocar seed do jogador
```
Parâmetros: { newClientSeed: string }
Validações: seed >= 1 char, <= 64 chars, alphanumeric
Ação:
  1. Revelar seed antigo (UPDATE active=0, revealed_at=NOW())
  2. Gerar novo serverSeed (crypto.randomBytes(32).toString('hex'))
  3. Hash do novo serverSeed (SHA-256)
  4. INSERT novo seed ativo
Retorno: {
  previousServerSeed: '...',   // revelado para verificação
  previousServerSeedHash: '...', // deve bater com o que player já tinha
  newServerSeedHash: '...',    // hash do novo (player guarda para verificar depois)
  nonce: 0                     // resetado
}
```

### `br_roulette:verify` — Verificar fairness de rodada
```
Parâmetros: { roundId }
Validações: rodada existe e seed já foi revelado
Retorno: {
  serverSeed, serverSeedHash, clientSeed, nonce,
  resultNumber, luckyNumbers,
  verificationSteps: "SHA256(serverSeed + clientSeed + nonce) → hex → parseInt(hex[0..7], 16) % 37 = resultNumber"
}
```
