## Problemas reais com codigo

### P1: Calendar midnight reset + timezone FiveM

FiveM nao tem `Date()` confiavel no client — `GetGameTimer()` conta ms desde
inicio do recurso, nao desde meia-noite. Solucao: server envia timestamp Unix
do proximo reset via evento `daily:syncClock`. Client calcula countdown local.

```lua
-- server-side: enviar proximo reset
local function getNextMidnight()
    local now = os.time()
    local date = os.date("*t", now)
    date.hour, date.min, date.sec = 0, 0, 0
    local midnight = os.time(date) + 86400
    return midnight
end

RegisterNetEvent("daily:requestClock")
AddEventHandler("daily:requestClock", function()
    local src = source
    TriggerClientEvent("daily:syncClock", src, getNextMidnight(), os.time())
end)
```
