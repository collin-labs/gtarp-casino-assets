## PESQUISA X0 — RODADAS 4 E 5 (especificas para Client Lua + React Canvas TS)

**Rodada 4 — FiveM RegisterNUICallback + SetNuiFocus pattern:**
- Query: "FiveM RegisterNUICallback TriggerServerEvent SetNuiFocus client lua pattern 2025 2026"
- Fontes: docs.fivem.net/nui-callbacks, docs.fivem.net/client-functions, github.com/imThaku/fivem-es_extended/client/main.lua, github.com/FiveM-Scripts/customchat/chat_client.lua, github.com/N3MTV/gcphone/client/client.lua, codepal.ai/fivem-lua-menu-system
- Descobertas: (1) Padrao NUI callback: `RegisterNUICallback('name', function(data, cb) ... cb(result) end)`. Data e JSON automatico. (2) `SetNuiFocus(true, true)` captura teclado+mouse. OBRIGATORIO `SetNuiFocus(false, false)` no close — se esquecer, jogador fica preso no NUI. (3) `SendNUIMessage({})` envia dados Lua→NUI (React recebe via window.addEventListener('message')). (4) Pattern confirmado em 5+ scripts: RegisterNUICallback → TriggerServerEvent → RegisterNetEvent para resposta do server. (5) gcphone usa `DisableControlAction` em thread enquanto NUI aberto — previne jogador atirar/andar. (6) Fluxo completo: NUI JS fetch() → RegisterNUICallback Lua → TriggerServerEvent → Server handler → emitNet → RegisterNetEvent Lua → SendNUIMessage → NUI JS window.addEventListener.
- Impacto: O Daily-Free usa o pattern simplificado (RegisterNUICallback direto, sem DisableControlAction porque o painel ja gerencia focus). Confirma que cb() DEVE ser chamado sempre, mesmo em erro.

**Rodada 5 — React Canvas 2D TypeScript useRef + useEffect + rAF:**
- Query: "React useRef useEffect Canvas 2D animation wheel spin component TypeScript 2025 2026"
- Fontes: medium.com/@ignatovich.dm (Canvas+React), github.com/Sivamani-18/SpinWheel (ts-spin-wheel), spin.atomicobject.com (Canvas animations React), petecorey.com (Canvas React hooks), dev.to/ptifur (Canvas rAF React), hashnode.blainegarrett.com (Canvas Refs TypeScript), ankursheel.com (draw hooks TS)
- Descobertas: (1) Pattern correto: `useRef<HTMLCanvasElement>(null)` + `useRef<CanvasRenderingContext2D>(null)` para contexto. (2) Canvas context so disponivel apos mount — OBRIGATORIO usar useEffect com check `if (canvasRef.current)`. (3) Animacao via rAF: usar ref para angle/velocity (NAO useState — evita re-render a cada frame). Pattern: `const angleRef = useRef(0)` + `const velocityRef = useRef(0)` + rAF loop que le/escreve refs. (4) Cleanup: `return () => cancelAnimationFrame(animRef.current)` no useEffect. (5) ts-spin-wheel expoe `spin()` via `useImperativeHandle` + `forwardRef` — validacao do pattern imperativo. (6) Device pixel ratio: `window.devicePixelRatio || 1` para canvas nitido. No NUI FiveM provavelmente 1 (resolucao do jogo). (7) Offscreen canvas: `document.createElement('canvas')` no useEffect mount, desenhar roda 1x, reutilizar via drawImage no rAF.
- Impacto: Confirma que WheelCanvas.tsx deve usar refs para estado de animacao (nao useState). Offscreen pre-render no mount. forwardRef + useImperativeHandle para expor `startSpin(targetSegment)` ao componente pai.

**O que teriamos feito sem estas pesquisas:**
- Usado useState para angle da roda (re-render 60x/s = lag)
- Esquecido cancelAnimationFrame no cleanup (memory leak)
- Esquecido cb() no RegisterNUICallback em caso de erro (NUI trava)
- Sem DisableControlAction awareness (jogador poderia atirar enquanto no menu)

-- ====================================================
-- BLACKOUT CASINO — DAILY-FREE CLIENT (#19)
-- Ponte NUI <-> Server
-- Versao 1.0 - 24/03/2026
-- ====================================================

local dailyFreeOpen = false

-- Verificar status do dia (ao abrir a tela)
RegisterNUICallback('daily-free:check', function(data, cb)
  TriggerServerEvent('daily-free:check')
  cb({ ok = true })
end)

-- Executar spin (girar a roda)
RegisterNUICallback('daily-free:spin', function(data, cb)
  TriggerServerEvent('daily-free:spin', {
    clientSeed = data.clientSeed or 'default',
    charId = data.charId or nil
  })
  cb({ ok = true })
end)

-- Buscar historico
RegisterNUICallback('daily-free:history', function(data, cb)
  TriggerServerEvent('daily-free:history')
  cb({ ok = true })
end)

-- Verificar Provably Fair de um claim especifico
RegisterNUICallback('daily-free:verifyPF', function(data, cb)
  TriggerServerEvent('daily-free:verifyPF', {
    claimId = data.claimId
  })
  cb({ ok = true })
end)

-- Fechar o NUI (devolver controle ao GTA)
RegisterNUICallback('daily-free:close', function(_, cb)
  SetNuiFocus(false, false)
  dailyFreeOpen = false
  cb({ ok = true })
end)

-- ---- LISTENERS DO SERVER (respostas assincronas) ----

-- Resposta do check
RegisterNetEvent('daily-free:checkResult')
AddEventHandler('daily-free:checkResult', function(result)
  SendNUIMessage({
    type = 'daily-free:checkResult',
    data = result
  })
end)

-- Resposta do spin
RegisterNetEvent('daily-free:spinResult')
AddEventHandler('daily-free:spinResult', function(result)
  SendNUIMessage({
    type = 'daily-free:spinResult',
    data = result
  })
end)

-- Resposta do historico
RegisterNetEvent('daily-free:historyResult')
AddEventHandler('daily-free:historyResult', function(result)
  SendNUIMessage({
    type = 'daily-free:historyResult',
    data = result
  })
end)

-- Resposta da verificacao PF
RegisterNetEvent('daily-free:verifyPFResult')
AddEventHandler('daily-free:verifyPFResult', function(result)
  SendNUIMessage({
    type = 'daily-free:verifyPFResult',
    data = result
  })
end)

-- Resposta do reload config (admin)
RegisterNetEvent('daily-free:configReloaded')
AddEventHandler('daily-free:configReloaded', function(result)
  SendNUIMessage({
    type = 'daily-free:configReloaded',

  })
end)
