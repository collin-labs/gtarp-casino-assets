## Especificações do Spinner

**Animação do donut (CSS transform rotate + Framer Motion):**

A técnica ideal para animar o spinner é rotacionar o ELEMENTO inteiro (não o conic-gradient), pois CSS trata gradientes como imagens geradas que não podem ser interpoladas em transitions. O `transform: rotate()` é GPU-accelerated e performante.

```javascript
// GSAP timeline de 2 fases (extraído da pesquisa)
// Fase 1: aceleração (2s) — giro rápido
// Fase 2: desaceleração (6s) — para no segmento vencedor

const targetAngle = calculateWinnerAngle(winnerTicket, totalTickets, players);
const extraSpins = 3; // 3 voltas completas antes de desacelerar
const finalRotation = (extraSpins * 360) + targetAngle;

// Alternativa Framer Motion (se GSAP não instalado):
<motion.div
  animate={{ rotate: finalRotation }}
  transition={{
    duration: 8,
    ease: [0.15, 0.85, 0.35, 1], // cubic-bezier desaceleração natural
  }}
  onAnimationComplete={() => revealWinner()}
/>
```

**Cálculo do ângulo alvo:**
```javascript
function calculateWinnerAngle(winnerTicket, totalTickets, players) {
  let cumulativePercent = 0;
  for (const p of players) {
    const startPercent = cumulativePercent;
    cumulativePercent += (p.amount / totalTickets) * 100;
    if (winnerTicket >= p.ticketStart && winnerTicket <= p.ticketEnd) {
      // Ângulo = centro da seção do vencedor
      const midPercent = startPercent + (cumulativePercent - startPercent) / 2;
      return (midPercent / 100) * 360;
    }
  }
  return 0;
}
```
