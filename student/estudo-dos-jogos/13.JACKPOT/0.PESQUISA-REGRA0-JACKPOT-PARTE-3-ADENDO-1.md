## 1.2 O que é o Jackpot (definição da indústria)

O Jackpot PvP é um dos formatos de gambling mais antigos e reconhecidos na comunidade de skin gambling. A mecânica é universalmente consistente entre todos os sites pesquisados:

**Conceito:** Múltiplos jogadores depositam valor (skins ou GCoin) em um pot compartilhado. A chance de cada jogador ganhar é proporcional ao valor que depositou em relação ao pot total. Quando o timer fecha, um vencedor é selecionado aleatoriamente (ponderado) e leva tudo, menos a taxa da casa.

**Diferença fundamental do Coinflip:**
- Coinflip: 2 jogadores, 50/50, aposta igual
- Jackpot: 2-N jogadores, probabilidade proporcional ao valor, apostas variáveis

## 1.3 Mecânica detalhada (baseada em CSGOEmpire, Howl.gg, RustFunPot, SkinsLuck)

**Fluxo de uma rodada:**

1. **Pot abre** — Timer inicia (tipicamente 60-120 segundos)
2. **Jogadores depositam** — Cada jogador deposita valor no pot. Pode depositar múltiplas vezes.
3. **Odds calculadas** — Chance de cada jogador = (valor depositado / valor total do pot) × 100%
4. **Timer fecha** — Quando timer chega a zero, novos depósitos são bloqueados
5. **Vencedor selecionado** — Algoritmo provably fair seleciona vencedor baseado nas porcentagens
6. **Vencedor leva tudo** — Recebe todo o pot menos taxa da casa (3-10%, tipicamente 5%)

**Exemplo numérico:**
```
Player A deposita: 500 GCoin  → 500/2000 = 25% de chance
Player B deposita: 1.000      → 1000/2000 = 50% de chance
Player C deposita: 300        → 300/2000 = 15% de chance
Player D deposita: 200        → 200/2000 = 10% de chance
────────────────────────────────────────
Pot total: 2.000 GCoin
Rake 5%: 100 GCoin
Vencedor recebe: 1.900 GCoin

Se Player B ganhar (50% chance): lucro = 1.900 - 1.000 = +900
Se Player D ganhar (10% chance): lucro = 1.900 - 200 = +1.700 (!)
```

**Observação crítica:** O Jackpot é o jogo mais "alto risco, alta recompensa" do cassino. Um jogador com 10% de chance pode ganhar 10x seu investimento. Isso cria picos de adrenalina enormes.

## 2.2 O Spinner (elemento visual central)

O spinner é o coração visual do Jackpot. Existem 2 abordagens na indústria:

### Abordagem A: Wheel/Roda (CSGOEmpire, RustFunPot, SkinsLuck)

Roda circular dividida em setores coloridos. Cada setor é proporcional à porcentagem do jogador. Pointer/seta fixa no topo. A roda gira e desacelera até parar com o setor vencedor alinhado ao pointer.

- **Técnica:** CSS `transform: rotate(Xdeg)` com animação de desaceleração
- **Visual:** Setores com cores diferentes por jogador (máx 10 cores distintas)
- **Duração:** 8-12 segundos de spin
- **Easing:** Desaceleração exponencial (começa rápido, desacelera suavemente)

### Abordagem B: Strip/Fita horizontal (Howl.gg, CSGO500)

Fita horizontal com avatares dos jogadores repetidos proporcionalmente. Scroll horizontal rápido que desacelera. Pointer centralizado.

- **Técnica:** CSS `transform: translateX(Xpx)` com desaceleração
- **Visual:** Avatares lado a lado, repetidos conforme porcentagem
- **Mais simples** de implementar que a roda

**Decisão para o Blackout Casino:** Usar **Abordagem A (Wheel/Roda)** — mais visual, mais premium, mais impactante. Combina com a identidade dourada do casino (roda dourada com setores coloridos).

## 4.2 Ticket System — implementação detalhada

O sistema de tickets é o padrão da indústria para Jackpot provably fair. Cada unidade monetária depositada = 1 ticket. Tickets são sequenciais e atribuídos na ordem de depósito.

```
EXEMPLO COMPLETO:

Rodada #55
  Player A deposita 500 GCoin  → tickets 0-499     (25%)
  Player B deposita 1.000      → tickets 500-1499   (50%)
  Player A deposita mais 200   → tickets 1500-1699   (+10%, total A = 35%)
  Player C deposita 300        → tickets 1700-1999   (15%)
  
  Total tickets: 2.000
  
  ANTES da rodada: server gera serverSeed + serverSeedHash (público)
  QUANDO timer fecha: publicSeed = hash de todos os depósitos + timestamp
  
  Resultado:
    hash = SHA256(serverSeed + '-' + publicSeed + '-' + roundId)
    winningTicket = parseInt(hash.substring(0, 8), 16) % 2000
    winningTicket = 1542 → pertence a Player A (tickets 1500-1699)
    Player A vence!
```

**Diferença crucial do Coinflip:** No Coinflip, o resultado é binário (modulo 2). No Jackpot, o resultado é modulo totalTickets, e precisa ser mapeado para ranges de tickets.

## 4.3 Implementação JavaScript para o server handler

```javascript
function resolveJackpotRound(serverSeed, publicSeed, roundId, deposits) {
  // 1. Calcular total de tickets
  const totalTickets = deposits.reduce((sum, d) => sum + d.amount, 0);
  
  // 2. Gerar winning ticket via hash
  const combined = `${serverSeed}-${publicSeed}-${roundId}`;
  const hash = crypto.createHash('sha256').update(combined).digest('hex');
  const first8 = hash.substring(0, 8);
  const winningTicket = parseInt(first8, 16) % totalTickets;
  
  // 3. Mapear ticket → jogador
  // Agrupar depósitos por jogador (um jogador pode depositar múltiplas vezes)
  const playerTotals = {};
  deposits.forEach(d => {
    playerTotals[d.player_id] = (playerTotals[d.player_id] || 0) + d.amount;
  });
  
  // 4. Encontrar vencedor
  let accumulated = 0;
  let winnerId = null;
  for (const [playerId, total] of Object.entries(playerTotals)) {
    accumulated += total;
    if (winningTicket < accumulated) {
      winnerId = parseInt(playerId);
      break;
    }
  }
  
  return {
    winnerId,
    winningTicket,
    totalTickets,
    hash,
    // Calcular ângulo do spinner: (winningTicket / totalTickets) * 360
    spinAngle: (winningTicket / totalTickets) * 360
  };
}
```

## 4.4 Verificação pelo jogador

O jogador pode verificar:
1. `SHA256(serverSeed) === serverSeedHash` (integridade)
2. Recalcular `SHA256(serverSeed-publicSeed-roundId)` → winning ticket
3. Verificar que o winning ticket pertence ao range do vencedor declarado
4. Verificar que seu próprio range de tickets está correto (soma dos depósitos)
