## 2.1 Variantes de Jogos de Dados em Cassinos

A pesquisa revelou 5 variantes principais de jogos de dados no mercado de cassinos online em 2025-2026:

### Lightning Dice (Evolution Gaming, 2019)

O Lightning Dice e a variante mais visualmente impressionante e mais adequada para entretenimento digital. Tres dados sao fisicamente lancados em uma torre transparente chamada Lightning Tower — uma estrutura vertical de perspex com multiplos niveis de obstaculos e pins que alteram aleatoriamente o caminho dos dados conforme caem. Um apresentador puxa uma alavanca para liberar os dados no topo, e uma camera close-up segue toda a trajetoria ate a base.

O diferencial e o sistema de multiplicadores Lightning: antes de cada rodada, 2 a 4 totais (dos possiveis 3-18) sao selecionados via RNG e recebem multiplicadores que variam de 50x ate 1.000x. Os totais mais raros (3 e 18, com apenas 1 combinacao cada em 216 possiveis) recebem os multiplicadores mais altos (500x-1.000x), enquanto os totais mais comuns (10 e 11, com 27 combinacoes cada) recebem multiplicadores menores (ate 50x). Isto cria um equilibrio onde o RTP final fica entre 96.03% e 96.21% — sem os multiplicadores, o house edge seria de 25-30%, tornando o jogo injogavel.

O studio e produzido em ambiente Art Deco preto e dourado, com efeitos de raio, iluminacao dramatica e apresentadores carismaticos. A rodada inteira leva aproximadamente 60 segundos. Apostas adicionais incluem Low (3-10), High (11-18), Any Double e Any Triple.

### Sic Bo (Tradicional Chines)

Sic Bo ("Dados Preciosos") e um jogo milenar chines com 3 dados agitados em um chest pelo dealer. O layout e uma mesa complexa com 52 opcoes de aposta agrupadas em categorias:

- **Big/Small:** Total entre 11-17 (Big) ou 4-10 (Small). Paga 1:1. House edge 2.78%. O mais seguro
- **Roll Totals:** Aposta em total especifico (4-17). Payouts de 6:1 (totais comuns) ate 60:1 (totais raros)
- **Specific Triple:** Tres dados iguais com numero especifico. Paga 150:1 (Macau) ou 180:1 (Atlantic City). House edge 16-30%
- **Any Triple:** Qualquer tres iguais. Paga 24:1 (Macau) ou 30:1 (Atlantic City)
- **Domino Combinations:** Aposta em combinacao de 2 faces especificas
- **Single Dice:** Numero aparece em pelo menos 1 dado. Paga 1:1 (1 dado), 2:1 (2 dados), 3:1 (3 dados)

O Sic Bo tradicional e popular em Macau (segundo lugar atras do baccarat) mas tem layout complexo e 52 opcoes que intimidam jogadores casuais. O Super Sic Bo da Evolution adiciona multiplicadores (similar ao Lightning) com RTP ate 97.22%.

### Craps (Mesa Americana)

Craps usa 2 dados com um shooter (jogador que lanca). O fluxo e complexo: come-out roll (7/11 = win, 2/3/12 = loss), point established (4/5/6/8/9/10), fase de point (repetir ponto antes de 7). Dezenas de tipos de aposta: Pass Line (1.41% house edge), Don't Pass (1.36%), Come, Don't Come, Place bets, Hardways, Proposition bets. House edge varia de 0% (Laying Odds) ate 16.67% (Big 6/8).

Craps e o jogo de dados mais complexo e social, popular nos EUA. Porem, o layout e enorme, as regras sao complexas, e a mecanica de multi-roll torna a implementacao em NUI FiveM impraticavel — exigiria uma mesa completa com dezenas de areas de aposta e multiplas fases por rodada.

### Bitcoin Dice (Over/Under)

Genero cripto-nativo: jogador escolhe um target em range 0-100 via slider, prediz se resultado cai acima ou abaixo, e payout e calculado inversamente a probabilidade. Se escolhe 50 (chance 50%), paga ~1.98x. Se escolhe 95 (chance 5%), paga ~19x. House edge tipico de 1% (RTP 99%). Rodadas instantaneas, auto-bet, provably fair nativo.

E o jogo de dados mais simples que existe — nao envolve dados visuais reais, apenas um numero aleatorio. A interacao se resume a um slider e um botao. Visualmente pobre para um cassino premium como o Blackout Casino.

### Hi-Lo (Over/Under Simplificado)

Versao simplificada: dado(s) rolado(s), jogador prediz se proximo resultado sera maior ou menor que o anterior. Payouts fixos baseados na probabilidade. Variantes incluem predizer faixa de resultado. Mecanica basica, adequada como mini-game mas nao como jogo standalone.

### Opcao A: CSS 3D Transforms (preserve-3d)

Cubo construido com 6 divs filhas posicionadas via rotateX/Y + translateZ. Animacao de giro via @keyframes ou Framer Motion controlando rotateX/Y/Z do container. Para mostrar face especifica: calcular angulos finais (ex: face 6 = rotateX(0) rotateY(0), face 1 = rotateX(0) rotateY(180deg)).

```css
.dice {
  transform-style: preserve-3d;
  width: 100%;
  height: 100%;
  position: relative;
  transition: transform 1.5s cubic-bezier(0.2, 0.8, 0.3, 1);
}
.dice.rolling {
  animation: diceRoll 1.2s ease-out;
}
.dice-face {
  position: absolute;
  width: 100%;
  height: 100%;

  background: linear-gradient(145deg, #1a1a1a, #0a0a0a);
  border: 1px solid rgba(212, 168, 67, 0.3);
  box-shadow: inset 0 0 15px rgba(0, 0, 0, 0.5);

.dice-face--front  { transform: rotateY(0deg)   translateZ(50px); }
.dice-face--back   { transform: rotateY(180deg) translateZ(50px); }
.dice-face--right  { transform: rotateY(90deg)  translateZ(50px); }
.dice-face--left   { transform: rotateY(-90deg) translateZ(50px); }
.dice-face--top    { transform: rotateX(90deg)  translateZ(50px); }
.dice-face--bottom { transform: rotateX(-90deg) translateZ(50px); }
```

**Pros:** Zero dependencias, funciona em NUI CEF (Chromium), leve (~50 linhas CSS), customizavel com dourado/neon do DS.
**Contras:** Sem fisica real (dados nao quicam), animacao pre-determinada.

### Opcao B: Three.js + Cannon-es (Fisica 3D)

Renderizacao WebGL com Three.js + fisica simulada com Cannon-es. BoxGeometry com cantos arredondados, luzes e sombras reais. sleepEvent do Cannon-es detecta quando dado para de rolar.

**Pros:** Fisica realista (dados quicam e rolam naturalmente), visual AAA.
**Contras:** Three.js pesa ~600KB+ minificado, Cannon-es +50KB. GPU compartilhada no NUI FiveM pode causar lag. Setup complexo. Cannon-es descontinuada.

### Opcao C: @3d-dice/dice-box (BabylonJS + AmmoJS)

Modulo npm pronto: BabylonJS para render + AmmoJS para fisica, web workers e offscreenCanvas.

**Pros:** Pronto para uso, API simples (roll("3d6")), temas customizaveis.
**Contras:** BabylonJS pesa ~1MB+, overkill para 3 dados. NUI FiveM nao suporta offscreenCanvas em todas as builds. Dependencia pesada.

### Opcao D: Framer Motion com CSS 3D (HIBRIDO)

Combinar CSS 3D para a estrutura do cubo com Framer Motion para controlar a animacao (rotateX, rotateY, rotateZ como motion values). Resultado pre-determinado server-side, animacao calcula angulos finais para a face correta.

**Pros:** Ja instalado no projeto (Framer Motion 11), leve, integra com React de forma idiomatica, animacoes suaves com spring physics do Framer. Customizavel com easing curves do DS.
**Contras:** Sem fisica real (mesma limitacao da Opcao A), porem spring physics do Framer simula bounce de forma convincente.
