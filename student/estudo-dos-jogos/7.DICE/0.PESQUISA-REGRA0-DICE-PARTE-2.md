## 8.1 Algoritmo

**HMAC-SHA256** (padrao Stake.com 2026).

O resultado dos 3 dados e determinado pelo hash HMAC-SHA256 gerado a partir do server seed, client seed e nonce. O hash de 64 caracteres hexadecimais e convertido em 3 numeros de 1 a 6 que representam as faces dos dados.

## 8.2 Fluxo Completo (10 Passos)

**ANTES DO JOGO:**

1. **Server gera server seed:** String aleatoria de 64 caracteres hex via crypto.randomBytes(32).toString('hex'). Este seed determina TODOS os resultados futuros da sessao do jogador
2. **Server calcula hash do server seed:** SHA-256(serverSeed) = hash de 64 chars. Este hash e enviado ao jogador ANTES de qualquer aposta, provando que o seed ja existia e nao sera alterado
3. **Jogador define client seed:** O browser gera um seed aleatorio automaticamente, mas o jogador pode escolher qualquer string. Isto injeta aleatoriedade que o servidor nao controla — equivale a "cortar o baralho" num cassino fisico
4. **Nonce inicializa em 0:** Contador que incrementa a cada aposta, garantindo que mesmo com seeds iguais, cada resultado e diferente

**DURANTE O JOGO (a cada aposta):**

5. **Gerar bytes via HMAC-SHA256:** O servidor calcula HMAC-SHA256(serverSeed, clientSeed:nonce:0). O resultado e um buffer de 32 bytes. Para gerar os 3 dados, usa 4 bytes por dado (12 bytes total):
   - Dado 1: bytes[0-3] convertidos para float 0.0-1.0, depois mapeados para 1-6
   - Dado 2: bytes[4-7] convertidos para float 0.0-1.0, depois mapeados para 1-6
   - Dado 3: bytes[8-11] convertidos para float 0.0-1.0, depois mapeados para 1-6

6. **Conversao bytes para resultado:** Para cada dado, os 4 bytes sao combinados: `(byte[0] / 256^1 + byte[1] / 256^2 + byte[2] / 256^3 + byte[3] / 256^4)` = float entre 0 e 1. Depois: `Math.floor(float * 6) + 1` = numero de 1 a 6

7. **Calcular total e avaliar apostas:** Total = dado1 + dado2 + dado3. Comparar com as apostas do jogador. Verificar se alguma aposta coincide com os Lightning Numbers. Calcular payout

8. **Incrementar nonce:** nonce += 1 para a proxima aposta

**APOS O JOGO (verificacao):**

9. **Revelar server seed:** Quando o jogador encerra a sessao ou solicita rotacao de seed, o servidor revela o server seed original (nao hashado). O jogador pode entao recalcular SHA-256(serverSeed) e comparar com o hash recebido no passo 2 — se forem iguais, prova que o servidor nao alterou o seed apos as apostas

10. **Verificacao independente:** O jogador pode recalcular QUALQUER resultado usando: serverSeed + clientSeed + nonce da aposta especifica. Se o resultado calculado corresponde ao resultado exibido durante o jogo, a rodada foi justa. Ferramentas de verificacao online (como as do Stake.com) permitem colar os seeds e nonce para recalcular automaticamente

## 8.3 Variaveis Explicadas

| Variavel | Quem define | Quando | Para que |
|----------|:-----------:|:------:|----------|
| Server Seed | Servidor | Inicio da sessao | Fonte primaria de aleatoriedade. Determina todos os resultados da sessao. Mantido secreto ate a sessao encerrar |
| Server Seed Hash | Servidor | Inicio da sessao | Prova criptografica de que o seed existia antes das apostas. SHA-256 e one-way — impossivel derivar o seed do hash |
| Client Seed | Jogador | Inicio da sessao (editavel) | Injeta aleatoriedade controlada pelo jogador. Garante que o servidor sozinho nao controla o resultado. Editavel a qualquer momento |
