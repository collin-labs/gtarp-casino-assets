# SECAO 3 — TIMELINES DE ANIMACAO (ms)

Sequencia completa de uma rodada do Dice, do clique no ROLAR DADOS ate o retorno ao estado IDLE.

## 3.1 Timeline — Rodada Normal (sem multiplicador acertado)

```
t=0ms      Jogador clica ROLAR DADOS
           → Estado muda para LIGHTNING
           → Som: bet_confirm.mp3 (300ms)
           → Botao desabilita, grid escurece parcialmente
           → Dados escurecem (opacity 0.4)

t=100ms    Lightning Phase inicia
           → Barra de multiplicadores aparece (fade-in 200ms)
           → Som: lightning_ambient.mp3 inicia (loop, volume 0.3)

t=300ms    Lightning Strike #1
           → Badge multiplicador aparece (spring scale 0→1, 300ms)
           → Card no grid recebe animacao raio (400ms)
           → Som: lightning_strike.mp3 (800ms)

t=800ms    Lightning Strike #2 (se 2+ totais)
           → Identico ao #1 com delay 500ms

t=1300ms   Lightning Strike #3 (se 3+ totais)
t=1800ms   Lightning Strike #4 (se 4 totais)

t=2000ms   Lightning Phase encerra (se 2 strikes)
           → Som: lightning_ambient.mp3 fade-out (300ms)
           → Dados iniciam SHAKE
           → Estado muda para ROLLING

t=2000ms   SHAKE inicia
           → Dados tremem (Framer Motion x/y vibration)
           → Bordas piscam dourado/verde (200ms cycle)
           → Som: dice_shake.mp3 (500ms, pode fazer loop se resultado nao chegou)

t=2800ms   SHAKE encerra (minimo 800ms ou quando resultado chega)
           → ROLL inicia

t=2800ms   ROLL inicia
           → Dado 1 comeca rotacao 3D (spring, 1.8s)
           → Dado 2 comeca 200ms depois
           → Dado 3 comeca 400ms depois
           → Som: dice_roll.mp3 (1.5s)
           → Blur de movimento ativo (filter blur 1.5px) nos primeiros 0.5s

t=3300ms   Blur remove (dados desacelerando, faces visiveis)

t=4600ms   Dado 1 pousa — LANDING
           → Bounce (scale 1→1.08→0.96→1.02→1, 300ms)
           → Flash de luz (opacity 0→0.8→0, 300ms)
           → Sombra projetada aparece (fade-in 200ms)
           → Som: dice_bounce.mp3 (200ms)

t=4800ms   Dado 2 pousa
           → Identico ao dado 1
           → Som: dice_bounce.mp3

t=5000ms   Dado 3 pousa
           → Identico
           → Som: dice_bounce.mp3

t=5200ms   TODOS pousaram
           → Som: dice_land.mp3 (300ms, impacto final mais grave)
           → Estado muda para RESULT

t=5400ms   Total REVEAL
           → Numero aparece (spring scale 0.3→1, 400ms)
           → Son: result_reveal.mp3 (500ms)
           → Grid reage: cards acertados brilham verde, erros fade

t=5600ms   Payout calculado
           → Win display: countup 0→valor (500-2000ms)
           → Som: win_small.mp3 (se ganhou) ou loss.mp3 (se perdeu)

t=7600ms   Resultado persiste (2s de visibilidade)

t=7600ms   Transicao para IDLE
           → Total fadeOut (500ms)
           → Grid retorna ao normal (1s transition)
           → Botao ROLAR reabilita com pulse
           → Estado = IDLE, pronto para proxima rodada

TOTAL RODADA NORMAL: ~7.6 segundos
```
