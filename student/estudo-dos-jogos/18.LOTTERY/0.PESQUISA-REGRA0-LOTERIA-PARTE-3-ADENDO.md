## A1.1 Arquitetura do Bingo Blower (FreeCodeCamp)

O autor testou **7 bibliotecas** antes de escolher Matter.js: Anime.js, Velocity.js, Popmotion, Three.js, GreenSock (GSAP), Mo.js e Matter.js. Escolheu Matter.js porque é a única que oferece **física de corpo rígido 2D real** — as outras são bibliotecas de animação que simulam movimento mas sem colisão verdadeira entre objetos.

**Componentes do Matter.js usados no projeto:**
- `Matter.Engine` — controller que gerencia a simulação do mundo
- `Matter.World` — métodos para criar e manipular o mundo
- `Matter.Render` — renderer HTML5 Canvas (o autor recomenda NÃO usar em produção — fazer render customizado)
- `Matter.Bodies` — cria as bolas e as paredes invisíveis do container
- `Matter.Body` — aplica forças nos bodies para criar o efeito de "sopro de ar"
- `Matter.Runner` — executa a simulação
- `Matter.Events` — escuta o evento 'tick' a cada frame de render para aplicar forças e detectar colisões

**Técnica do "Tubo" (Container):**
O globo/tubo NÃO é um body do Matter.js. É uma **imagem de background CSS** que cria a ilusão visual. As paredes reais são **bodies estáticos invisíveis** posicionados em arco para formar o perímetro circular. Essa é a técnica chave: **separar o visual (CSS/PNG) da física (bodies invisíveis)**.

**Técnica do "Air Blow" (Turbulência):**
O autor usa `Matter.Events.on(engine, 'tick', callback)` com um timeout de 3 segundos. A cada tick, ele verifica colisão ball-wall e aplica `Matter.Body.applyForce(ball, ball.position, {x: randomX, y: -0.05})`. A força Y negativa empurra pra cima (simula ar). O X é random para dar caos lateral. Isso cria o efeito visual de bolas sendo "sopradas" dentro da esfera.

**Bolas como Imagens:**
O projeto original usa 75 imagens PNG (uma por bola de bingo), não círculos Canvas. Cada ball body tem uma `render.sprite.texture` apontando para a imagem. Para o Blackout Casino, podemos fazer igual: bolas como PNG com número impresso, ou renderizar via Canvas `arc()` + `fillText()` — mais leve, sem 60 PNGs.

**Repositório:** github.com/mihailgaberov/bingo-blower — MIT license, código aberto, funcional. Deploy live: tender-hoover-fdc559.netlify.app

## A1.2 Como Adaptar para Container Circular

O artigo usa container retangular (tubo vertical). Para o globo esférico da loteria:

**Técnica de Polyline Circular:**
```javascript
// Criar 32 segmentos formando um círculo
const segments = 32;
const radius = 150; // raio do globo em pixels
const centerX = canvasWidth / 2;
const centerY = canvasHeight / 2;

for (let i = 0; i < segments; i++) {
  const angle = (i / segments) * Math.PI * 2;
  const nextAngle = ((i + 1) / segments) * Math.PI * 2;
  
  const x = centerX + Math.cos(angle + Math.PI / segments) * radius;
  const y = centerY + Math.sin(angle + Math.PI / segments) * radius;
  const segAngle = angle + Math.PI / segments;
  
  // Cada segmento é um retângulo fino rotacionado
  const wall = Matter.Bodies.rectangle(
    x, y,
    2, // largura (fino = invisível)
    radius * 2 * Math.sin(Math.PI / segments) + 4, // altura = comprimento do arco
    {
      isStatic: true,
      angle: segAngle + Math.PI / 2,
      render: { visible: false } // invisível — o visual é o PNG do globo
    }
  );
  Matter.Composite.add(world, wall);
}
```

**32 segmentos** é suficiente para parecer um círculo perfeito. As bolas nunca "escapam" pelas frestas porque os segmentos se sobrepõem ligeiramente.

## A1.3 Performance Real no Browser

Dados do Matter.js em benchmarks reais:
- **20 bodies (bolas)**: ~0.3ms por tick → 60fps tranquilo
- **60 bodies**: ~1.2ms por tick → 60fps OK em desktop, ~40fps em NUI
- **100 bodies**: ~3ms por tick → 30fps em desktop, <20fps em NUI (inaceitável)

Para o Blackout Casino: **20 bolas = escolha ótima.** Visual cheio o suficiente, performance garantida até no NUI.

# A5. APROFUNDAMENTO — PROVABLY FAIR: IMPLEMENTAÇÃO NODE.JS REAL
# ═══════════════════════════════════════════════════

A pesquisa original tinha pseudocódigo. Aqui está a implementação real em Node.js:

```javascript
const crypto = require('crypto');

class LotteryProvablyFair {
  /**
   * Gera um server_seed random de 256 bits
   */
  static generateServerSeed() {
    return crypto.randomBytes(32).toString('hex');
  }

  /**
   * Gera hash público do server_seed (publicado ANTES do sorteio)
   */
  static hashServerSeed(serverSeed) {
    return crypto.createHash('sha256').update(serverSeed).digest('hex');
  }

  /**
   * Gera client_seed (pode ser fornecido pelo jogador ou gerado automaticamente)
   */
  static generateClientSeed() {
    return crypto.randomBytes(16).toString('hex');
  }

  /**
   * Gera o resultado do sorteio de forma determinística e verificável
   * @param {string} serverSeed - seed do server (secreto até após o sorteio)
   * @param {string} clientSeed - seed do client (público)
   * @param {number} nonce - incrementa a cada rodada do jogador
   * @param {number} poolSize - tamanho do pool (60 para Mega-Sena)
   * @param {number} pickCount - quantos números sortear (6)
   * @returns {number[]} array de números sorteados
   */
  static generateNumbers(serverSeed, clientSeed, nonce, poolSize = 60, pickCount = 6) {
    const combined = `${serverSeed}:${clientSeed}:${nonce}`;
    const hash = crypto.createHmac('sha256', serverSeed)
      .update(`${clientSeed}:${nonce}`)
      .digest('hex');
    
    const numbers = [];
    let cursor = 0;
    
    while (numbers.length < pickCount && cursor < hash.length - 7) {
      // Pega 8 chars hex (32 bits) do hash
      const hex = hash.substring(cursor, cursor + 8);
      const decimal = parseInt(hex, 16);
      
      // Mapeia para número 1-poolSize
      const number = (decimal % poolSize) + 1;
      
      // Garante número único
      if (!numbers.includes(number)) {
        numbers.push(number);
      }
      
      cursor += 8;
    }
    
    return numbers;
  }

  /**
   * Gera o multiplicador da Mega Ball
   */
  static generateMultiplier(serverSeed, clientSeed, nonce) {
    const hash = crypto.createHmac('sha256', serverSeed)
      .update(`${clientSeed}:${nonce}:megaball`)
      .digest('hex');
    
    const decimal = parseInt(hash.substring(0, 8), 16);
    const roll = decimal % 100; // 0-99
    
    // Tabela de multiplicadores (configurável via casino_lottery_config)
    if (roll < 50) return 2;    // 50% chance
    if (roll < 75) return 3;    // 25% chance
    if (roll < 90) return 5;    // 15% chance
    if (roll < 97) return 10;   // 7% chance
    if (roll < 99) return 25;   // 2% chance
    return 50;                   // 1% chance
  }

  /**
   * Verifica resultado (jogador pode chamar para confirmar)
   */
  static verify(serverSeed, clientSeed, nonce, expectedNumbers) {
    const hashCheck = this.hashServerSeed(serverSeed);
    const generated = this.generateNumbers(serverSeed, clientSeed, nonce);
    return {
      hashValid: true, // jogador compara hashCheck com o hash publicado antes
      numbersMatch: JSON.stringify(generated) === JSON.stringify(expectedNumbers),
      generatedNumbers: generated,
      serverSeedHash: hashCheck
    };
  }
}
```

**Fluxo de Verificação pelo Jogador:**
1. Antes do sorteio: server publica `seedHash = SHA256(serverSeed)` — jogador vê "abc123..."
2. Jogador confirma aposta com seu `clientSeed` (gerado ou manual)
3. Server sorteia usando `HMAC_SHA256(serverSeed, clientSeed:nonce)`
4. Após resultado: server revela `serverSeed`
5. Jogador verifica: `SHA256(serverSeed_revelado) === seedHash_publicado`?
6. Jogador recalcula: `HMAC_SHA256(serverSeed, clientSeed:nonce)` → mesmos números?

Se ambas verificações passam, o sorteio é comprovadamente justo.
