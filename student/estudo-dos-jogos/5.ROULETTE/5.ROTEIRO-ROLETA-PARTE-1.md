## RELATORIO DA PESQUISA (REGRA X0)

**Pesquisas realizadas para o Roteiro:**
1. `Framer Motion 11 animate rotate performance 2025 2026` - 10 resultados
2. `Web Crypto API HMAC SHA256 SubtleCrypto Chromium CEF FiveM NUI 2025` - 10 resultados
3. `FiveM oxmysql JSON column query MariaDB async callback 2025 2026` - 10 resultados

**Fontes relevantes:**
- motion.dev (changelog mar/2026): Motion v12.34+ com `useScroll` hardware-accelerated. `rotate` e `transform` sao compositor-thread — GPU-accelerated sem layout/paint. Versao 11 estavel, API de `animate={{ rotate }}` inalterada
- MDN Web Crypto API (atualizado set/2025): `crypto.subtle.sign('HMAC', key, data)` retorna Promise(ArrayBuffer). Suportado em Chromium 37+ (NUI FiveM e Chromium 93+). `importKey('raw', ...)` para HMAC-SHA256 funcional
- OxMySQL docs (coxdocs.dev): MariaDB recomendado sobre MySQL 8. JSON fields com `DEFAULT '[]'` funciona em MariaDB mas NAO em MySQL 8 (Longtext/JSON sem default). Queries async com `exports.oxmysql` ou `MySQL.Async` compat layer
- Refine.dev (set/2025): FM v11 com "improved layout animations" e "better rendering pipelines for large numbers of animated elements"

**O que a pesquisa trouxe de novo:**
1. **`crypto.subtle` vs `crypto.createHmac`**: No browser (NUI), usar Web Crypto API async. No server (Node.js FiveM), usar `require('crypto').createHmac`. Dois codigos diferentes para o mesmo algoritmo
2. **JSON DEFAULT em MariaDB vs MySQL 8**: MariaDB aceita `DEFAULT '[]'` e `DEFAULT '{}'`. MySQL 8 rejeita default em JSON columns. Nosso server usa MariaDB — seguro
3. **Motion rotate e GPU-accelerated**: `animate={{ rotate }}` roda no compositor thread. Sem impacto em layout/paint. Seguro para spin de 4-5s com 1800-2880 graus
4. **OxMySQL `exports.oxmysql`**: Sintaxe moderna: `exports.oxmysql.query_async(query, params)`. Compativel com `MySQL.Async.fetchAll` para bases vRP

**Sem pesquisa teria errado:**
- `crypto.createHmac` no client NUI — nao existe no browser, crash
- `DEFAULT '[]'` em MySQL 8 — silently fails, config vazia
- Tentaria usar CSS animation em vez de Framer Motion rotate — menos controle de easing

    result_number          TINYINT UNSIGNED NOT NULL             COMMENT '0-36',

    INDEX idx_result (result_number)

COMMENT='Historico de spins da Roleta';

    revealed_at            DATETIME       DEFAULT NULL           COMMENT 'Quando a seed foi revelada (rotacao)',

) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
COMMENT='Seeds Provably Fair da Roleta';
