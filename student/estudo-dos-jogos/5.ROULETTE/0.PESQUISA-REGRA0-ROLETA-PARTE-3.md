## 6.1 Como Funciona na Roleta (baseado em Stake.com, Provably.com, Gamingtec)

**Variaveis:**
- Server Seed: string aleatoria gerada pelo servidor. Hash SHA-256 publicado ANTES do spin.
- Client Seed: string gerada pelo browser do jogador OU customizada pelo jogador.
- Nonce: inteiro que comeca em 0 e incrementa +1 a cada spin. Garante unicidade.

**Processo por spin:**

```
ANTES DA SESSAO:
  1. Server gera serverSeed (ex: "a7b3c9d4e1f2...")
  2. Server calcula hash: SHA256(serverSeed) -> "8f2a1b..."
  3. Hash e publicado ao jogador (nao o seed, so o hash)
  4. Jogador ve o hash e pode salvar para verificar depois

A CADA SPIN:
  5. Jogador confirma clientSeed (auto-gerado ou custom)
  6. Sistema combina: HMAC-SHA256(serverSeed, clientSeed + ":" + nonce)
  7. Resultado: hex string de 64 chars (ex: "3a9f1c2b...")
  8. Primeiros 8 chars hex -> parseInt(hex, 16) -> inteiro grande
  9. Inteiro % 37 -> numero 0-36 (European Roulette)
  10. Este e o numero onde a bola para
  11. Nonce incrementa +1

APOS SESSAO (ou quando jogador troca seed):
  12. Server revela serverSeed original (nao-hashado)
  13. Jogador pode verificar: SHA256(serverSeed) == hash publicado antes?
  14. Se sim: server NAO alterou o seed durante o jogo
  15. Jogador pode recalcular cada spin: HMAC-SHA256(serverSeed, clientSeed:nonce) e confirmar que os resultados batem
```

**Implementacao para o Blackout Casino (server handler JS):**

```javascript
// Pseudocodigo do calculo - sera expandido no Roteiro
const crypto = require('crypto');

function gerarResultadoRoleta(serverSeed, clientSeed, nonce) {
  const mensagem = clientSeed + ':' + nonce;
  const hmac = crypto.createHmac('sha256', serverSeed).update(mensagem).digest('hex');
  const inteiro = parseInt(hmac.substring(0, 8), 16);
  const numero = inteiro % 37; // 0-36 para European
  return numero;
}

function gerarLuckyNumbers(serverSeed, clientSeed, nonce) {
  // Usa bytes DIFERENTES do mesmo hash para Lucky Numbers
  const mensagem = clientSeed + ':' + nonce + ':lightning';
  const hmac = crypto.createHmac('sha256', serverSeed).update(mensagem).digest('hex');

  // Quantidade de Lucky Numbers: 1-5 (ponderado: 2-3 mais comum)
  const qtdByte = parseInt(hmac.substring(0, 2), 16);
  const qtd = qtdByte < 51 ? 1 : qtdByte < 153 ? 2 : qtdByte < 230 ? 3 : qtdByte < 248 ? 4 : 5;
  // ~20% chance de 1, ~40% de 2, ~30% de 3, ~7% de 4, ~3% de 5

  const luckyNumbers = [];
  for (let i = 0; i < qtd; i++) {
    const offset = 2 + (i * 4);
    const numByte = parseInt(hmac.substring(offset, offset + 4), 16);
    let num = numByte % 37;
    // Evitar duplicatas
    while (luckyNumbers.includes(num)) {
      num = (num + 1) % 37;
    }
    luckyNumbers.push(num);
  }

  // Multiplicadores para cada Lucky Number
  const multiplicadores = [50, 100, 200, 300, 400, 500];
  const luckyData = luckyNumbers.map((num, idx) => {
    const multOffset = 22 + (idx * 2);
    const multByte = parseInt(hmac.substring(multOffset, multOffset + 2), 16);
    const multIdx = Math.floor((multByte / 256) * multiplicadores.length);
    return { number: num, multiplier: multiplicadores[multIdx] };
  });

  return luckyData;
}
```

## 7.4 Simulacao Economica

**Cenario: 100 jogadores apostando media de 500 GCoins/spin, 10 spins/sessao:**
- Volume por sessao: 100 x 500 x 10 = 500.000 GCoins
- House edge 2.70%: servidor arrecada ~13.500 GCoins por sessao
- Por dia (3 sessoes/dia media): ~40.500 GCoins/dia
- Por mes: ~1.215.000 GCoins/mes

**Modo Lightning adiciona volatilidade mas NAO muda o edge geral significativamente.**
