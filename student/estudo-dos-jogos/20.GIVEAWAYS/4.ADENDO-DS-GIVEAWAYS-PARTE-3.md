## 3.1 Estado LISTA IDLE (navegando sorteios disponiveis)

```
FUNDO: bg-casino.png + vinheta (inset 150px)

COR DOMINANTE: ouro #D4A843 (bordas, titulos) + verde #00E676 (premio, CTA)
SOM AMBIENTE: nenhum (silencioso, jogador decidindo)
ANIMACOES: shineSweep 4s loop no card ativo, stagger nos cards mini (0.05s delay), countdown flip cada segundo
MOVIMENTO: sutil — cards mini hover translateY -2px, progress bar fill transiciona suave
INTENSIDADE: baixa — estado de navegacao calmo, sem pressao
```

## 3.2 Estado URGENTE (countdown < 60 minutos)

```
FUNDO: mesmo, sem mudanca
ELEMENTOS ATIVOS: mesmos + badge URGENTE pulsando rapido
COR DOMINANTE: ouro + vermelho #FF1744 (countdown digits, badge)
SOM: countdown_urgency.mp3 loop sutil (heartbeat)
DIFERENCA: countdown digits ficam vermelhos, pulseUrgent 0.5s loop, badge URGENTE 0.5s pulse rapido
ANIMACOES: tudo de IDLE + pulseUrgent nos digits + badge + glow vermelho sutil no card
INTENSIDADE: media-alta — pressao temporal, urgencia visual crescente
TRANSICAO: apos countdown zerar → play('draw_start') → estado DRAWING
```

## 3.3 Estado DRAWING (drum em andamento, resultado desconhecido)

```
FUNDO: overlay rgba(5,5,5,0.92) cobre lista (blur 4px no lobby atras)
ELEMENTOS ATIVOS: drum viewport central, frame dourado, fade gradients, ticket items scrollando
COR DOMINANTE: dourado #FFD700 (frame glow) + preto profundo
SOM SEQUENCIA: draw_start.mp3 → drum_scroll.mp3 loop → drum_decelerate.mp3 → drum_stop.mp3
ANIMACOES:
  - FASE RAPIDA (0-2s): translateY linear, track blur(2px), items indistinguiveis
  - FASE DECELERA (2-5s): translateY ease [0.2,0,0,1], blur remove gradual, items ficam legiveis
  - FASE PARADA (5-5.3s): frame bounce scale 1→1.08→1, glowPulseGold intensifica
  - PAUSA DRAMATICA (5.3-5.5s): silencio, item winner highlight dourado
MOVIMENTO: intenso no inicio, desacelera ate parar — arco dramatico
INTENSIDADE: maxima — impossivel desviar olhar do drum
TRANSICAO: apos pausa → winner_reveal.mp3 ou winner_self.mp3 → overlay resultado
```

```
WIN (jogador eh o vencedor):

LOSE (outro jogador ganhou):

  COR DOMINANTE: #A8A8A8 (heading) + #D4A843 (nome) — sem glow intenso
  SOM: nenhum especifico (silencio respeitoso)
  ANIMACOES: fade simples 0.5s, sem confete, sem pulse, sem glow. Tudo opacity transition tween
  INTENSIDADE: minima — rapido, nao punitivo, nao humilha

TRANSICAO AMBOS: botao VOLTAR AO LOBBY aparece com delay 1.5s (opacity 0→1, y 20→0)

```

```
            play('ticket_purchase').

            INSERT tickets no MySQL. UPDATE saldo jogador (-quantity * price).
            Retorna: { success, ticketNumbers[], newBalance }.

t=80ms      Modal fecha com exit animation.
            Toast: "X tickets comprados!"
            Progress bar atualiza (fill width transiciona 0.5s).
            Card refresh com novos dados de chance (%).

            Countdown flip cards rodam a cada segundo.
            Jogador pode comprar mais tickets, clamar gratuito.
            Outros jogadores compram tickets (progress bar atualiza via polling/NUI event).

FASE DRAW (countdown zerou):
```
